# 담양군 중대재해예방 시스템 — 디자인 시스템 v0.2

> Figma 시안(`디자인_대시보드`, fileKey `nhJRPPCgdeVuqlaUoI58Q4`)에서 추출한 실측 토큰을 기준으로 재정리.
> v0.1 추정치는 폐기하고 시안 변수명을 1차 출처로 둠. 코드 토큰명(`--brand-*`)은 하위호환을 위해 별칭으로 유지.

---

## 1. 디자인 원칙

| 원칙 | 설명 |
|---|---|
| **신뢰성 (Trust)** | 공공/안전 분야에 어울리는 차분한 그린(`#209F84`). 채도 폭주 지양. |
| **명확성 (Clarity)** | 상태/카테고리를 색·뱃지로 즉시 구분. 숫자·KPI를 큰 폰트로 시각 강조. |
| **밀도 (Density)** | 행정업무 특성상 한 화면에서 다량 정보 비교. 카드(20px radius) 기반 그리드. |
| **일관성 (Consistency)** | 상태값(대기/검토/조치/완료/반려)은 화면 무관 같은 색 페어 사용. |

| 토큰 | 값 |
|---|---|
| `shadow-online` | `0 0 0 2px #FFFFFF` (온라인 점 둘레) |

## 6.21 컴포넌트 카탈로그 (마크업 패턴)

이미 글로벌(`style.css`)에 등록된 컴포넌트들의 사용 패턴. 신규 화면 작업 시 본 카탈로그의 마크업을 그대로 차용하면 일관성이 보장됩니다.

### A. 페이지 헤더 (자동 주입 권장)

`<body>`에 `data-page-title` / `data-page-subtitle` 만 부여하면 `layout.js`가 자동으로 dot + h1 + 부제 마크업을 주입합니다. 백 링크가 필요하면 `data-back-href` / `data-back-label`.

```html
<body data-dy-page="rsk-improve"
      data-page-title="개선조치"
      data-page-subtitle="위험성평가에서 자동 생성된 개선조치 진행 관리"
      data-back-href="risk-list.html"
      data-back-label="평가 목록">
```

> 우상단에 등록 버튼 등 부가 액션이 필요한 경우엔 페이지가 직접 `.rsk-page-head` 구조로 마크업 (자동 주입은 `dy-page-title`이 이미 있으면 skip).

### B. 필터 행 (`.rsk-filter-row`)

```html
<div class="rsk-filter-row">
    <div class="form-group">
        <span class="form-label-inline">부서</span>
        <select class="select"><option>전체</option>...</select>
    </div>
    <!-- ... 다른 셀렉트들 ... -->
    <div class="search-bar">
        <input class="search-input" type="text" placeholder="검색어">
        <button class="search-submit" type="button">...검색</button>
    </div>
    <button class="clear-filters">초기화</button>
</div>
```

JS 빌더: `DYLayout.renderFilterRow([{label,options:[...]}], {search:{placeholder}, clear:true})`

### C. 합본 검색 바 (`.search-bar`)

셀렉트 + input + 그린 검색 버튼이 하나의 라운드 사각형(`radius-button`)으로 묶인 형태. 단독 사용도 가능.

### D. Stat 그리드 (`.statbox-grid.cols-{2|3|4|5}`)

3·4·5분할 통계 박스 배치. 자식은 `.statbox` 또는 `.rsk-level-cell`.

```html
<div class="statbox-grid cols-4">
    <div class="statbox danger">
        <div class="statbox-num">3</div>
        <div class="statbox-label">기한초과</div>
    </div>
    <!-- ... 3개 더 ... -->
</div>
```

반응형: 1280 이하에서 cols-4/5 → 2열, 768 이하 → 1열.

### E. 페이지네이션

**권장: 데이터 마커** — `layout.js`가 mount 시 자동 렌더:
```html
<div data-pagination data-current="1" data-total="4"></div>
```

JS 빌더 (동적 렌더링 시):
```js
section.innerHTML += DYLayout.renderPagination({ current: 1, total: 4 });
```

직접 마크업 (특수 케이스):
```html
<div class="pagination">
    <a class="pagination-item is-disabled" href="#">‹</a>
    <a class="pagination-item is-active" href="#">1</a>
    <a class="pagination-item" href="#">2</a>
    ...
    <a class="pagination-item" href="#">›</a>
</div>
```

## 10. 접근성 (Accessibility)

- 색 대비: 텍스트/배경 4.5:1 이상 (WCAG AA).
  - 주의: `#EA8F56` on `#FFF9EF` 약 3.0:1 — chip 텍스트는 라벨로만 사용, 본문 텍스트 색으로 단독 사용 금지.
- 상태 색은 라벨 병기 (색만으로 의미 전달 금지).
- 포커스 링: 모든 인터랙티브 요소 `outline 2px --main + offset 2`.
- 폼: label-control 1:1 매핑, error 메시지 텍스트 동반.
- 키보드: Tab 순서 시각 흐름 순응, 모달 focus trap.
