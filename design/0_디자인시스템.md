# 담양군 중대재해예방 시스템 — 디자인 시스템 (초안)

> 시안 3종(대시보드 / 목록 / 상세)에서 추출한 초안. 추후 Figma 토큰화 시 본 문서를 기준선으로 사용.

---

## 1. 디자인 원칙

| 원칙 | 설명 |
|---|---|
| **신뢰성 (Trust)** | 공공기관·안전 분야에 어울리는 차분한 그린 톤. 과한 채도 지양. |
| **명확성 (Clarity)** | 상태/카테고리를 색·뱃지로 즉시 구분. 숫자·KPI를 시각적으로 강조. |
| **밀도 (Density)** | 행정업무 특성상 한 화면에서 다량의 정보 비교 가능해야 함. 카드/표 기반 레이아웃. |
| **일관성 (Consistency)** | 상태값(승인/검토중/초안/폐기 등)은 화면 무관 같은 색·뱃지 형태 유지. |

## 6. 컴포넌트

### 6.1 Header / GNB
- 헤더: 흰 배경, 좌측 로고(아이콘 + 텍스트), 우측 사용자 드롭다운(아바타 + 이름 + 소속)
- GNB: `--brand-700` 배경, 화이트 텍스트, 활성 메뉴는 하단 4px 화이트 인디케이터

### 6.2 Sidebar
- 폭 220px, 흰 배경
- 섹션 타이틀: `gray-700`, 14/700
- 메뉴 아이템: 14/400, 아이콘 16px + 라벨
- **Active**: 좌측 4px 브랜드 인디케이터 바 + `--brand-50` 배경 + 텍스트 `--brand-700`

### 6.3 Card
- 배경 `white`, radius 12, shadow-sm, padding 24
- 헤더: 카드 타이틀 (h2) + 우측 보조 액션 ("바로가기 >")

### 6.4 Button

| 종류 | 배경 | 텍스트 | 보더 |
|---|---|---|---|
| Primary | `--brand-600` | white | none |
| Primary (hover) | `--brand-700` | white | none |
| Secondary | white | `--gray-700` | `--gray-300` |
| Ghost | transparent | `--brand-600` | none |
| Disabled | `--gray-100` | `--gray-500` | none |

- 사이즈: `sm` 32 / `md` 40 / `lg` 48 (높이)
- 패딩: 가로 16(sm) / 20(md) / 24(lg)
- radius 8, 아이콘 좌측 8px 간격

### 6.5 Badge (상태/카테고리)

- Shape: `radius-pill`, 높이 24~28, 좌우 padding 10

### 6.6 Filter Chip
- 비활성: `white` 배경 / `--gray-300` 보더 / `--gray-700` 텍스트
- 활성: `--brand-600` 배경 / white 텍스트
- 카운트 표시: 라벨 뒤 공백 + 숫자 (`전체 12`)

### 6.7 Tab
- 언더라인 스타일
- 활성: 텍스트 `--gray-900` / 하단 2px `--brand-600` 인디케이터
- 비활성: `--gray-500`

### 6.8 KPI Card
```
┌────────────────────────────────┐
│ ☑ 위험성평가 완료율     [72%] │
│ 목표 100% · 현재 72%           │
│ ████████░░░░ (진행률 바)       │

```
- 우측 상단: 진행률 뱃지 (달성 정도에 따라 색 변화: ≥80% success / 40~79% warning / <40% danger)
- 진행률 바: 8px 높이, `--gray-200` 트랙

### 6.9 Progress Bar (다중 세그먼트)
- 높이 28, radius-pill
- 세그먼트: 완료(`--brand-600`) · 진행(`--brand-400`) · 계획(`--gray-200`)
- 각 세그먼트 내부에 인라인 라벨 + 퍼센트
- 우측 상단에 총 달성률(63%) 강조

### 6.10 Stat Box (대기중/검토중/조치중/...)
- 폭 100~120, 높이 80
- 배경: 상태색의 100 톤 / 텍스트: 700 톤
- 상단 큰 숫자 (24/700), 하단 라벨 (12/500)
- 가로 나열 시 gap 12

### 6.11 Table
- 컬럼 헤더: `--gray-500`, 12/600, padding 12 16
- 행: 흰 배경, 보더 하단 `--gray-200`, padding 14 16
- 행 hover: `--gray-50`
- 셀 내 뱃지 사용: 카테고리 뱃지는 첫 칸, 상태 뱃지는 별도 칸
- **Alert 행**: 기한초과 등 위험 행은 상단에 빨강 배너(`#FBDCDC` 배경 / 좌측 ⓘ 아이콘) + 해당 행 배경 `#FFF6F6`

### 6.12 Search Bar
- 좌측 드롭다운(필터 카테고리) + 입력 필드 + 우측 검색 버튼

- 검색 버튼: primary 그린

### 6.13 Input / Form

- 인풋: 높이 44, padding 12 14, 보더 `--gray-300`, radius 8

### 6.14 File Attachment Row
- PDF 아이콘 뱃지 (`--status-danger` 톤) + 파일명 + 용량 + 우측 [미리보기] [다운로드] 버튼

### 6.15 Chart

- 기간 토글(6개월/12개월): 세그먼트 컨트롤 — 활성 `--brand-600` / 비활성 `--gray-100`

### 6.16 Alert Banner

## 7. 상태 매핑 가이드

## 9. 반응형 (초안)

| Breakpoint | 폭 | 레이아웃 |
|---|---|---|
| `xl` | ≥1280 | 사이드바 + 12-col |
| `lg` | 1024~1279 | 사이드바 collapse 옵션, 12-col |
| `md` | 768~1023 | 사이드바 off-canvas, KPI 카드 2x2 |
| `sm` | <768 | 단일 컬럼, GNB 햄버거 |

> 본 시스템은 행정 사용자가 PC 환경(≥1280)에서 주로 사용한다는 전제 하에 우선순위는 xl/lg.

## 10. 접근성 (Accessibility)

- 색 대비: 텍스트/배경 4.5:1 이상 (WCAG AA). 상태 뱃지 색은 텍스트만으로 의미 구분 가능하도록 라벨 병기 (색만으로 의미 전달 금지).
- 포커스 링: 모든 인터랙티브 요소 `outline 2px --brand-600 + offset 2`
- 폼: label-control 1:1 매핑, error 메시지 텍스트 동반
- 키보드: Tab 순서 시각 흐름 순응, 모달 focus trap
