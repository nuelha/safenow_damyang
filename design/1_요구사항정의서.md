## 0. 문서 개요

### 0.1 목적

본 문서는 담양군 중대재해예방 통합관리시스템(이하 "본 시스템")의 기능적·비기능적 요구사항을 정의한다. 발주처(담양군청)와 수행사(㈜다온플레이스) 간의 공통 이해 기반을 제공하며, 후속 산출물(메뉴구조도, DB설계서, 인터페이스정의서, 화면설계서)의 작성 근거가 된다.

### 0.2 적용 범위

- 적용 대상: 담양군청 본청·읍·면·출장소·소속기관 전 직원, 도급업체, 컨설팅업체
- 적용 업무: 중대재해처벌법 시행령 제4조 의무이행 관리, 산업안전보건법 의무이행 관리, 안전보건경영 전반
- 적용 기간: 시스템 구축 완료 시점부터 유지보수 기간 종료 시점까지

### 0.3 용어 정의

| 용어 | 정의 |
|------|------|
| 관리주체 | 시스템의 최상위 단위. 본 시스템에서는 담양군청 단일 instance |
| 담당부서 | 행정포털 조직도와 연동되는 2단계 단위. 자치행정과·농업기술센터 등 |
| 관리대상 | 본 시스템의 3단계 작업 단위. 시설·업무·사업을 포괄하는 개념 |
| 시설형 관리대상 | `target_type='FACILITY'`. 청사·정수장·차량 등 물리적 시설 |
| 업무형 관리대상 | `target_type='TASK'`. 행정지원 업무 등 |
| 사업형 관리대상 | `target_type='BUSINESS'`. 농기계 임대사업 등 |
| 안전NOW | ㈜다온플레이스의 기존 B2B SaaS 제품. 본 시스템의 기능 참조 모델 |
| 새올 포틀릿 | 행정포털 내 알림 표시 영역. 본 시스템 알림의 적재 대상 |
| 온나라 결재 | 정부 표준 전자결재 시스템. 본 시스템은 안내 팝업만 표시하고 자체 결재 미구현 |
| 시기도래 업무 | 마감일이 가까워진 업무. 1개월·1주·당일·초과 임계값으로 색상 구분 |

## 1. 시스템 개요

### 1.1 시스템 목적

1. 중대재해처벌법 시행령 제4조에 따른 경영책임자 의무이행 사항을 통합 관리
2. 산업안전보건법상 안전관리 의무이행 사항을 체계적으로 기록·관리
3. 담양군 관할 시설·업무·사업에 대한 위험성평가·점검·이행관리 전 과정 디지털화
4. 도급·용역·위탁 사업의 안전보건 점검 체계 구축
5. 종사자 의견청취·산업안전보건위원회 운영 체계 구축

### 1.2 시스템 구성도 (논리)

```
[행정포털] ──SSO/조직도──┐
[FMS]      ──시설정보──┤
[차세대 e호조] ──도급사업── ├→ [담양 중대재해통합관리시스템] ←→ [도급업체/컨설팅업체]
[온나라]   ←─결재 진행──┤                    │
[새올]     ←─알림 적재──┘                    ↓
                                        [데이터베이스]
```

### 1.3 이해관계자

| 분류 | 대상 | 권한 코드 |
|------|------|---------|
| 경영책임자 | 군수 / 부군수 | CEO |
| 부서 책임자 | 실장 / 과장 / 단장 / 소장 | GM |
| 안전관리자 | 팀장 / 안전보건담당자 | SM / SHM |
| 종사자 | 공무원·근로자 | WKR |
| 도급업체 | 외부 협력업체 | SUB |
| 컨설팅업체 | 외부 안전관리 컨설팅 | CON |

### 2.2 외부 연동 시스템

| 외부 시스템 | 연동 목적 | 연동 방식 |
|-----------|---------|---------|
| 담양군 행정포털 | SSO 인증 + 조직도 동기화 | API (프로토콜 협의) |
| FMS 시설물관리시스템 | 시설형 관리대상 자동 동기화 | API (스펙 협의) |
| 차세대 e호조 | 도급계약 자동 import + 위험성평가 자동 생성 | API (스펙 협의) |
| 온나라 전자결재 | 결재 진행 (시스템에서는 안내만) | 미연동 (안내 팝업) |
| 새올 시스템 | 알림 표시 영역(포틀릿) 적재 | API 또는 DB 직접 적재 (협의) |

### 2.3 운영 환경 (제안 기준, 협의 사항)

- 서버: 클라우드 또는 IDC 호스팅
- DBMS: 관계형 데이터베이스 (PostgreSQL / MySQL / Oracle 중 협의)
- 개발 프레임워크: 전자정부 표준프레임워크 (백엔드) + 모던 SPA 프레임워크 (프론트엔드)
- 클라이언트: 웹 브라우저 (Chrome / Edge 최신 2개 메이저 버전), 모바일 웹 대응

### SFR-002 — 중대재해 관리대상 현황정보 관리 🆕

| 항목 | 내용 |
|------|------|
| ID | SFR-002 |
| 분류 | 신규 |
| 우선순위 | ★★★ (Phase 1) |
| 외부 연동 | FMS |

**요구사항**
1. 담양군 관할 시설·업무·사업을 "관리대상" 단위로 등록·관리
2. 관리대상은 `시설(FACILITY) / 업무(TASK) / 사업(BUSINESS)` 3종 type 구분
3. 시설형 관리대상은 중대재해처벌법 분류(공중이용시설/원료·제조물/공중교통수단)와 시설물안전법 종별(1·2·3종) 단일 분류 보유
4. 시설형 관리대상은 FMS와 자동 연동 (`fms_id`)
5. 관리대상 신규 등록 시 해당 부서의 부서장을 관리책임자(RESPONSIBLE)로 자동 매핑
6. 관리대상 목록·상세·등록·수정·비활성 기능
7. FMS 동기화 이력·수동 동기화 기능
8. 관리대상 일괄등록 (SFR-016 연계)

**관련 결정사항**: 결정사항 #3, #4, #6, #7 참조

## 8. 가정 및 의존성

### 8.1 가정 (Assumptions)

1. 행정포털 SSO는 OAuth2 또는 SAML 표준 지원
2. FMS는 시설물 정보를 REST API로 제공
3. 차세대 e호조는 도급계약 정보를 REST API로 제공
4. 새올 포틀릿은 외부 시스템 알림 적재 인터페이스를 제공
5. 발주처는 본 시스템 운영 인프라(서버·DBMS·SMTP)를 제공
6. 발주처는 표준 디자인 가이드를 제공하거나 위임

### 8.2 의존성 (Dependencies)

| 의존 항목 | 영향받는 SFR | 미해소 시 |
|---------|-----------|---------|
| 차세대 e호조 API 스펙 | SFR-007, 010, 013 | 도급 자동화 부분 placeholder |
| FMS API 스펙 | SFR-002, 016, 020 | 시설 자동 동기화 미동작 |
| 행정포털 SSO 프로토콜 | SFR-015 | 내부 직원 로그인 미동작 (전체 시스템 진입 불가) |
| 행정포털 조직도 API 스펙 | SFR-006 | 부서 마스터 수동 등록 필요 |
| 새올 포틀릿 연계 방식 | SFR-017 | 알림 시스템 내 화면에서만 확인 |
| SMTP 서버 설정 | SFR-015, 017 | 외부 사용자 이메일 발송 미동작 |
| 권한 매핑 규칙 초기값 | SFR-015 | 권한 수동 부여 |

### 8.3 위험 (Risks)

| 위험 | 영향 | 완화 방안 |
|------|------|---------|
| 외부 시스템 스펙 수령 지연 | 일정 지연 | 외부 의존 부분 placeholder로 구현, 스펙 수령 후 보강 |
| 외부 시스템 API 응답 불안정 | 데이터 정합성 | 재시도 큐 + 운영자 알림 + 수동 동기화 옵션 |
| 행정포털 직급 ↔ 권한 매핑 누락 | 권한 부여 오류 | 초기 매핑 규칙 협의 + 수동 보정 UI |
| 5분류 평가서·서약서 표준 부재 | 콘텐츠 작성 지연 | 발주처 협의 + 안전NOW 표준 양식 참조 |
