# 담양군 중대재해통합관리시스템 — 메뉴구조도·권한매트릭스

> 문서 ID: IA-001
> 버전: **v1.2**
> 작성일: 2026-05-12
> 작성자: ㈜다온플레이스
> 관련 문서: SRS-001 (요구사항정의서)

## 0. 문서 개요

### 0.1 목적

본 문서는 담양군 중대재해통합관리시스템의 정보구조(Information Architecture, IA)를 정의한다. GNB·LNB 메뉴 트리, 권한별 메뉴 접근 매트릭스, 화면 ID 체계, 데이터 스코프 정책을 포함하며, 화면설계서 작성의 기준이 된다.

### 0.2 적용 범위

- GNB 메뉴 (상단 주요 메뉴)
- LNB / 탭 (모듈 내부 보조 메뉴)
- 권한별 메뉴 접근 매트릭스
- 화면 ID 체계 및 URL 패턴

## 1. 권한 코드 체계

### 1.1 권한 코드 정의

| 코드 | 표시명 | 인증 경로 | 데이터 스코프 |
|------|--------|---------|------------|
| CEO | 군수 / 부군수 | 행정포털 SSO | 1단계 (군청 전체) |
| GM | 실/과/단/소장 | 행정포털 SSO | 2단계 (본 부서 전체) |
| SM | 팀장 | 행정포털 SSO | 3단계 (담당 관리대상) |
| SHM | 안전보건담당자 | 행정포털 SSO | 3단계 (담당 관리대상) |
| WKR | 근로자 (공무원) | 행정포털 SSO | 3단계 (본인 소속 관리대상) |
| SUB | 도급업체 | 별도 로그인 (LOCAL) | 계약된 관리대상에 한정 |
| CON | 컨설팅업체 | 별도 로그인 (LOCAL) | 위임된 부서·관리대상에 한정 |

### 1.2 권한 상속 규칙

- CEO ⊃ GM ⊃ SM/SHM ⊃ WKR (내부 직원 간)
- SUB, CON은 별도 인증 경로 — 내부 권한과 상속 관계 없음
- SHM은 SM과 동급 권한 + 안전보건 모듈 추가 권한

## 4. 화면 ID 체계

### 4.1 명명 규칙

```
{모듈 약어}{번호}-{화면 유형}[-{보조 코드}]

모듈 약어: 3~4자리 영문 대문자 (ORG/TGT/AUTH/NTF/RSK/CON 등)
번호: 2자리 숫자 (01~99)
화면 유형:
  V — View (현황·대시보드)
  L — List (목록)
  D — Detail (상세)
  F — Form (등록·수정 폼, 풀페이지)
  M — Modal (모달, 보조 모달은 M2, M3 등)
  S — Settings (설정)
  EXEC — 실행 (점검 실시 등)

예시:
  ORG01-V    → 전담조직 모듈 1번 화면, View 유형
  ORG02-M2   → 전담조직 모듈 2번 화면의 2번째 모달
  RSK02-D    → 위험성평가 모듈 2번 화면, Detail 유형
  CON09-L    → 도급관리 모듈 9번 화면 (TBM), List 유형
```

### 4.3 URL 패턴 (v1.1 — LNB 사이드 메뉴 일원화)

| 패턴 | 용도 | 예시 |
|------|------|------|
| `/{module-name}` | 모듈 기본 진입 | `/organization` → ORG01-V |
| `/{module-name}/{sub}` | 모듈 LNB 하위 화면 (사이드 메뉴) | `/organization/members` → ORG02-L |
| `/{module-name}/:id` | 상세 화면 | `/targets/:id` → TGT02-D |
| `/{module-name}/new` | 신규 등록 폼 | `/contracts/new` → CON04-F |
| `/{module-name}/:id/edit` | 수정 폼 | `/contracts/:id/edit` → CON04-F |
| `/{module-name}/:id/step{N}` | 다단계 폼 | `/contracts/:id/step2` → CON05-F |
| `/admin/{section}` | 시스템 관리 | `/admin/external-users` → AUTH06-S |
| `?section=` / `?type=` 등 쿼리 | **화면 내부 서브 탭** (페이지 안 컨텍스트 전환) | `/targets?type=facility` → TGT01-V 시설 서브 탭 |

> **v1.1 (2026-05-12) 핵심 변경**: 모듈 화면 간 이동은 LNB 사이드 메뉴(`/{module}/{sub}`)로 일원화. `?tab=` 쿼리 패턴은 폐기. 한 화면 안의 컨텍스트 전환만 쿼리 파라미터로 표현.

## 6. 데이터 스코프 정책

> 권한별 어떤 데이터에 접근 가능한지의 범위 규칙.

### 6.1 데이터 스코프 단위

| 단위 | 정의 |
|------|------|
| ALL | 시스템 전체 데이터 (CEO만) |
| DEPT | 본 부서 산하 모든 데이터 (GM) |
| TARGET | 본인이 담당자로 등록된 관리대상의 데이터 (SM/SHM) |
| MEMBER | 본인이 소속된 관리대상의 데이터 (WKR) |
| CONTRACT | 본인이 계약자인 도급계약의 데이터 (SUB) |
| CONSULTING | 본인이 위임 받은 부서·관리대상의 데이터 (CON) |

### 6.2 권한별 데이터 스코프

| 권한 | 데이터 스코프 | 비고 |
|------|------------|------|
| CEO | ALL | 군청 전체 |
| GM | DEPT | 본 부서 산하 |
| SM | TARGET | target_assignees 매칭 |
| SHM | TARGET | target_assignees 매칭 |
| WKR | MEMBER | target_assignees.role='WORKER' 매칭 |
| SUB | CONTRACT | external_user_access.access_type='CONTRACT' |
| CON | CONSULTING | external_user_access.access_type='CONSULTING' |

### 6.3 데이터 스코프 검증 미들웨어

```
모든 데이터 접근 요청 시 검증:
  1. 사용자 권한 코드 확인
  2. 요청 대상 데이터의 소속 확인
  3. 권한 스코프 vs 데이터 소속 매칭 검증
  4. 매칭 시 허용, 미매칭 시 403 Forbidden
```
