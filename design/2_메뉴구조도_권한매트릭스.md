# 담양군 중대재해통합관리시스템 — 메뉴구조도·권한매트릭스

> 문서 ID: IA-001
> 버전: **v1.4**
> 작성일: 2026-05-14
> 작성자: ㈜다온플레이스
> 관련 문서: SRS-001 (요구사항정의서), PRD-DAMYANG-001 (조직·인력 모듈 통합), PRD-DAMYANG-002 (GNB 그룹핑 통합)

---

## 0. 문서 개요

### 0.1 목적

본 문서는 담양군 중대재해통합관리시스템의 정보구조(Information Architecture, IA)를 정의한다. GNB·LNB 메뉴 트리, 권한별 메뉴 접근 매트릭스, 화면 ID 체계, 데이터 스코프 정책을 포함하며, 화면설계서 작성의 기준이 된다.

### 0.2 적용 범위

- GNB 메뉴 (상단 주요 메뉴)
- LNB / 탭 (모듈 내부 보조 메뉴)
- 권한별 메뉴 접근 매트릭스
- 화면 ID 체계 및 URL 패턴

## 1. 권한 코드 체계

### 1.1 권한 코드 정의

| 코드 | 표시명 | 인증 경로 | 데이터 스코프 |
|------|--------|---------|------------|
| CEO | 군수 / 부군수 | 행정포털 SSO | 1단계 (군청 전체) |
| GM | 실/과/단/소장 | 행정포털 SSO | 2단계 (본 부서 전체) |
| SM | 팀장 | 행정포털 SSO | 3단계 (담당 관리대상) |
| SHM | 안전보건담당자 | 행정포털 SSO | 3단계 (담당 관리대상) |
| WKR | 근로자 (공무원) | 행정포털 SSO | 3단계 (본인 소속 관리대상) |
| SUB | 도급업체 | 별도 로그인 (LOCAL) | 계약된 관리대상에 한정 |
| CON | 컨설팅업체 | 별도 로그인 (LOCAL) | 위임된 부서·관리대상에 한정 |

### 1.2 권한 상속 규칙

- CEO ⊃ GM ⊃ SM/SHM ⊃ WKR (내부 직원 간)
- SUB, CON은 별도 인증 경로 — 내부 권한과 상속 관계 없음
- SHM은 SM과 동급 권한 + 안전보건 모듈 추가 권한

## 2. 메뉴 트리 (GNB / LNB)

### 2.1 내부 직원용 GNB (SSO 인증) — v1.4 업무 성격별 8개 그룹

```
홈 (대시보드 — 권한별 자동 라우팅)

GNB 1. 위험성평가 (RSK)              ← 가장 자주 진입
├─ 현황 (RSK01-V)
├─ 평가 목록 (RSK02-L)
├─ 개선조치 (RSK03-L)
└─ 설정 (RSK04-S, 콘텐츠 마스터만)   ← CEO/GM/SHM

GNB 2. 점검·진단                     ← 일상 점검 활동
├─ 안전점검 (INS)
│   ├─ 점검 현황 (INS01-V)
│   ├─ 점검 목록 (INS02-L)
│   └─ 설정 (INS03-S)               ← CEO/GM/SHM
└─ 업무문서 (DOC)
    ├─ 매뉴얼 (DOC01-L)
    └─ 점검표 (DOC02-L)

GNB 3. 도급 관리 (CON)               ← 외부 SUB 협업 + e호조 연동 워크플로우
├─ 도급계약 (CON01-L)
├─ 수급업체 (CON02-L)
├─ 협의체 (CON03-L)
├─ TBM (CON09-L)                    ← 도급공사 계약 존재 시만 노출
└─ 설정 (CON10-S, 콘텐츠 마스터만)   ← CEO/GM/SHM

GNB 4. 종사자 참여 (OPN)
├─ 현황 (OPN01-V)
├─ 의견 목록 (OPN01-L)
├─ 산업안전보건위원회 (OPN03-L)      ← 별도 메뉴 (결정사항 #20)
└─ 설정 (OPN04-S)

GNB 5. 이행·증명                     ← 사후 추적·집계·증명
├─ 개선조치 (IMP)
│   ├─ 현황 (IMP01-V)
│   ├─ 전체 목록 (IMP01-L)
│   └─ 재발방지대책 (IMP01-L `/incident_prevention`)
├─ 이행 관리 (CMP)
│   ├─ 이행 현황 (CMP01-V)
│   ├─ 이행 목록 (CMP02-L)
│   └─ 법령 마스터 (CMP03-S)
├─ 통계 (STA)
│   ├─ 종합 통계 (STA01-V)
│   ├─ 모듈별 통계 (STA02-V)
│   └─ 설정 (STA03-S)
└─ 제증명 (CRT)
    ├─ 발급 이력 (CRT01-V)
    ├─ 증명서 발급 (CRT02-F)
    └─ 설정 (CRT03-S)

GNB 6. 안전보건 경영                 ← 연간 사이클 (빈도 낮음)
├─ 안전보건 방침 (POL)
│   ├─ 방침 관리 (POL01-L)
│   ├─ 목표·KPI (POL03-V)
│   └─ 경영방침 점검 (POL04-V)
├─ 안전보건 계획 (PLN)
│   ├─ 현황 (PLN01-V)
│   ├─ 계획 목록 (PLN01-L)
│   ├─ 캘린더 (PLN03-V)
│   └─ 설정 (PLN04-S)
├─ 안전보건 예산 (BGT)
│   ├─ 예산 현황 (BGT01-V)
│   ├─ 예방 항목 트리 (BGT02-V)
│   └─ 설정 (BGT05-S)
└─ 인력 평가 (EVL)
    ├─ 현황 (EVL01-V)
    ├─ 평가 목록 (EVL01-L)
    └─ 설정 (EVL03-S)

GNB 7. 기반 관리                     ← 초기 셋팅·가끔 업데이트
├─ 관리대상 (TGT)
│   └─ 관리대상 현황 (TGT01-V)
│       └─ (화면 내부 서브 탭) 전체 / 시설 / 업무 / 사업 — `?type=`
├─ 조직 (ORG)                        ← v1.4 명칭 변경 (구 "전담조직")
│   ├─ 조직도 (ORG01-V)
│   └─ 구성원 (ORG02-L)
└─ 안전보건 인력 (STF)               ← v1.4 명칭 변경 (구 "안전관리자 인력") + ORG03 통합
    ├─ 전체 현황 (STF01-V)
    ├─ 부서별 전담인력 (STF04-V)     ← PRD-001: 구 ORG03-V 이전 (URL `/staff/dedicated`)
    ├─ 공사별 선임 현황 (STF02-V)
    └─ 설정 (STF03-S)

GNB 8. 시스템 관리                   ← CEO/GM/SHM 권한
├─ ⭐ 외부 시스템 연동 (SYS01-S)     ← v1.3 신규 — SSO·조직도·FMS·e호조·새올·SMTP 통합
├─ 외부 사용자 관리 (AUTH06-S)       ← CEO/GM/SHM
├─ 알림 운영 (NTF03-S)               ← CEO/GM/SHM (적재 현황·실패 큐만)
└─ 감사 로그                         ← CEO만

(우측 상단)
├─ 알림 (NTF01-L)
└─ 내 정보 (MY01-D)
```

> **v1.4 변경 요약** — 18개 GNB → 8개 업무 성격별 그룹 + 홈.
> 사용 빈도 기준 순서 (위험성평가 → 시스템 관리).
> 화면 ID·URL·DB·SFR 매핑은 변동 없음 (IA 레벨 그룹핑만).
> 선행 PRD-001 결과 통합: ORG03-V→STF04-V, ORG03-M→STF04-M.
> 모듈 명칭 변경: 전담조직→조직, 안전관리자 인력→안전보건 인력.

## 3. 권한별 메뉴 접근 매트릭스

### 3.1 표 범례

- ⭕ : 접근 허용
- ⓪ : 데이터 스코프 제한 접근 (본인 소속만)
- ⏷ : 부분 권한 (열람만)
- ✗ : 접근 불허
- — : 권한 무관

## 4. 화면 ID 체계

### 4.1 명명 규칙

```
{모듈 약어}{번호}-{화면 유형}[-{보조 코드}]

모듈 약어: 3~4자리 영문 대문자 (ORG/TGT/AUTH/NTF/RSK/CON/SYS 등)
번호: 2자리 숫자 (01~99)
화면 유형:
  V — View (현황·대시보드)
  L — List (목록)
  D — Detail (상세)
  F — Form (등록·수정 폼, 풀페이지)
  M — Modal (모달, 보조 모달은 M2, M3 등)
  S — Settings (설정)
  EXEC — 실행 (점검 실시 등)

예시:
  ORG01-V    → 조직 모듈 1번 화면, View 유형
  ORG02-M2   → 조직 모듈 2번 화면의 2번째 모달
  RSK02-D    → 위험성평가 모듈 2번 화면, Detail 유형
  CON09-L    → 도급관리 모듈 9번 화면 (TBM), List 유형
  SYS01-S    → 시스템 관리 모듈 1번 화면, Settings 유형 (신규 v1.3)
```

## 6. 데이터 스코프 정책

> 권한별 어떤 데이터에 접근 가능한지의 범위 규칙.

### 6.1 데이터 스코프 단위

| 단위 | 정의 |
|------|------|
| ALL | 시스템 전체 데이터 (CEO만) |
| DEPT | 본 부서 산하 모든 데이터 (GM) |
| TARGET | 본인이 담당자로 등록된 관리대상의 데이터 (SM/SHM) |
| MEMBER | 본인이 소속된 관리대상의 데이터 (WKR) |
| CONTRACT | 본인이 계약자인 도급계약의 데이터 (SUB) |
| CONSULTING | 본인이 위임 받은 부서·관리대상의 데이터 (CON) |

### 6.2 권한별 데이터 스코프

| 권한 | 데이터 스코프 | 비고 |
|------|------------|------|
| CEO | ALL | 군청 전체 |
| GM | DEPT | 본 부서 산하 |
| SM | TARGET | target_assignees 매칭 |
| SHM | TARGET | target_assignees 매칭 |
| WKR | MEMBER | target_assignees.role='WORKER' 매칭 |
| SUB | CONTRACT | external_user_access.access_type='CONTRACT' |
| CON | CONSULTING | external_user_access.access_type='CONSULTING' |

### 6.3 데이터 스코프 검증 미들웨어

```
모든 데이터 접근 요청 시 검증:
  1. 사용자 권한 코드 확인
  2. 요청 대상 데이터의 소속 확인
  3. 권한 스코프 vs 데이터 소속 매칭 검증
  4. 매칭 시 허용, 미매칭 시 403 Forbidden
```

```
SYS01-S는 시스템 운영 정보 — 데이터 스코프 적용 없음 (전체 조회)
단, 변경 권한 분리:
  - 열람: CEO/GM/SHM
  - 테스트·수동 동기화: CEO/GM
  - 설정 수정·매핑 규칙 편집: CEO만
```
