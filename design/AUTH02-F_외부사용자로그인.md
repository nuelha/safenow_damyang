## 7. 로그인 동작 (자동 처리 로직)

```
1. 클라이언트 검증:
   - 아이디·비밀번호 필수 입력 확인
   - 통과 시 서버 호출

2. 서버 검증:
   a. users 조회: WHERE auth_source='LOCAL' AND local_login_id=:input
   b. 사용자 미존재 → 실패 처리 (§8 케이스 1)
   c. is_active=false → 실패 처리 (§8 케이스 3)
   d. login_failures >= 5 AND last_failure_at >= NOW() - 30분 → 잠금 처리 (§8 케이스 4)
   e. bcrypt 검증:
      - 성공: §7.1 로그인 성공 처리
      - 실패: §7.2 로그인 실패 처리

3. audit_logs INSERT (action='LOGIN_SUCCESS' 또는 'LOGIN_FAIL', ip, user_agent)
```

### 7.1 로그인 성공 처리

```
1. users UPDATE:
   - last_login_at = NOW()
   - login_failures = 0
2. 기존 세션 무효화 (동시 로그인 1세션만 허용)
3. 새 세션 발급 (HttpOnly + Secure + SameSite=Strict 쿠키)
4. CSRF 토큰 발급
5. /external/dashboard 리다이렉트
```

### 7.2 로그인 실패 처리

```
1. users UPDATE:
   - login_failures = login_failures + 1
   - last_failure_at = NOW()
2. 응답: "아이디 또는 비밀번호가 일치하지 않습니다"
   (아이디 미존재 vs 비밀번호 오류 구분 안 함 — 보안)
3. login_failures >= 5 도달 시 잠금 안내 추가 표시
```

## 8. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 1. 아이디·비밀번호 불일치 | "아이디 또는 비밀번호가 일치하지 않습니다" | 인라인 에러 표시, 폼 유지 |
| 2. 필수 필드 누락 | "{필드명}을 입력해주세요" | 인라인 에러 표시 |
| 3. 비활성 계정 | "계정이 비활성 상태입니다. 시스템 관리자에게 문의하세요" | 인라인 에러 표시 |
| 4. 연속 실패 잠금 | "5회 연속 실패로 계정이 30분간 잠금되었습니다. (잠금 해제: {시각})" | 인라인 에러, 로그인 버튼 비활성 |
| 5. 서버 오류 (5xx) | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요" | 인라인 에러 + 재시도 버튼 |

## 9. 보안 정책

- 비밀번호는 평문 저장 안 함 (bcrypt 해시)
- 로그인 폼은 HTTPS only
- CSRF 토큰 (Submit 시점에 검증)
- Brute-force 방어: 5회 연속 실패 시 30분 잠금
- 아이디 enumeration 방어: 실패 메시지 통일
