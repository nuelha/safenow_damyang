# AUTH03-M — 외부 사용자 초대 모달

> 화면 ID: **AUTH03-M**
> 모듈: AUTH (통합 인증)
> SFR: SFR-015 / SFR-013 (도급) / 컨설팅 위임
> 버전: v1.0 | 작성일: 2026-05-11
> 관련 문서: SRS-001 §3 SFR-015, DB-001 §2.2 users / §2.5 external_user_access / §2.6 invitation_tokens, IF-006 SMTP

## 2. 진입 경로

- CON07-F [+ 초대 정보 입력] / [전체 초대 발송] (도급계약 흐름 — SUB 권한)
- AUTH06-S [+ 직접 초대 발급] (컨설팅 위임 등 — CON 권한)
- 컨설팅업체 연결 화면 (Phase 2 — CON 권한)

> 진입 경로에 따라 권한(SUB/CON)과 접근 범위가 자동 결정.

```
┌──────────────────────────────────────────────┐

│ 사업자번호     [______________]              │
│                                              │
│ 담당자 정보                                   │
│ 담당자 이름   [_________________________]  │

│                                              │

```

## 4. 입력 항목

| 필드 | 타입 | 필수 | 검증 | 비고 |
|------|------|------|------|------|
| 회사명 | TEXT | ✅ | 1~200자 | 도급계약 진입 시 수급업체명 prefill |
| 사업자번호 | TEXT | ❌ | 형식 검증 (123-45-67890) | |
| 담당자 이름 | TEXT | ✅ | 1~100자 | |
| 담당자 이메일 | EMAIL | ✅ | 형식 + 시스템 내 UNIQUE | 초대 이메일 발송 대상 |
| 담당자 연락처 | TEL | ❌ | | |
| 권한 | RADIO | ✅ | SUB / CON | 진입 경로에 따라 prefill (변경 가능) |
| 접근 범위 | 자동 | — | — | 진입 경로에 따라 자동 결정 (§5 참조) |

## 5. 접근 범위 자동 결정

| 진입 경로 | 권한 | 접근 범위 |
|---------|------|---------|
| CON07-F [+ 초대 정보 입력] | SUB | `external_user_access` (access_type='CONTRACT', reference_id=contracts.id) |
| AUTH06-S [+ 직접 초대 발급] | 사용자 선택 (SUB/CON) | 사용자가 별도 입력 필요 (도급계약 ID 또는 컨설팅 위임 정보) |
| 컨설팅업체 연결 화면 | CON | `external_user_access` (access_type='CONSULTING', reference_id=consulting_assignments.id) |

## 6. 버튼·링크 액션

| 요소 | 위치 | 액션 | 조건·비고 |
|------|------|------|---------|
| `[×]` 닫기 | 모달 우상단 | 모달 닫기 (변경사항 확인 alert) | |
| 권한 RADIO | 본문 | 진입 경로에 따라 prefill | 변경 가능 (드물게) |
| `[취소]` | 푸터 좌측 | 모달 닫기 | |
| `[초대 발송]` | 푸터 우측 | 계정 발급 + 이메일 발송 (§7) | 필수 필드 + 이메일 UNIQUE 검증 시 활성 |

## 7. 계정 발급 동작

```
[초대 발송] 클릭:
  1. 필수 필드 검증
  2. users INSERT:
     - auth_source='LOCAL'
     - local_login_id = auto_generated (이메일 prefix + 4자리 랜덤, 예: kim_a1b2)
     - local_password_hash = NULL (첫 로그인 시 설정)
     - source_type='LOCAL_INVITED'
     - external_company = 입력 회사명
     - name = 담당자 이름
     - email = 담당자 이메일
     - phone = 담당자 연락처
     - permission_role = SUB or CON
     - is_active = false (첫 로그인 시 활성화)
  3. invitation_tokens INSERT (token=UUID, expires_at=NOW+7일)
  4. external_user_access INSERT (access_type, reference_id, granted_by, expires_at)
  5. email_queue INSERT (template_code='invitation_email',
     변수: 회사명, 발급자(부서/담당자명), 초대 링크 AUTH05-F, 만료일)
  6. audit_logs INSERT (action='EXTERNAL_INVITATION_SENT')
  7. 모달 결과 표시:
     "초대 발송됨 — {담당자 이메일}
      만료: D-7"
  8. 진입 화면(CON07-F 또는 AUTH06-S) 갱신
```

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| 회사명 | 1~200자 |
| 이메일 | 형식 검증 + 시스템 내 UNIQUE (이미 다른 사용자가 사용 중인 이메일 차단) |
| 사업자번호 | 입력 시 형식 검증 (`NNN-NN-NNNNN`) |
| 권한 | SUB / CON 중 1개 |
| 접근 범위 | 진입 경로별 자동 결정 (직접 발급 시 사용자 입력) |

## 9. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 이메일 중복 (시스템 내) | "이미 등록된 이메일입니다. 다른 이메일을 사용하세요" | 인라인 에러 |
| 이메일 형식 오류 | "이메일 형식이 올바르지 않습니다" | 인라인 에러 |
| 사업자번호 형식 오류 | "사업자번호 형식이 올바르지 않습니다 (예: 123-45-67890)" | 인라인 에러 |
| 이메일 발송 실패 | "이메일 발송에 실패했습니다. 잠시 후 재시도됩니다" | 토스트 + IF-006 재시도 큐 |
| users INSERT 실패 (트랜잭션 오류) | "초대 발송 중 오류가 발생했습니다" | 토스트 + 모달 유지 |

## 10. 관련 화면 흐름

```
CON07-F [+ 초대 정보 입력] / AUTH06-S [+ 직접 초대 발급]
   → AUTH03-M
   ↓ [초대 발송]
   → users INSERT (LOCAL_INVITED) + invitation_tokens + external_user_access + email_queue
   → 모달 결과 표시 → 진입 화면 갱신
   
도급업체 측:
   이메일 [계정 활성화 링크] → AUTH05-F (첫 로그인 비밀번호 설정)
```

## 11. 관련 DB 테이블

- `users` (DB-001 §2.2): 외부 사용자 INSERT (auth_source='LOCAL')
- `invitation_tokens` (DB-001 §2.6): 초대 토큰
- `external_user_access` (DB-001 §2.5): 접근 범위 매핑
- `email_queue` (DB-001 §2.11): 이메일 발송 큐
- `audit_logs` (DB-001 §2.12): 초대 발급 감사

## 12. 관련 인터페이스

- **IF-006 SMTP**: 초대 이메일 발송 (`invitation_email` 템플릿)
