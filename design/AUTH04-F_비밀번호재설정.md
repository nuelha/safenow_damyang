## 7. 자동 처리 로직

### 7.1 STEP 1 — 이메일 입력 동작

```
[재설정 링크 발송] 클릭:
  1. users 조회 (WHERE auth_source='LOCAL' AND email=:input)
  2. 사용자 존재 여부와 무관하게 동일 메시지 응답 — 보안 (enumeration 방어):
     "이메일이 발송되었습니다. 확인해주세요."
  3. 사용자 존재 시:
     - 기존 password_reset_tokens 무효화 (used_at=NOW())
     - password_reset_tokens INSERT (token=UUID, expires_at=NOW+1h)
     - email_queue INSERT (template_code='password_reset',
       변수: 사용자명, 재설정 링크, 만료 시간)
  4. audit_logs INSERT (action='PASSWORD_RESET_REQUEST')
  5. STEP 1 화면에 안내 메시지 표시 후 화면 전환 안 함 (보안)
```

### 7.2 STEP 2 — 비밀번호 변경 동작

```
링크 클릭 → /password-reset/:token 진입:
  1. password_reset_tokens 검증:
     - token 존재 + expires_at >= NOW() + used_at IS NULL
     - 무효 → AUTH99-V `sso_token_invalid` 또는 별도 메시지 표시
  
[비밀번호 변경] 클릭:
  1. 비밀번호 정책 검증 (§7.3)
  2. 비밀번호 확인 일치 검증
  3. 비밀번호 이력 검증 (password_history 최근 3건과 비교)
  4. 통과 시:
     - users.local_password_hash 갱신 (bcrypt)
     - password_history INSERT (이전 해시 저장)
     - password_reset_tokens.used_at = NOW()
     - 기존 세션 모두 무효화
     - login_failures = 0 (잠금 해제)
  5. audit_logs INSERT (action='PASSWORD_CHANGE')
  6. AUTH02-F (/login) 리다이렉트 + "비밀번호 변경 완료" 메시지
```

### 7.3 비밀번호 정책

| 항목 | 정책 |
|------|------|
| 길이 | 8자 이상 |
| 조합 | 영문·숫자·특수문자 3종 조합 |
| 이력 | 최근 3회 사용 비밀번호 재사용 금지 |
| 만료 안내 | 90일 경과 후 변경 권장 (강제 아님) |

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| 이메일 형식 | RFC 5322 |
| 비밀번호 정책 | §7.3 모두 충족 |
| 비밀번호 확인 일치 | new == confirm |
| 비밀번호 이력 | password_history 최근 3건과 다름 |
| 토큰 유효 | expires_at >= NOW() AND used_at IS NULL |
| LOCAL 사용자 한정 | SSO 사용자는 본 화면 사용 불가 (행정포털에서 처리) |

| 이메일 발송 실패 | (사용자에게는 동일 메시지 노출 — 보안) + 운영자 로그 | — |

## 10. 보안 정책

- 이메일 enumeration 방어: 사용자 존재 여부 무관 동일 응답
- 토큰 만료: 1시간
- 토큰 단일 사용 (used_at 갱신)
- 비밀번호 변경 시 기존 세션 전체 무효화
- 변경 직후 자동 로그인 안 함 (사용자가 새 비밀번호로 로그인하도록 유도 — 안전)
- audit_logs 기록 (요청·완료·실패)

## 11. 관련 화면 흐름
