## 2. 진입 경로

인증·권한 처리 중 다양한 오류 발생 시 시스템이 자동 리다이렉트:

- AUTH01-V (SSO 처리) 토큰 무효·API 오류·비활성 계정 등
- AUTH04-F·AUTH05-F 토큰 만료·사용됨
- 데이터 스코프 검증 실패 (`permission_denied`)
- 세션 만료 (`session_expired`)
- 일반 권한 부족

## 7. 자동 처리 로직

### 7.1 진입 시 처리

```
/auth/error?code=:code&from=:url 진입:
  1. URL 파라미터 검증:
     - code: 화이트리스트(§4)에 포함된 코드만 허용. 미포함 → 'unknown'
     - from: URL 형식 검증 + 외부 도메인 차단 (open redirect 방어)
  2. audit_logs INSERT (action='AUTH_ERROR', metadata={code, from})
  3. 사용자 권한·세션 상태 확인 후 액션 버튼 분기 처리
```

### 7.2 세션 만료 처리

```
세션 만료(`session_expired`)는 시스템이 인터셉터에서 감지:
  1. 현재 진입 URL을 ?from= 으로 저장
  2. AUTH99-V 리다이렉트
  3. 사용자가 [로그인 이동] 클릭 → 로그인 후 from URL로 자동 이동
```

### 7.3 외부 도메인 리다이렉트 방어

```
?from= 파라미터는 외부 URL 차단:
  - 절대 URL (https://other.com/...) → 무시, 기본값으로 대체
  - 상대 URL (/dashboard) → 허용
  - 경로 traversal (../) → 차단
```

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| 오류 코드 | 화이트리스트만 허용 |

| 자동 리다이렉트 | 사용자 동의 없는 외부 이동 차단 |

## 9. 오류 처리

본 화면 자체가 오류 처리 화면이므로 추가 오류 케이스 없음. 단:

| 케이스 | 처리 |
|--------|------|
| 잘못된 code 파라미터 | 'unknown' 코드로 처리 |
| from 파라미터 외부 URL | from 무시, 기본 액션 |

## 10. 관련 화면 흐름

```
모든 인증·권한 오류 케이스

   → 대상 화면 (SSO 로그인 / 외부 로그인 / 대시보드 / 행정포털)
```

## 13. 디자인 가이드

- 사용자 비난 안 함: "권한이 없습니다"보다 "접근 권한이 없습니다. 필요한 경우 관리자에게 문의하세요" 같은 친절한 표현
- 액션 버튼 1-2개로 제한: 사용자가 다음 단계를 명확히 선택할 수 있도록
- 운영자 연락처 항상 표시: footer 영역에 시스템 관리자 이메일·전화 (정적)
- 기술적 오류 코드 노출 안 함: 사용자에게는 인간 친화적 메시지만, audit_logs에는 상세 코드 기록
