> 화면 ID: **BGT02-V**
> 모듈: BGT (예산)
> SFR: SFR-008
> 버전: v1.0 | 작성일: 2026-05-11
> 관련 문서: SRS-001 §3 SFR-008, 결정사항 #22 (3 depth 부서별 별도 트리)

> ※ 핵심 의사결정: 예방 항목은 최대 3 depth + 부서별 별도 트리 운영. 공통 항목과 부서 특화 항목을 함께 관리.

## 4. 표시 항목

### 4.1 트리 구조 (결정사항 #22)

| Depth | 의미 | 예시 (공통) | 예시 (부서 특화) |
|-------|------|----------|--------------|
| 1단계 | 대분류 카테고리 | 안전보건교육 | 농기계 안전관리 (농업기술센터) |
| 2단계 | 세부 카테고리 | 정기교육 / 신규 채용 교육 | 농기계 정기 점검 |
| 3단계 | 실행 단위 | 사무직 정기교육 / 현장직 정기교육 | 트랙터 1호 점검 |

> **공통 항목 (department_id=NULL)**: 모든 부서가 사용 가능. 시스템 표준.
> **부서 특화 항목 (department_id=값)**: 해당 부서만 사용. 부서별 독립 트리.

### 4.2 항목별 표시

각 노드:
- 카테고리명 + 펼침/접힘 아이콘
- 편성액 / 집행액 / 집행률 (해당 연도 + 본인 권한 범위)
- 우측 [상세] / [수정] / [편성 보기] / [집행 보기] 액션 버튼 (호버 시 표시)

### 4.3 권한별 표시

| 권한 | 보이는 범위 |
|------|----------|
| CEO | 공통 + 모든 부서 특화 |
| GM | 공통 + 본 부서 특화 |
| SHM | 공통 + 본 부서 특화 + 권한 위임 다부서 |
| SM | 공통 + 본 부서 (열람만) |

## 7. 자동 처리 로직

### 7.1 부서장 변경 시 부서 특화 트리 영향

```
부서장 변경 자체는 트리에 영향 없음 (트리는 부서 단위 자체 관리)
단, 부서장이 트리 관리 권한 보유 (BGT05-S에서 부서 특화 항목 관리 가능)
```

### 7.2 신년 트리 복사 (BGT05-S 설정에 따라)

```
새 회계연도 시작 시 BGT05-S [신년 트리 자동 복사] 옵션 ON이면:
  - 이전 연도 budget_categories의 활성 항목 복사
  - 단, budget_items / budget_executions는 복사 안 함 (편성·집행은 신년 새로 입력)
  
사용자가 트리 자체는 수정 가능 (depth 추가·항목 삭제·이름 변경)
```

### 7.3 집행률 자동 계산 캐싱

```
각 노드 집행률은 비싼 집계.
- 1시간 캐싱
- budget_executions INSERT/UPDATE 시 해당 트리 invalidate

상위 노드 집행률 = 하위 노드 합산 (재귀)
```

## 8. BGT05-M 항목 추가 모달 (보조)

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 상위 항목 | TREE_SELECT | ❌ | NULL이면 1단계. 선택 시 2·3단계 |
| 부서 구분 | RADIO | ✅ | 공통(NULL) / 본 부서 특화 |
| 카테고리명 | TEXT | ✅ | 1~200자 |
| 코드 | TEXT | ❌ | 자동 생성 가능 |
| 정렬 순서 | NUMBER | ❌ | 기본 자동 |

검증:
- 3 depth 초과 차단 ("최대 3 depth까지만 가능합니다")
- 상위 항목이 부서 특화이면 하위도 같은 부서 자동 적용 (depth 일관성)
- 상위 항목이 공통(NULL)이면 하위는 공통/부서 둘 다 가능

저장:
```
budget_categories INSERT
audit_logs INSERT
트리 새로고침
```

## 13. 관련 DB 테이블

BGT01-V §10 참조. 본 화면 핵심: `budget_categories` 재귀 조회 (parent_id 기반 트리).

```sql
-- 재귀 조회 (PostgreSQL CTE 예시)
WITH RECURSIVE category_tree AS (
  SELECT id, code, name, parent_id, depth, department_id, sort_order
  FROM budget_categories
  WHERE parent_id IS NULL
    AND is_active=true
    AND (department_id IS NULL OR department_id = :current_dept)
  
  UNION ALL
  
  SELECT c.id, c.code, c.name, c.parent_id, c.depth, c.department_id, c.sort_order
  FROM budget_categories c
  INNER JOIN category_tree t ON c.parent_id = t.id
  WHERE c.is_active = true
)
SELECT * FROM category_tree ORDER BY depth, sort_order;
```

## 15. 성능 고려

- 트리 데이터 캐싱 (30분, 변경 시 invalidate)
- 집행률 집계는 별도 view 또는 캐싱
