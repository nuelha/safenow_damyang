## 3. 레이아웃

### 3.1 화면 내부 서브 탭 구성

> 본 상세 화면은 한 도급계약의 정보를 8개 영역으로 분할 표시 — 페이지 내부 서브 탭으로 전환 (모듈 화면 간 이동과 무관 / 같은 화면 안의 컨텍스트 전환).

```
[개요] [수급업체] [평가] [위험성평가] [협의체] [TBM] [서약서] [개선조치]
```

> TBM 영역은 `contract_type='CONSTRUCTION'` 일 때만 노출.

### 3.2 개요 영역


## 4. 표시 항목 (개요 영역)

### 4.1 상단 헤더

| 항목 | 데이터 소스 | 비고 |
|------|------------|------|
| 사업명 | `contracts.name` | |
| 계약유형 배지 | `contract_type` | 5분류: 공사/용역/구매설치/위탁/기타 |
| 계약번호 | `ehojo_contract_no` 또는 시스템 내부 ID | e호조 자동 import 시 |

### 4.2 스텝 인디케이터 (4단계)

| 단계 | 매핑 | 상태 표시 |
|------|------|---------|
| 1. 기본정보 | step=1 | DRAFT 완료 시 ● |
| 2. 평가 | step=2 | EVALUATING 완료 시 ● |
| 3. 결재 | step=3 | APPROVED 완료 시 ● |
| 4. 초대 | step=4 | 모든 선정 업체 서명 완료 시 ● → IN_PROGRESS 전환 |

### 4.3 기본정보

| 항목 | 데이터 소스 |
|------|------------|
| 사업명 | `contracts.name` |
| 계약유형 | `contract_type` |
| 출처 | `source` 배지 + `ehojo_contract_no` (e호조 자동 import 시) |
| 발주처 | `funder_department` |
| 관리대상 | JOIN `targets.name + type 배지` |
| 기간 | `start_date ~ end_date` |
| 예산 | `budget` (원 단위, 콤마 구분) |
| 담당자 | JOIN `users.name + position` (created_by 또는 default_evaluator_id) |

### 4.4 안전관리 인력 충족 여부 (D-3 연동, SFR-010)

```
공사 규모(예산)별 법정 안전관리자 N명 필요:
  - 공사금액 50억 이상 → 안전관리자 1명 이상
  - 공사금액 120억 이상 → 안전관리자 2명 이상
  (시행령 표 기준)

target_assignees + dedicated_personnel 집계 → 충족 여부 산출

미달 시:
  - 카드에 [미충족] ⚠ + 부족 인원 수 명시
  - 결재 진행 버튼 비활성
  - "안전관리자 N명 추가 선임 후 결재 진행 가능" 툴팁
```

### 4.5 결재 상태

| approval_status | 표시 | 추가 노출 |
|----------------|------|----------|
| `NOT_REQUESTED` | 미요청 | `[온나라 결재 진행하기]` 버튼 |
| `REQUESTED` | 결재 진행 중 | `[결재 결과 업데이트]` 버튼 |
| `APPROVED` | 결재 완료 (날짜·결재자) | 시공 진행 카드 노출 |
| `REJECTED` | 결재 반려 | 사유 표시 + STEP 2 재진입 안내 |
| `BLOCKED` 상태일 때 | 결재 차단 | 안전관리자 미달 안내 |

### 4.6 시공 진행 카드 (IN_PROGRESS 이후)

| 항목 | 데이터 소스 | 비고 |
|------|------------|------|
| 진행률 | `(NOW - start_date) / (end_date - start_date) × 100` | 시간 기준 단순 계산 |
| TBM 수 | `COUNT(tbm_records WHERE contract_id)` | 공사일 때만 |
| 협의체 회의 수 | `COUNT(council_meetings WHERE contract_id)` | |
| 개선조치 (도급 관련) | `COUNT(improvements WHERE source_id=contract_id)` | |

## 5. 영역별 내용

### 5.1 수급업체 영역

해당 계약의 선정 업체 목록 + 평가 점수 + 서약 상태:

| 컬럼 | 데이터 소스 |
|------|------------|
| 업체명 | JOIN `subcontractors.company_name` |
| 사업자번호 | `business_number` |
| 평가 점수·결과 | `evaluation_score`, `evaluation_result` |
| 평가자 | JOIN `users.name` (evaluator_id) |
| 선정 여부 | `selected` 배지 |
| 초대 상태 | `invited_at`, `signed_at` 기반 산출 (PENDING/ACCEPTED/EXPIRED/REVOKED) |
| 서약 완료일 | `signed_at` |

### 5.2 평가 영역

CON05-F STEP 2의 평가서·평가 결과 표시 (CON01-D 내 read-only 뷰)

### 5.3 위험성평가 영역

`source_contract_id` 기반 자동 생성된 위험성평가 목록 표시. [위험성평가 상세] → RSK02-D 이동.

### 5.4 협의체 영역

해당 계약의 협의체 회의 목록 (CON03-L의 본 계약 필터링 뷰)

### 5.5 TBM 영역 (도급공사만)

해당 계약의 TBM 기록 목록 (CON09-L의 본 계약 필터링 뷰). [+ TBM 작성] 진입 가능.

### 5.6 서약서 영역

선정 업체별 서약서 작성 상태 + 서약서 본문 + 첨부 파일 표시

### 5.7 개선조치 영역

해당 계약에서 발생한 개선조치 목록 (도급공사 위험성평가 결과의 개선조치 등)

## 7. 자동 처리 로직

### 7.1 안전관리자 미달 자동 차단 (D-3)

```
계약 등록·수정 또는 SFR-010 인력 변경 감지 시:
  - 공사 규모(budget)별 법정 안전관리자 N명 필요
  - target_assignees + dedicated_personnel SAFETY_MGR 수 집계
  - 미달 시: contracts.status='BLOCKED' 자동 전환
  - 본 화면 노출 시 [결재 진행] 버튼 비활성 + 안내 메시지

운영자가 STF04-V에서 안전관리자 추가 선임 시:
  - 충족 검증 자동 재실행
  - 충족 시 contracts.status='EVALUATING' 자동 복귀 (또는 DRAFT)
```

### 7.2 결재 흐름

CON06-F STEP 3에서 진행. 본 화면은 결재 상태만 표시 + 결재 결과 수동 업데이트 진입점.

### 7.3 시공 종료 자동 처리

```
contracts.end_date < NOW():
  status='COMPLETED' 자동 전환 (배치 작업, 매일 00:00)
  새올 포틀릿 알림 (담당자: SHM+, category='contract',
    "{사업명} 계약 종료 — 종료평가 시기 도래")
```

## 8. 데이터 스코프

| 권한 | 보이는 범위 |
|------|----------|
| CEO | 전체 |
| GM | 본 부서 산하 관리대상 관련 계약 |
| SM/SHM | 본인 담당 + 본 부서 |
| WKR | 본 화면 접근 불가 |
| SUB | `external_user_access.access_type='CONTRACT'` 본인 계약만. 일부 영역만 접근 (개요·서약서·TBM) |
| CON | 위임 범위 계약 |

## 9. 검증 규칙

| 항목 | 검증 |
|------|------|
| 수정 권한 | `status='DRAFT'`일 때만 [수정] 활성 |
| 결재 진행 | `status='EVALUATING'` + 평가 완료 + 안전관리자 충족 시 활성 |
| TBM 작성 | `contract_type='CONSTRUCTION'` + `status IN ('APPROVED','IN_PROGRESS')` 시 활성 |
| 서약서 작성 | SUB 사용자 + 본인 계약 + 서약 미완료 시 활성 |

## 10. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 계약 미존재 | "계약을 찾을 수 없습니다" | 404 + CON01-L 이동 |
| 데이터 스코프 권한 없음 | "접근 권한이 없습니다" | AUTH99-V |
| 안전관리자 미달 상태 | "안전관리자 N명이 미달입니다. 결재를 진행할 수 없습니다" | 인라인 안내 + [전담인력 화면으로] 링크 |
| 결재 반려 상태 | "결재 반려: 사유 {message}. STEP 2 재진행 필요" | 인라인 안내 + [STEP 2로 이동] |

## 12. 관련 DB 테이블
