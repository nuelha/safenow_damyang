## 4. 표시 항목

### 4.1 목록 컬럼

| 컬럼 | 데이터 소스 | 비고 |
|------|------------|------|
| 업체명 | `subcontractors.company_name` | 클릭 시 CON02-D 상세 |
| 사업자번호 | `business_number` | |
| 대표자 | `ceo_name` | |
| 업종 | `industry` 배지 | 건설/설비/전기/운송/청소·시설관리/기타 |
| 최근 평가 점수 | `MAX(contract_subcontractors.evaluation_score)` (서브쿼리) | 평가 이력 없으면 "—" |
| 결과 | `evaluation_result` 배지 (최근) | 적격(QUALIFIED) / 부적격(UNQUALIFIED) |
| 참여 계약 수 | `COUNT(contract_subcontractors WHERE subcontractor_id=:s.id)` | 누적 |

### 4.2 적격 여부 배지

| evaluation_result | 표시명 | 색상 |
|------------------|--------|------|
| `QUALIFIED` | 적격 | 초록 |
| `UNQUALIFIED` | 부적격 | 빨강 |
| (NULL) | 미평가 | 회색 |

## 8. CON02-D 수급업체 상세 (보조 — 본 문서에 통합 정의)

| 항목 | 내용 |
|------|------|
| URL | `/subcontractors/:id` |
| 화면 유형 | D (Detail) |
| 접근 권한 | SHM+ |

### 8.1 표시 항목

- 회사 기본정보 (회사명·사업자번호·대표자·업종·주소·연락처)
- 참여 계약 이력 목록 (contract_subcontractors JOIN contracts)
- 평가 점수 추이 차트 (시기별 evaluation_score)
- 안전 사고 이력 (Phase 2/3 — 안전사고 모듈 연계)

### 8.2 버튼·링크 액션

| 요소 | 액션 |
|------|------|
| `[수정]` | CON02-M 모달 (수정 모드) — SHM+ |
| `[비활성화]` | subcontractors.is_active=false. CEO/GM |
| 계약 행 클릭 | CON01-D 이동 |

## 9. 데이터 스코프

| 권한 | 보이는 범위 |
|------|----------|
| CEO | 전체 |
| GM | 본 부서 계약 참여 업체 |
| SM/SHM | 본인 담당 계약 참여 업체 |
| 외부 | 본 화면 접근 불가 |

## 10. 검증 규칙

| 항목 | 검증 |
|------|------|
| 사업자번호 | 형식 검증 (NNN-NN-NNNNN). UNIQUE 권장 (중복 시 경고) |
| 비활성화 | 진행 중 계약(IN_PROGRESS)에 참여 중이면 차단 |

## 11. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 사업자번호 중복 | "이미 등록된 사업자번호입니다. 기존 업체를 확인하세요" | 인라인 + 기존 행 링크 |
| 비활성화 시 진행 중 계약 | "진행 중인 계약이 있어 비활성화할 수 없습니다" | 차단 |

## 12. 관련 화면 흐름

```
CON02-L

```

## 13. 관련 DB 테이블

- `subcontractors` (DB-001 §5.2): 본 화면의 주 데이터
- `contract_subcontractors` (DB-001 §5.3): 참여 계약·평가 이력
- `contracts` (JOIN): 계약명·기간
