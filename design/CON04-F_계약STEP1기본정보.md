## 2. 진입 경로

- CON01-L `[+ 계약 등록]` (신규 수동)
- CON01-D `[수정]` (수정 모드, `status='DRAFT'`일 때만)
- CON01-D 스텝 인디케이터 [STEP 1] 클릭 (read-only 진입)
- 차세대 e호조 자동 import 직후 운영자가 담당 부서·관리대상 확정을 위해 진입
- 임시저장 후 재진입 (`status='DRAFT'`)

## 5. 표시 항목

### 5.1 출처 영역

| 출처 | 표시 |
|------|------|
| 수동 등록 | "수동 등록" 배지 |
| 차세대 e호조 자동 import | "e호조 자동 #{ehojo_contract_no}" 배지 + 마지막 동기화 시각 |

### 5.2 관리대상 인라인 정보

관리대상 선택 시 다음 정보 자동 표시 (TGT02-F §5.1과 동일):
- 담당부서, 유형 배지, 중대재해법 분류·종별 (FACILITY만)

### 5.3 계약유형 변경 안내

```
계약유형 RADIO 변경 시 인라인 안내:
  "계약유형 변경 시 STEP 2 평가서와 STEP 4 서약서가 자동으로 변경됩니다."

이미 STEP 2 진입 후 변경 시 추가 경고:
  "기존 STEP 2의 평가 결과가 초기화될 수 있습니다. 계속하시겠습니까?"
```

## 6. 버튼·링크 액션

| 요소 | 위치 | 액션 | 조건·비고 |
|------|------|------|---------|
| 스텝 인디케이터 | 상단 | 현재 STEP 강조. 완료 STEP 클릭 시 read-only 진입 | |
| `[← 도급관리]` | 헤더 좌측 | CON01-L 복귀 (변경사항 확인 alert) | |
| 출처 RADIO (수동/자동) | 본문 | e호조 자동은 readonly (시스템 결정) | 수동 등록은 사용자 선택 가능 |
| 관리대상 selector | 본문 | 권한 범위 내 targets 목록 | 부서·type 배지 인라인 표시 |
| 계약유형 RADIO | 본문 | 5분류 중 1개 선택 + 안내 모달 | 변경 시 §5.3 안내 |
| `[이전]` | 푸터 좌측 | 숨김 (STEP 1) | |
| `[취소]` | 푸터 | CON01-L 복귀 (변경사항 alert) | |
| `[임시저장]` | 푸터 우측 | contracts UPSERT (step=1, status='DRAFT') → 토스트 | 어느 STEP에서나 가능 |
| `[다음 →]` | 푸터 우측 (Primary) | 검증 → contracts UPSERT (step=1, status='EVALUATING') → STEP 2 진입 | 필수 필드 검증 + 종료일 ≥ 시작일 |

## 7. 저장 동작

### 7.1 신규 (수동 등록)

```
[다음 →] 클릭:
  1. 필수 필드 검증
  2. contracts INSERT:
     - source='MANUAL'
     - status='EVALUATING' (STEP 2 진입 의미)
     - step=1 (STEP 1 완료)
     - 입력 필드 모두 반영
  3. audit_logs INSERT
  4. CON05-F (STEP 2) 진입
```

### 7.2 e호조 자동 import 항목 수정

```
e호조 자동 import 직후 contracts.status='DRAFT' 상태:
  - 운영자가 본 화면 진입 → target_id 확정·담당자 지정 등
  - [다음 →] 클릭 시 status='EVALUATING' 전환

e호조 import 데이터는 일부 필드 readonly (사업명·발주처·예산 등 — 원본 보호):
  - 단, 관리대상·계약유형·기본 평가자는 사용자 입력 필수
  - 차세대 e호조 contract_type 코드 자동 매핑 결과는 변경 가능
```

### 7.3 수정

```
status='DRAFT' 일 때만 수정 가능:
  contracts UPDATE
  - 계약유형 변경 시 STEP 2·STEP 4 영향 안내 → 진행 중 평가 있으면 초기화 경고
```

### 7.4 임시저장

```
[임시저장]:
  contracts UPSERT (step 유지, status='DRAFT')
  → 토스트 "임시저장되었습니다"
  → 본 화면 유지 (다른 화면으로 이동 안 함)
```

## 8. 자동 처리 로직

### 8.1 e호조 자동 import 데이터 prefill

```
URL `?ehojo_contract_no=:no` 또는 contracts.source='EHOJO_AUTO' 진입 시:
  - 사업명, 발주처, 시작일, 종료일, 예산: 자동 채움 (readonly 또는 lock)
  - 계약유형: e호조 코드 매핑 결과 prefill (변경 가능)
  - 관리대상: 자동 매칭 결과 prefill (변경 가능 — IF-004 §4.5)
    매칭 실패 시 빈 selector + 안내 "관리대상을 직접 선택해주세요"
```

### 8.2 계약유형 변경 시 영향 처리

```
RADIO 변경 감지:
  1. 다른 유형으로 변경 → 안내 (§5.3)
  2. STEP 2 진입 후 STEP 1로 회귀하여 변경 시:
     "STEP 2 평가 결과가 초기화됩니다. 계속하시겠습니까?"
     - 취소: 변경 취소
     - 확인: contract_subcontractors.evaluation_status='PENDING' 일괄 변경
           + evaluation_form_id 자동 갱신 (계약유형 매칭 폼)
```

### 8.3 관리대상 변경 시 RESPONSIBLE 영향 없음

```
contracts.target_id 변경은 contracts 단위 정보만 변경
target_assignees는 별도 — 관리대상 본체 부서 이동 시점에만 자동 매핑
```

## 9. 검증 규칙

| 항목 | 검증 |
|------|------|
| 관리대상 | 권한 범위 내만 가능. 권한 외 자동 필터링 |
| 계약유형 | 5분류 중 1개 필수 |
| 사업명 | 1~200자 |
| 계약 기간 | 종료일 ≥ 시작일 |
| 예산 | 0 이상 정수 |
| 기본 평가자 | 비활성 사용자 차단 |
| e호조 자동 import 시 | ehojo_contract_no UNIQUE 검증 |

## 10. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 필수 필드 누락 | "{필드명}을(를) 입력해주세요" | 인라인 에러 + STEP 이동 차단 |
| 종료일 < 시작일 | "종료일은 시작일 이후여야 합니다" | 인라인 에러 |
| 권한 범위 외 관리대상 | (자동 필터링) | — |
| e호조 자동 import 시 관리대상 미매칭 | "관리대상을 직접 선택해주세요" | 안내 메시지 + selector 비움 |
| 계약유형 변경 시 STEP 2 영향 확인 | "STEP 2 평가 결과가 초기화됩니다" | 확인 모달 |
| 저장 실패 | "저장 중 오류가 발생했습니다" | 토스트 + 폼 유지 |
