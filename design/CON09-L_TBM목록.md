# CON09-L — TBM 목록 (도급공사)

> 화면 ID: **CON09-L**
> 모듈: CON (도급관리)
> SFR: SFR-013
> 버전: v1.0 | 작성일: 2026-05-11
> 관련 문서: SRS-001 §3 SFR-013, DB-001 §5.7 tbm_records, 결정사항 #9 (TBM 도급공사에만)

> ⚠ TBM 모듈은 **도급공사(`contract_type='CONSTRUCTION'`)에만 적용** (결정사항 #9). 일반 관리대상(시설/업무/사업)에는 TBM 미적용.

## 2. 진입 경로

- GNB **의무이행 관리** > LNB **TBM** (도급공사 1건 이상 시 노출)
- CON01-D TBM 영역 (`contract_type='CONSTRUCTION'`일 때)
- 도급업체(SUB) 대시보드 [TBM 작성] 카드

> 모듈 내 다른 화면(도급계약 / 수급업체 / 협의체 / 도급관리 설정)으로의 이동은 좌측 사이드 메뉴(LNB)로 처리.

## 7. CON09-F TBM 작성 화면 (보조)

| 항목 | 내용 |
|------|------|
| URL | `/tbm/new?contract_id=:id` 또는 `/tbm/:id/edit` |
| 화면 유형 | F (Form) |
| 접근 권한 | SHM+ / SUB (본인 계약) |

### 7.1 입력 항목

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 계약 | SELECT (contracts) | ✅ | `contract_type='CONSTRUCTION'`만 |
| 수급업체 | SELECT | ❌ | 계약 참여 업체 중 선택 (또는 전체) |
| 공정/작업명 | TEXT | ✅ | 자유 텍스트 (PRC 폐기) |
| 작업일자 | DATE | ✅ | 기본값: 오늘 |
| 참석자 | 복합 입력 | ✅ | 내부 직원 USER_SEARCH + 외부 SUB 자유 입력 |
| 위험요인 | 다중 입력 | ✅ | RSK 자동 불러오기 + 현장 추가 |
| 안전조치 | 다중 입력 | ✅ | 각 위험요인에 대한 조치 |
| 첨부 (사진·서명) | FILE | ❌ | 현장 사진·서명 기록 |

### 7.2 위험요인 자동 불러오기

```
계약 선택 시:
  GET /api/risk-factors?contract_id=:contract_id
  → 해당 도급공사의 위험성평가(`source='EHOJO_AUTO' or 'MANUAL'`)에서 위험요인 목록 가져옴
  → 체크박스로 표시 (선택적 포함)
  → 현장 추가 위험요인은 [+ 위험요인 추가] 입력 필드로 추가
```

### 7.3 저장 동작

```
1. tbm_records INSERT (
     contract_id, subcontractor_id, work_name, work_date,
     participants JSON, hazards JSON, safety_actions JSON,
     created_by, created_at)
2. audit_logs INSERT
3. 새올 포틀릿 알림 (대상자: 계약 담당자 + SHM+, category='contract')
4. CON09-L 복귀
```

## 8. CON09-D TBM 상세 (보조)

| 항목 | 내용 |
|------|------|
| URL | `/tbm/:id` |
| 화면 유형 | D (Detail) |

표시 항목: TBM 본체 + 참석자·위험요인·안전조치·첨부 일체. 수정·삭제 버튼 (작성자 또는 SHM+).

## 9. 자동 처리 로직

### 9.1 TBM 작성 알림

```
SUB(도급업체)가 TBM 작성 시:
  - 계약 담당자(SHM+)에게 새올 포틀릿 알림
  - 시공 진행 카드(CON01-D)에 TBM 수 증가

SHM+가 TBM 작성 시:
  - SUB 담당자에게 이메일 알림 (옵션)
```

### 9.2 TBM 미실시 모니터링 (Phase 2 확장)

```
공사 진행 중 N일 동안 TBM 기록 0건이면:
  운영자 알림 ("최근 N일간 TBM 기록 없음 — 현장 점검 필요")
```

## 10. 데이터 스코프

| 권한 | 보이는 범위 |
|------|----------|
| CEO | 전체 |
| GM | 본 부서 계약의 TBM |
| SM/SHM | 본인 담당 계약의 TBM |
| SUB | 본인 계약(`external_user_access`)의 TBM만 |
| CON | 위임 범위 계약의 TBM |

## 11. 검증 규칙

| 항목 | 검증 |
|------|------|
| 계약 선택 | `contract_type='CONSTRUCTION'`만 |

| 위험요인 | 1건 이상 필수 |
| 안전조치 | 위험요인 1건당 1건 이상 필수 |

## 12. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 도급공사 외 계약 선택 시도 | "TBM은 도급공사(CONSTRUCTION)에만 작성 가능합니다" | 옵션에서 제외 |
| 종료 계약에 TBM 작성 시도 | "종료된 계약입니다" | 인라인 안내 (가능 — 사후 기록 케이스) |
| SUB가 다른 계약의 TBM 접근 | "접근 권한이 없습니다" | AUTH99-V |

## 14. 관련 DB 테이블

- `tbm_records` (DB-001 §5.7): 본 화면 주 데이터
- `contracts` (DB-001 §5.1): 도급공사 계약

- `risk_factors` (DB-001 §4.2): 위험요인 자동 불러오기
- `audit_logs` (DB-001 §2.12): TBM 작성 감사
