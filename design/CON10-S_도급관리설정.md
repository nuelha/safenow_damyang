## 4. 표시 항목

### 4.1 평가서 마스터

## 6. 자동 처리 로직

### 6.1 평가서 변경 시 영향

```
evaluation_forms 수정 시:
  - 이미 사용 중인 계약(contract_subcontractors.evaluation_form_id 참조)에는 영향 없음 (역사 보존)
  - 새로 진입하는 STEP 2에서만 변경된 평가서 적용
```

### 6.2 서약서 변경 시 영향

```
agreement_forms 수정 시:
  - 이미 서약 완료된 contract_agreements는 변경 안 함 (역사 보존)
  - 새로 진입하는 STEP 4에서만 변경된 서약서 적용
```

### 6.3 차세대 e호조 정기 동기화

```
스케줄러: 매일 06:00 (IF-004 §4.2)
  → e호조 API 호출
  → 신규/변경 contracts UPSERT
  → 도급공사(CONSTRUCTION) 위험성평가 자동 생성
  → sync_logs INSERT

본 화면의 동기화 이력에 자동 반영
```

### 6.4 [전체 동기화] / [매칭 실패 항목 재시도]

```
[전체 동기화]:
  changed_since 무시하고 모든 e호조 계약 재조회 → UPSERT
  (주의: 운영 부하 — CEO/GM만 가능)

[매칭 실패 항목 재시도]:
  WHERE source='EHOJO_AUTO' AND target_id IS NULL
  → target_id 매칭 로직 재실행 (관리대상 변경·신규 등록 후 재매칭에 유용)
```

## 7. 검증 규칙

| 항목 | 검증 |
|------|------|
| 평가서 코드 | UNIQUE |
| 평가서 항목 점수 합계 | max_score와 일치 |
| 서약서 HTML | XSS 검증 (sanitize) |
| 연동 설정 변경 | CEO만 가능 |

## 8. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 평가서 항목 점수 합계 불일치 | "항목별 점수 합계가 만점과 일치해야 합니다" | 인라인 |
| 사용 중 평가서 비활성화 시도 | "이 평가서는 진행 중 평가에 사용 중입니다. 비활성화하면 새 진입에만 영향" | 확인 모달 |
| 동기화 중 [전체 동기화] 재클릭 | "동기화가 이미 진행 중입니다" | 토스트 |

## 9. 관련 화면 흐름

```
CON10-S
   ├─ 평가서 마스터 [편집] → 평가서 편집 모달
   ├─ 서약서 마스터 [편집] → 서약서 편집 모달 (HTML 에디터)
   ├─ e호조 [수정] → 연동 설정 모달
   ├─ e호조 [테스트] → 응답 결과 표시
   ├─ e호조 [전체 동기화] → 비동기 작업 + 결과 모달
   └─ e호조 이력 [보기] → sync_logs 상세 모달
```
