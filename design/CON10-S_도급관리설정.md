## 4. 평가서 마스터 탭

```
┌──────────────────────────────────────────────────────────┐
│ 평가서 마스터                                              │
│                                                          │
│ 5분류별 평가서 + 일반 평가서 (결정사항 #12)               │
│                                                          │
│ 코드                          │계약유형     │만점│적격│ 관리│
│ ─────────────────────────────┼────────────┼───┼───┼────│
│ CONSTRUCTION_SAFETY          │공사         │100│ 70│[편집]│
│ SERVICE_SAFETY              │용역         │100│ 70│[편집]│
│ INSTALL_SAFETY              │구매설치     │100│ 70│[편집]│
│ CONSIGNMENT_SAFETY          │위탁사업     │100│ 70│[편집]│
│ GENERAL_SAFETY              │기타·공통    │100│ 70│[편집]│
│                                                          │
│                                          [+ 평가서 추가]   │
└──────────────────────────────────────────────────────────┘
```

### 4.1 데이터 소스

`evaluation_forms` 마스터 (DB-001 §5.4).

### 4.2 [편집] 모달

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 코드 | TEXT (UNIQUE) | ✅ | |
| 계약유형 | SELECT (5분류) | ✅ | |
| 만점 | NUMBER | ✅ | |
| 적격 기준 | NUMBER | ✅ | 만점 이하 |
| 항목 (JSON 또는 표 입력) | 다중 입력 | ✅ | 항목별 배점 합 = 만점 |
| 활성 여부 | BOOLEAN | ❌ | |

검증: 항목 배점 합계 = 만점. 활성·사용 중 평가서 비활성화 시 진행 중 평가에는 영향 없음 (역사 보존).

## 5. 서약서 마스터 탭

평가서와 동일 구조. 5분류별 서약서 + 일반 서약서.

```
┌──────────────────────────────────────────────────────────┐
│ 서약서 마스터                                              │
│                                                          │
│ 코드                          │계약유형     │전자서명│관리 │
│ ─────────────────────────────┼────────────┼───────┼─── │
│ CONSTRUCTION_AGREEMENT       │공사         │ 필수   │[편집]│
│ SERVICE_AGREEMENT            │용역         │ 필수   │[편집]│
│ ...                                                      │
│                                                          │
│                                          [+ 서약서 추가]   │
└──────────────────────────────────────────────────────────┘
```

### 5.1 [편집] 모달

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 코드 | TEXT (UNIQUE) | ✅ | |
| 계약유형 | SELECT (5분류) | ✅ | |
| 제목 | TEXT | ✅ | |
| 본문 HTML | HTML 에디터 | ✅ | XSS sanitize |
| 필수 첨부 항목 (JSON) | 다중 입력 | ❌ | 예: 사업자등록증·안전관리계획서 등 |
| 전자 서명 필수 | BOOLEAN | ✅ | |
| 활성 여부 | BOOLEAN | ❌ | |

## 6. 자동 처리

> e호조 연동 자동 처리(도급사업 자동 import 등)는 **SYS01-S §6.4**에서 관리.

본 화면은 콘텐츠 마스터 변경 영향만:

```
평가서·서약서 변경 시:
  - 이미 사용 중인 계약(contract_subcontractors.evaluation_form_id / agreement_form_id 참조)에는 영향 없음 (역사 보존)
  - 새로 진입하는 STEP 2 / STEP 4에서만 변경된 마스터 적용
  
평가서·서약서 비활성화 시:
  - 사용 중이면 비활성만 가능 (DELETE 차단)
  - 새 진입에만 영향
```

## 7. 버튼·링크 액션

| 요소 | 액션 |
|------|------|
| 탭 전환 | URL `?sub=` 동기화 |
| 상단 연동 상태 위젯 [→ 외부 시스템 연동 보기] | SYS01-S 이동 (e호조 탭) |
| 평가서·서약서 행 `[편집]` | 편집 모달 |
| `[+ 추가]` | 신규 추가 모달 |
| 행 클릭 | 미리보기 모달 (read-only) |

## 8. 검증·오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 평가서 항목 배점 합 ≠ 만점 | "항목 배점 합계가 만점과 일치해야 합니다" | 인라인 |
| 사용 중 평가서·서약서 삭제 시도 | "사용 중인 평가서는 삭제할 수 없습니다. 비활성화 처리하세요" | 인라인 안내 |
| 활성 평가서 0건 (해당 계약유형) | "{계약유형}의 활성 평가서가 없습니다. 신규 계약 진행 불가" | 운영자 경고 |

## 9. 관련 화면 흐름

```
CON10-S
   ├─ 상단 연동 위젯 [→ 외부 시스템 연동 보기] → SYS01-S e호조 탭
   ├─ 탭 [평가서 마스터] → 평가서 CRUD
   └─ 탭 [서약서 마스터] → 서약서 CRUD
```

## 10. 관련 DB 테이블

- `evaluation_forms` (DB-001 §5.4): 평가서 마스터
- `agreement_forms` (DB-001 §5.5): 서약서 마스터
- `audit_logs` (DB-001 §2.12): 설정 변경 감사

> e호조 연동 관련 테이블(`sync_logs`, `integration_configs`, `integration_mapping_rules`)은 SYS01-S에서 관리.

## 11. 관련 인터페이스

본 화면 자체는 외부 인터페이스 직접 호출 없음. 차세대 e호조 연동 정책(자동 계약 import)이 본 모듈에 영향을 미침 — SYS01-S에서 관리.
