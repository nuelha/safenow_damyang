## 4. 표시 항목 (위젯)

### 4.1 내 업무 요약

```
SM/SHM 본인이 담당자(target_assignees) 또는 작성자(created_by)인 항목 집계:
  - 위험성평가, 개선조치, 이행 항목, 점검 등
  - 진행 중: status='IN_PROGRESS' or 'PREPARING'
  - 완료: status='DONE' (올해 누적)
  - 기한초과: due_date < NOW() AND status != 'DONE'
```

### 4.2 시기도래 내 업무 (담당)

DSH01-V §4.4와 동일 색상 정책. 본인 담당 항목만 필터.

```
모듈 통합:
  - 정기 위험성평가 (RSK02-D 진입)
  - 이행 항목 (CMP01-V 진입)
  - 안전점검 (INS02-D 진입)
  - 안전보건교육 (Phase 3 EDU)
표시: 모듈 배지 + 제목 + 마감일 + [작성/실시/이행] 액션 버튼 (모듈별)
```

### 4.3 내 담당 관리대상

TGT01-V 요약 카드를 `target_assignees.user_id=:me` 스코프로 적용.

### 4.4 평가자 지정 위젯 (CON05-F 연계)

```
CON05-F STEP 2에서 본인이 평가자(evaluator_id)로 지정된 항목:
  WHERE contract_subcontractors.evaluator_id = :me
    AND evaluation_status IN ('PENDING', 'IN_PROGRESS')

표시: 업체명·계약명·[작성] 버튼 → CON08-D 진입
```

### 4.5 위험성평가 진행

본인 담당(`owner_user_id=:me`) 평가의 cycle_stage 별 카운트.

### 4.6 개선조치 (담당)

`improvements WHERE assigned_to=:me` 집계.

### 4.7 의견청취 위젯 (SHM 한정 노출)

```
SM은 미노출, SHM만 노출 (결정사항 #20 — 처리자는 SHM)

미처리 의견:
  WHERE assigned_to=:me OR (department_id 본 부서 AND status='PENDING')

긴급 분류 (작업중지요청)는 별도 강조 표시
```

### 4.8 알림 메시지

NTF02-D 동일.

| 요소 | 액션 |

## 6. 데이터 스코프

SM/SHM = `target_assignees`에 본인 매핑된 관리대상 + 본인 담당 작업.
SHM은 본 부서 의견청취도 추가 (assigned_to=:me OR department_id 본 부서).

## 7. 자동 갱신·캐싱

| 위젯 | 갱신 |
|------|------|
| 내 업무 요약 | 진입 시 + 5분 |
| 시기도래 | 진입 시 + 10분 |
| 평가자 지정 | 실시간 |
| 의견청취 (긴급) | 실시간 (작업중지요청은 즉시 표시) |
| 알림 | 30초 폴링 |

## 8. SM vs SHM 차이

| 항목 | SM | SHM |
|------|------|------|
| 의견청취 위젯 | 미노출 | 노출 |
| 평가자 지정 위젯 | 본인이 평가자인 경우만 (드물게) | 자주 노출 (주 평가자) |
| 개선조치 | 본인 담당 | 본인 담당 + 부서 전체 모니터링 |
| 산업안전보건위원회 안건 (Phase 3) | 미노출 | 노출 |
