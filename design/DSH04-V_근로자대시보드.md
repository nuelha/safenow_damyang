## 4. 표시 항목 (위젯)

### 4.1 내 소속 정보

| 항목 | 데이터 소스 |
|------|------------|
| 부서·직급 | `users.department_id` JOIN |
| 소속 관리대상 | `target_assignees WHERE user_id=:me AND role='WORKER'` |
| 본인 직종 | `users.worker_type` |

### 4.2 내 안전보건교육 (Phase 3 EDU 모듈 연계)

```
worker_type에 따른 법정 교육 기준 비교:
  사무직: 정기교육 반기 6h
  현장직: 정기교육 분기 6h
  관리감독자: 연 16h
  안전보건관리자: 신규선임 34h
  기타: 법정 기준 없음

본인 이수 시간 vs 필요 시간 → 진행률 + 마감일
```

### 4.3 위험·아차사고 신고 (SFR-011/019 OPN 모듈 연계)

```
근로자가 직접 등록 가능한 OPN 카테고리:
  - hazard_report (위험요인 신고 — SFR-019)
  - near_miss (아차사고 보고)
  - safety_proposal (안전제안)
  - work_stop_request (작업중지요청 — 긴급, 별도 표시)

본인이 등록한 의견 카운트 + 신규 등록 진입 버튼
```

### 4.4 의견청취 응답 요청 (Phase 3)

```
SHM이 부서 단위로 발송한 의견청취에 본인 응답 대기 중인 항목.
표시: 의견 제목 + 마감일 + [응답] 버튼 → OPN02-F (Phase 3)
```

### 4.5 내 소속 관리대상 현황

```
target_assignees.user_id=:me 매칭 관리대상 표시:
  - 관리대상명
  - 안전관리자(SAFETY_MGR) 이름·연락처
  - 관리책임자(RESPONSIBLE) 이름
  - 최근 위험성평가 일자 (참고용)

긴급 시 연락처 안내 의미
```

### 4.6 알림 메시지

NTF02-D 동일.

## 6. 데이터 스코프

WKR = 본인 소속 관리대상(`target_assignees.role='WORKER'`) + 본인 등록 의견·교육 이력만.

## 7. 자동 갱신·캐싱

| 위젯 | 갱신 |
|------|------|
| 소속 정보 | 진입 시 (변경 빈도 낮음) |
| 교육 이수 | 진입 시 + 1시간 |
| 의견청취 응답 요청 | 진입 시 (실시간) |

## 8. 디자인 가이드

- 정보 밀도 낮음 — 5-6개 위젯
- 긴급 연락 정보(안전관리자) 명확히 노출
- 위험 신고·아차사고 보고 버튼은 크고 눈에 띄게 (사고 직후 빠른 진입)
- 모바일 우선 디자인 (현장 직원도 접근 가능)
