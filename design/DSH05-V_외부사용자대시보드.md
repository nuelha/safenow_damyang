## 4. 표시 항목 (위젯)

### 4.1 SUB — 내 계약 현황

```
WHERE external_user_access.access_type='CONTRACT'
  AND external_user_access.user_id=:me
contracts JOIN으로 status 별 카운트
```

### 4.2 SUB — 서약서 작성 필요

```
WHERE contract_subcontractors.signed_at IS NULL
  AND contract_subcontractors.selected=true
  AND contracts.status='APPROVED' 또는 'IN_PROGRESS'
  AND 본인 계약

표시: 사업명·기간 + [서약서 작성하기] → CON07-F-SUB
```

### 4.3 SUB — TBM 작성 (도급공사만)

```
WHERE contracts.contract_type='CONSTRUCTION'
  AND contracts.status='IN_PROGRESS'
  AND 본인 계약

표시: 사업명 + 최근 TBM 일자 + [+ TBM 작성하기] → CON09-F
```

### 4.4 SUB — 내 계약 목록

CON01-L 본인 계약 한정 뷰.

### 4.5 CON — 위임 받은 부서·관리대상 요약

```
WHERE external_user_access.access_type='CONSULTING'
  AND external_user_access.user_id=:me

위임 부서·관리대상 카운트 (consulting_assignments — Phase 2)
```

### 4.6 CON — 작업 대기

```
위임 범위 내 본인이 작성 가능한 작업:
  - 위험성평가 작성 (위임 부서·관리대상에 대해 SHM이 위임한 경우)
  - 이행 점검표 작성 (Phase 3)
  - 안전계획 수립 보조 (Phase 2)

표시: 모듈별 대기 건수 + [전체 보기] 링크
```

### 4.7 CON — 위임 부서 목록

위임 받은 부서 + 산하 관리대상 카운트.

### 4.8 알림 메시지

NTF02-D 동일. SUB/CON는 이메일로도 받음.

## 5. 버튼·링크 액션

### 5.1 SUB

| 요소 | 액션 |
|------|------|
| [서약서 작성하기] | CON07-F-SUB `/external/contracts/:id/agreement` |
| [+ TBM 작성하기] | CON09-F `/external/tbm/new?contract_id=:id` |
| 계약 행 클릭 | CON01-D (본인 계약만, 일부 탭만 노출) |
| [전체 보기] (계약 목록) | CON01-L 외부 사용자 view |

### 5.2 CON

| 요소 | 액션 |
|------|------|
| 위임 부서 행 클릭 | TGT01-V `?department_id=:id` (위임 범위 한정) |
| 작업 대기 행 클릭 | 해당 모듈 작업 화면 (위험성평가 → RSK02-F 등) |
| 알림 행 클릭 | NTF02-D 동일 |

## 6. 데이터 스코프

| 권한 | 보이는 범위 |
|------|----------|
| SUB | `external_user_access.access_type='CONTRACT'` 본인 계약만 |
| CON | `external_user_access.access_type='CONSULTING'` 위임 부서·관리대상 |

## 7. 자동 갱신·캐싱

| 위젯 | 갱신 |
|------|------|
| 서약서 작성 필요 (SUB) | 실시간 (긴급) |
| TBM 작성 (SUB) | 진입 시 + 10분 |
| 위임 범위 (CON) | 진입 시 + 30분 |
| 알림 | 30초 폴링 |

## 8. 디자인 가이드

- 최소 구성 — 핵심 작업 진입점 위주
- SUB 대시보드: 서약서·TBM 등 작업 진입 버튼 강조
- CON 대시보드: 위임 범위 명확히 표시 (어떤 부서·관리대상에 접근 가능한지)
- 본인이 접근 못 하는 영역은 메뉴·링크 자체를 안 보이게 (404 페이지 노출 최소화)

## 9. 외부 사용자 GNB 구성 (IA-001 §2.2 참조)

```
SUB:
  - 홈 (DSH05-V)

  - 서약서·TBM (계약 컨텍스트에서 진입)
  - 알림 / 내 정보

CON:
  - 홈 (DSH05-V)
  - 위임 받은 부서 (TGT01-V 위임 한정)

  - 알림 / 내 정보
```

## 10. 관련 화면 흐름

```
SUB:
   AUTH02-F → /external/dashboard (DSH05-V)
      ├─ [서약서 작성하기] → CON07-F-SUB
      ├─ [+ TBM 작성하기] → CON09-F (?contract_id=)
      ├─ 계약 행 → CON01-D (제한된 탭)
      └─ 알림 → NTF02-D

CON:
   AUTH02-F → /external/dashboard
      ├─ 위임 부서 행 → TGT01-V (위임 한정)
      ├─ 작업 대기 → RSK02-F 등
      └─ 알림 → NTF02-D
```

## 11. 관련 DB 테이블

- `users` (DB-001 §2.2): auth_source='LOCAL' 사용자
- `external_user_access` (DB-001 §2.5): 접근 범위 매핑
- `contracts`, `contract_subcontractors` (DB-001 §5.1, §5.3): SUB 계약·서약
- `tbm_records` (DB-001 §5.7): SUB TBM
- `consulting_assignments` (Phase 2 — 본 문서에서 임시 정의):
  ```sql
  CREATE TABLE consulting_assignments (
    id              UUID PRIMARY KEY,
    consulting_user_id UUID NOT NULL REFERENCES users(id),
    department_id   UUID REFERENCES departments(id),
    target_id       UUID REFERENCES targets(id),  -- 부서 또는 관리대상 단위
    scope           ENUM('FULL', 'READ_ONLY', 'PARTIAL'),
    granted_by      UUID NOT NULL REFERENCES users(id),
    granted_at      TIMESTAMP DEFAULT NOW(),
    expires_at      TIMESTAMP
  );
  ```

- 데이터 스코프 미들웨어 (IA-001 §6.3)에서 모든 데이터 접근 검증
