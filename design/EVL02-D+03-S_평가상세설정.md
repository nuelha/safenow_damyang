### A.4 표시 항목

| 항목 | 데이터 소스 |
|------|------------|
| 대상자 정보 | JOIN `users.name + position` (subject_user_id) |
| 사이클 | JOIN `evaluation_cycles` |
| 평가자 | JOIN `users.name` (evaluator_id) |
| 평가 항목 | `evaluation_items` (마스터 매핑) |
| 항목별 점수 입력 | `evaluation_items.score` |

### A.5 역할별 평가 항목

```
RESPONSIBLE (안전보건관리책임자) — 8항목:
  §1. 안전보건 경영방침 수립 및 시행 (15점)
  §2. 안전보건 목표 설정 및 달성 노력 (10점)
  §3. 안전보건관리체계 구축 (15점)
  §4. 안전보건 예산 편성 및 집행 (10점)
  §5. 안전관리자 등 인력 배치 및 관리 (10점)
  §6. 위험성평가·점검 실시 및 결과 관리 (15점)
  §7. 도급사업 안전보건 관리 (10점)
  §8. 종사자 의견청취 및 반영 (15점)
  -- 총 100점

SUPERVISOR (관리감독자) — 5항목:
  §1. 작업 전 안전점검 실시 (25점)
  §2. 종사자 안전수칙 교육 및 관리 (20점)
  §3. 위험요인 발견 및 보고 (20점)
  §4. 안전장비·보호구 착용 관리 (15점)
  §5. 사고 발생 시 응급조치 및 보고 (20점)
  -- 총 100점

마스터: evaluation_template_items (EVL03-S에서 관리)
```

### A.6 실시간 합계·결과 산출

```
점수 입력 시 blur 이벤트:
  total_score = SUM(evaluation_items.score)
  result = 
    score >= 90 ? 'EXCELLENT'
    score >= 70 ? 'GOOD'
    score >= 50 ? 'NEEDS_IMPROVEMENT'
    : 'INSUFFICIENT'

본문 하단 표시 (실시간)
```

### A.7 버튼·링크 액션

| 요소 | 액션 |
|------|------|
| `[← 목록]` | EVL01-L |
| `[PDF 출력]` | 평가서 인쇄용 |
| `[수정]` | 항목 재입력 (`status != 'COMPLETED'`일 때만) |
| `[임시저장]` | `status='IN_PROGRESS'` + 부분 점수 저장 |
| `[평가 완료]` | 검증 → `status='COMPLETED'` + 결과 산출 |
| `[결재요청]` | 안내 팝업 → `approval_status='REQUESTED'` + 새올 알림 |
| `[결재 결과 업데이트]` | 결과 입력 모달 (RSK02-D §7.1.2 동일) |
| `[파일 첨부]` (각 항목) | 증빙 파일 업로드 |

### A.8 자동 처리

#### A.8.1 [평가 완료]

```
검증:
  - 모든 항목 점수 입력됨
  - 종합 의견 1~2000자

저장:
  1. evaluations:
     - total_score, result 산출 저장
     - status='COMPLETED'
     - completed_at=NOW()
  2. 대상자에게 새올 포틀릿 알림 ("평가 완료 — 결과 확인")
  3. audit_logs INSERT
```

#### A.8.2 [결재요청]

```
RSK02-D §7.1 동일 패턴:
  - 안내 팝업 → status='REQUESTED'
  - 새올 알림 (CEO/GM, category='approval')
```

### A.9 권한별 동작

| 권한 | 동작 |
|------|------|
| 평가자 (evaluator_id) | 점수 입력·평가 완료·결재요청 |
| 대상자 (subject_user_id) | 본인 평가 read-only 열람만 |
| 다른 SHM·SM | 본 부서 평가 열람 (수정 불가) |
| CEO/GM | 전체 열람·결재 결과 등록 |

### A.10 검증

| 항목 | 검증 |
|------|------|
| 평가자 권한 | `evaluator_id=:me`만 점수 입력 가능 |
| 점수 범위 | 0 ≤ score ≤ max_score |
| 본인 평가 차단 | `evaluator_id=subject_user_id` 차단 |
| 결재 요청 | 평가 완료 후만 |

## 공통 — 관련 화면 흐름

```
EVL01-L 행 클릭 → EVL02-D
   ├─ 점수 입력 → 자동 산출
   ├─ [평가 완료] → status='COMPLETED' + 결과 자동
   ├─ [결재요청] → 안내 팝업 + 새올 알림
   └─ [결재 결과 업데이트] → status 변경

EVL03-S
   ├─ [평가 항목 마스터] → 마스터 편집
   ├─ [사이클 관리] → 사이클 CRUD
   └─ [평가자 매핑 규칙] → 규칙 CRUD
```

## 공통 — 관련 DB 테이블

```sql
-- 평가 항목 마스터
CREATE TABLE evaluation_template_items (
  id              UUID PRIMARY KEY,
  evaluation_role ENUM('RESPONSIBLE','SUPERVISOR') NOT NULL,
  item_no         INTEGER NOT NULL,
  item_title      VARCHAR(500) NOT NULL,
  evaluation_criteria TEXT,                          -- 평가 기준 (가이드)
  max_score       INTEGER NOT NULL,
  is_active       BOOLEAN DEFAULT true,
  UNIQUE (evaluation_role, item_no)
);

-- 평가자 매핑 규칙
CREATE TABLE evaluator_mapping_rules (
  id              UUID PRIMARY KEY,
  rule_name       VARCHAR(200),
  subject_role    ENUM('RESPONSIBLE','SUPERVISOR') NOT NULL,
  evaluator_source ENUM('SAME_DEPT_SHM','PARENT_DEPT_SHM','ASSIGNED'),
  priority        INTEGER,
  is_active       BOOLEAN DEFAULT true
);
```

EVL01-V §공통 §관련 DB 참조 (`evaluations`, `evaluation_items`, `evaluation_cycles`).
