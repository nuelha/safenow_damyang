## 4. 표시 항목

### 4.1 목록 컬럼

| 컬럼 | 데이터 소스 | 비고 |
|------|------------|------|
| 출처 배지 | `improvements.source_type` | 위험성평가·점검·의견·사고·수동 |
| 관리대상 | JOIN `targets.name` + type 배지 | |
| 제목 | `description` 앞 50자 + ... | 클릭 시 IMP01-D |
| 담당자 | JOIN `users.name` (`assigned_to`) | 미지정 시 "—" + [지정] |
| 기한 | `due_date` + D-N 산출 | 색상: D-1주 노랑 / D-당일 주황 / D+ 빨강 |
| 상태 | `status` 배지 | PENDING / IN_PROGRESS / DONE |

### 4.2 출처 배지

| source_type | 표시명 | 색상 |
|------------|--------|------|
| `risk_assessment` | 위험성평가 | 파랑 |
| `inspection` | 안전점검 | 보라 |
| `opinion` | 의견청취 | 노랑 |
| `incident` | 사고 (재발방지) | 빨강 |
| `manual` | 수동 등록 | 회색 |

## 7. 자동 처리 로직

본 화면 자체 트리거 없음. 다음 자동 작업이 본 화면에 반영:

- **위험성평가 불허 자동 INSERT** (DB-001 §9.3): 위험성평가 위험요인 `is_acceptable=false` + 감소대책 입력 시 → `improvements` 자동 생성
- **사고 등록 시 자동 INSERT**: incidents INSERT 후 재발방지조치 수립 단계로 `improvements` 자동 생성 (결정사항 #19 — 2단계)
- **안전점검 결과 불합격 시 자동 INSERT**: Phase 3 INS 모듈 결과 → `improvements`
- **의견청취 채택 시 자동 INSERT**: Phase 3 OPN 모듈 결과 → `improvements`

## 10. IMP02-F 개선조치 수동 등록 폼 (보조)

> 보통 자동 생성되나 운영자가 직접 등록 가능.

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 출처 | 자동 'manual' | — | readonly |
| 관리대상 | SELECT (targets) | ✅ | 권한 범위 내 |
| 제목·내용 | TEXTAREA | ✅ | |
| 감소대책·조치 내용 | TEXTAREA | ❌ | 등록 시 미입력 가능 |
| 담당자 | USER_SEARCH | ❌ | 미지정 시 추후 IMP01-M에서 지정 |
| 기한 | DATE | ✅ | |
| 첨부 | FILE | ❌ | |

저장: improvements INSERT (source_type='manual') + 담당자에게 새올 알림.

## 11. IMP01-M 담당자 지정·변경 모달 (보조)
