## 4. 표시 항목

### 4.1 알림 행 구성

| 영역 | 내용 |
|------|------|
| 좌측 아이콘 | 카테고리별 색상 점 (🔴🟢🟠🔵🟡🟣🟤⚪) |
| 카테고리 배지 | 카테고리 표시명 ([결재]/[지정]/...) |
| 제목 | `notifications.title` |
| 시각 | 오늘: HH:MM / 어제: "어제 HH:MM" / 이전: YYYY-MM-DD HH:MM |
| 본문 | `notifications.body` (옵션 — 1줄 truncate) |
| 액션 버튼 | `link_url` 있으면 [바로가기] 또는 카테고리별 텍스트 |
| 읽음 여부 | `read_at IS NULL` 이면 굵게 + 좌측 인디케이터 |

### 4.2 카테고리·아이콘 매핑 (DB-001 §8.5)

| 코드 | 표시명 | 아이콘 색상 |
|------|--------|-----------|
| approval | 결재 | 🔴 빨강 |
| assignment | 지정·매핑 | 🟢 초록 |
| compliance | 이행 | 🟠 주황 |
| inspection | 점검 | 🔵 파랑 |
| risk | 위험성평가 | 🟡 노랑 |

| contract | 도급계약 | 🟤 갈색 |
| system | 시스템 | ⚪ 회색 |

### 4.3 시간대 그룹화

- 오늘 (00:00 ~ 23:59 현재일)

- 이번 주 (월요일~)

## 7. 자동 처리 로직

### 7.1 알림 적재 (외부 트리거)

본 화면은 알림 적재 대상이 아니라 표시. 적재는 모듈별 이벤트 트리거에서:

```
이벤트 발생 → notifications INSERT (status='NEW')
  ↓
비동기 워커 → 새올 포틀릿 적재 (SSO) 또는 이메일 발송 (LOCAL)
  → status='SENT' / 'FAILED' / 'RETRY' 업데이트
```

> 본 화면(NTF01-L)은 `notifications` 테이블의 본인 행을 표시 (status 무관 — NEW/SENT/FAILED 모두 노출).

### 7.2 자동 읽음 처리

```
알림 행 클릭 시 (link_url 존재):
  1. read_at=NOW() UPDATE
  2. link_url 이동
  
[모두 읽음 처리] 클릭:
  1. WHERE recipient_user_id=:current_user AND read_at IS NULL → read_at=NOW()
  2. 안 읽음 카운트 0으로 갱신 (GNB 상단 배지 동기화)
```

### 7.3 NTF02-D 드롭다운과 동기화

- NTF01-L에서 [모두 읽음 처리] 시 NTF02-D 배지 카운트도 즉시 0
- 알림 클릭 후 NTF01-L 복귀 시 해당 알림 읽음 상태로 표시

## 8. 데이터 스코프

- 본인 알림만 조회: `WHERE recipient_user_id = :current_user`
- 관리자(NTF03-S 운영 화면)에서는 전체 알림 조회 가능 — 별도 화면

## 9. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 알림 없음 | "받은 알림이 없습니다" | 빈 화면 + 안내 |
| 필터 조합 결과 없음 | "조건에 맞는 알림이 없습니다" | 빈 목록 + 필터 초기화 |
| 모두 읽음 처리 확인 모달 | "안 읽음 알림 N건을 모두 읽음 처리하시겠습니까?" | [취소] [확인] |

## 10. 관련 화면 흐름

```
NTF01-L
   ├─ 알림 행 클릭 → notifications.link_url 이동
   │  (예: 결재 알림 → /risk-assessment/:id, 지정 알림 → /targets/:id)
   ├─ [모두 읽음 처리] → 확인 모달 → 일괄 read_at 갱신
   └─ GNB [🔔] → NTF02-D 드롭다운
```

## 11. 외부 사용자(LOCAL)의 알림

```
LOCAL 사용자는 새올 포틀릿 미사용 → 이메일 수신
이메일 본문의 링크 클릭 → 본 시스템 로그인 → /external/dashboard
대시보드 또는 NTF01-L에서 알림 내역 확인 가능
```

## 12. 관련 DB 테이블

- `notifications` (DB-001 §2.10): 알림 본체
- `users` (DB-001 §2.2): 사용자 정보 (auth_source 분기)

## 13. 관련 인터페이스

- **IF-005 새올 포틀릿**: SSO 사용자 알림 적재 (간접 — 본 화면은 시스템 내 표시만)
- **IF-006 SMTP**: LOCAL 사용자 이메일 발송 (간접)

## 14. 성능 고려

- 페이지네이션 (페이지당 20건 기본)
- 인덱스 활용: `idx_notifications_recipient` (recipient_user_id, created_at DESC)
- 안 읽음 카운트는 캐싱 (1분) — read_at 변경 시 invalidate
- 시간대 그룹화는 클라이언트 측 렌더링
