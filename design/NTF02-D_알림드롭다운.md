## 4. 표시 항목

### 4.1 GNB 아이콘

| 항목 | 데이터 소스 | 비고 |
|------|------------|------|
| 종 아이콘 | 정적 | 안 읽음 1건 이상 시 펄스 애니메이션 |
| 안 읽음 카운트 배지 | `COUNT(notifications WHERE recipient_user_id=:current_user AND read_at IS NULL)` | 99 초과 시 "99+" 표시 |

### 4.2 드롭다운 헤더

| 항목 | 내용 |
|------|------|
| "알림 (안 읽음 N건)" | 안 읽음 카운트 |
| `[모두 읽음]` | 안 읽음 1건 이상 시 활성 |

### 4.3 알림 행 (최대 5건)

| 영역 | 내용 |
|------|------|
| 좌측 아이콘 | 카테고리별 색상 점 (🔴🟢🟠🔵🟡🟣🟤⚪) — NTF01-L §4.2 매핑 |
| 카테고리 배지 | 카테고리 표시명 ([결재]/[지정]/...) |
| 제목 | `notifications.title` |
| 시각 | 오늘: HH:MM / 어제: "어제 HH:MM" / 이전: YYYY-MM-DD HH:MM |
| 안 읽음 강조 | `read_at IS NULL` 이면 굵게 + 좌측 인디케이터 |

### 4.4 표시 규칙

- 최근 5건 표시 (안 읽음 우선, 그 다음 최신순)
- 안 읽음이 5건 이상이면 안 읽음만 5건 표시 (전체 보기로 안내)
- 5건 초과 시 [전체 보기] 활성 (5건 이하면 비활성 — 모두 표시 중)

## 6. 자동 처리 로직

### 6.1 실시간 안 읽음 카운트 갱신

```
세션 활성 동안 폴링 (30초마다):
  GET /api/notifications/count
  응답: { unread_count: N, new_since_last: M }

처리:
  - unread_count → 배지 숫자 갱신
  - new_since_last >= 1 → [🔔] 아이콘 펄스 애니메이션
  - new_since_last >= 1 + 사운드 옵션 (사용자 설정, 기본 OFF)
```

### 6.2 드롭다운 열기 시 데이터 로드

```
[🔔] 클릭:
  GET /api/notifications?limit=5&order=unread_first,created_at_desc
  응답: notifications 최근 5건 (안 읽음 우선)
  → 드롭다운 렌더링
  → 드롭다운 열림 시점부터의 new_since_last=0 으로 리셋 (펄스 정지)
```

### 6.3 알림 클릭 시 처리

```
알림 행 클릭:
  1. PATCH /api/notifications/:id { read_at: NOW() } (옵티미스틱)
  2. 드롭다운 닫기
  3. link_url 이동 (window.location)
  4. 안 읽음 카운트 -1 (옵티미스틱 — 서버 응답으로 정정)
```
