# NTF03-S — 알림 운영 화면 (적재 현황·실패 큐)

> 화면 ID: **NTF03-S**
> 모듈: NTF (알림)
> SFR: SFR-017
> 버전: **v1.1** (새올/SMTP 설정을 SYS01-S로 이관)
> 작성일: 2026-05-11 (v1.0) / 2026-05-13 (v1.1)
> 관련 문서: SRS-001 §3 SFR-017, DB-001 §2.10 notifications / §2.11 email_queue

> ※ **v1.1 변경**: 결정사항 #24에 따라 새올 포틀릿·SMTP 연동 설정·이메일 템플릿 관리는 **SYS01-S 외부 시스템 연동**으로 이관. 본 화면은 알림 자체의 운영(적재 현황·실패 큐·재시도·이력)만 관리.

```
┌────────────────────────────────────────────────────┐

│                                                    │
│ 최근 24시간                                         │
│ ─────────────────────────────────                  │
│ 적재 시도 1,234건 / 성공 1,210건 / 실패 24건         │
│ 평균 적재 소요: 0.8초                                │
│                                                    │
│ 카테고리별 발송 통계 (이번 주)                       │
│ [그래프 — 카테고리별 막대]                          │
│                                                    │
│ 채널별 통계 (이번 달)                                │
│ 새올 포틀릿: N건 (성공률 99%)                        │
│ 이메일 (외부): N건 (성공률 95%)                      │
│                                                    │
│ 최근 알림 이력 (최근 100건)                          │
│ 일시        │대상자  │채널    │카테고리│상태│ 액션 │
│ ───────────┼──────┼──────┼──────┼────┼──── │
│ 14:23      │홍길동 │포틀릿  │결재   │성공│[보기]│
│ 14:22      │김도급 │이메일  │계약   │성공│[보기]│
│ 14:20      │이안전 │포틀릿  │이행   │실패│[재시도]│
│ ...                                                │
└────────────────────────────────────────────────────┘
```

### 4.1 표시 항목

| 항목 | 데이터 소스 |
|------|------------|
| 적재 시도·성공·실패 (24h) | `notifications` GROUP BY status WHERE created_at >= NOW - 24h |
| 평균 적재 소요 | `AVG(sent_at - created_at)` WHERE status='SENT' |
| 카테고리별 발송 통계 | GROUP BY category, week_of_year |
| 채널별 통계 | auth_source 기반 산출 (SSO=새올 / LOCAL=이메일) |
| 최근 알림 이력 | `notifications` ORDER BY created_at DESC LIMIT 100 |

`notifications` WHERE status IN ('FAILED', 'RETRY'). 재시도 정책은 SYS01-S §6.2 (5단계 백오프) 정의.

| 요소 | 액션 |

|------|------|

| 행 `[상세 로그]` | sync_logs 또는 notifications 상세 모달 |

## 7. 자동 처리 로직

> 새올·SMTP 적재 비동기 워커 / 재시도 백오프 / 운영자 알림 정책은 **SYS01-S §6**에서 관리.

본 화면은 알림 자체의 운영 흐름만:

### 7.1 [전체 재시도]

```
WHERE status='FAILED' OR status='RETRY' 모든 행:
  UPDATE status='NEW', retry_count=0, next_retry_at=NOW()
SYS01-S에서 정의한 워커가 즉시 다시 처리
```

### 7.2 [전체 무시]

```
WHERE status='FAILED' OR status='RETRY':
  UPDATE status='IGNORED'
재시도 대상에서 제외 (관리자 수동 처리)
```

## 11. 관련 DB 테이블

- `notifications` (DB-001 §2.10): 알림 적재 큐 (본 화면 핵심)
- `email_queue` (DB-001 §2.11): 이메일 발송 큐

> 외부 시스템 연동 설정·매핑 규칙(`integration_configs`, `integration_mapping_rules`)은 SYS01-S에서 관리.

## 12. 관련 인터페이스

본 화면 자체는 외부 인터페이스 직접 호출 없음. 새올 포틀릿(IF-005)·SMTP(IF-006) 연동 설정은 SYS01-S에서 관리.
