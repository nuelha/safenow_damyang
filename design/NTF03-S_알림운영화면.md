## 4. 표시 항목

### 4.1 적재 현황 탭

| 항목 | 데이터 소스 |
|------|------------|
| 적재 시도·성공·실패 (24h) | `notifications` GROUP BY status WHERE created_at >= NOW - 24h |
| 평균 적재 소요 | `AVG(sent_at - created_at)` WHERE status='SENT' |
| 카테고리별 발송 통계 | GROUP BY category, week_of_year |
| 채널별 통계 | SSO=새올, LOCAL=이메일 분리 집계 |
| 최근 알림 이력 | `notifications` ORDER BY created_at DESC LIMIT 100 |

### 4.2 실패 큐 탭

| 컬럼 | 데이터 소스 |
|------|------------|
| 일시 | `notifications.created_at` |
| 대상자 | JOIN `users.name` |
| 채널 | auth_source 기반 산출 |
| 카테고리 | `notifications.category` |
| 재시도 회수 | `retry_count` / 최대 5 |
| 오류 | `error_message` |

## 6. 자동 처리 로직

### 6.1 알림 적재 (비동기 워커)

```
스케줄러: 1초마다 큐 폴링
  SELECT FROM notifications WHERE status IN ('NEW', 'RETRY')
    AND (status='NEW' OR next_retry_at <= NOW())
    LIMIT 50

각 알림 처리:
  users.auth_source 분기:
    - SSO → 새올 포틀릿 API 호출 (IF-005 §5.3)
    - LOCAL → email_queue INSERT (IF-006 §6.3)
  
  결과:
    - 성공: status='SENT', sent_at=NOW()
    - 실패: status='RETRY', retry_count++, next_retry_at 백오프 계산
            5회 초과 시 status='FAILED'
```

### 6.2 재시도 백오프 (공통)

| 단계 | 간격 | 비고 |
|------|------|------|
| 1차 | 1분 | |
| 2차 | 5분 | |
| 3차 | 15분 | |
| 4차 | 1시간 | |
| 5차 | 6시간 | |
| 5회 초과 | status='FAILED' | 운영자 알림 |

### 6.3 [전체 재시도]

```
WHERE status='FAILED' OR status='RETRY' 모든 행:
  UPDATE status='NEW', retry_count=0, next_retry_at=NOW()
워커가 즉시 다시 처리
```

### 6.4 [전체 무시]

```
WHERE status='FAILED' OR status='RETRY':
  UPDATE status='IGNORED'
재시도 대상에서 제외 (관리자 수동 처리)
```

### 6.5 운영자 알림 정책

| 상황 | 알림 대상 | 카테고리 |
|------|---------|---------|
| 동기화 실패 (1회) | SHM+ | system |
| 동기화 연속 실패 (3회 이상) | CEO/GM/SHM | system (긴급) |
| 외부 API 인증 실패 | CEO/GM/SHM (즉시) | system (긴급) |
| 알림 발송 실패 누적 (50건/시간 초과) | SHM+ | system |
