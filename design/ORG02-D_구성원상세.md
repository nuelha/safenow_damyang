## 3. 레이아웃

### 3.1 화면 내부 서브 탭 구성

본 상세 화면은 한 사용자의 정보를 4개 영역으로 분할 표시 — 페이지 내부 서브 탭으로 전환 (모듈 화면 간 이동과 무관 / 같은 화면 안의 컨텍스트 전환).

```
[기본정보] [자격증] [교육이수] [담당 관리대상]
```

## 7. 자동 처리 로직

### 7.1 권한 변경 시 감사 로그

```
권한 변경 (CEO/GM이 ORG02-M에서 변경):
  1. users.permission_role UPDATE
  2. audit_logs INSERT (action='PERMISSION_CHANGE',
                       before_data={role: OLD}, after_data={role: NEW},
                       target_user_id, changed_by)
  3. 본인에게 새올 포틀릿 알림 (category='system')
```

### 7.2 비활성화 시 처리

```
[비활성화] 클릭:
  확인 모달: "{이름}을 비활성화 처리하시겠습니까?
            target_assignees 매핑은 유지되며,
            추후 활성화 시 자동 복원됩니다."
  
  비활성 처리:
  1. users.is_active = false
  2. SSO 사용자: 동기화 예외 플래그 ON (행정포털 재동기화로 자동 복원되지 않도록)
  3. 진행 중 세션 무효화
  4. audit_logs INSERT
```

### 7.3 부서 이동 (MANUAL 사용자만)

```
[부서 이동] 클릭 → 부서 선택 모달:
  1. users.department_id 변경
  2. target_assignees는 변경하지 않음 (별도 관리 — 부서 변경이 담당 자동 해제 의미 아님)
  3. audit_logs INSERT
```

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| SSO 사용자 부서 변경 시도 | 차단 ("SSO 사용자는 행정포털에서 부서 변경") |
| 권한 변경 시 CEO/GM 아닌 사용자 시도 | 차단 (403) |
| 본인 권한 변경 시도 | 차단 ("본인 권한은 변경할 수 없습니다") |
| 자격증 만료일 < 취득일 | 차단 |

## 9. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 사용자 미존재 | "사용자를 찾을 수 없습니다" | 404 페이지 + ORG02-L 이동 |
| 권한 부족 | "접근 권한이 없습니다" | AUTH99-V |
| 비활성 상태에서 권한 변경 시도 | "비활성 사용자의 권한을 먼저 활성화하세요" | 인라인 에러 |
