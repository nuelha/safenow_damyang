## 8. 자동 처리 로직

### 8.1 서버 검증 ([검증 시작])

```
1. 파일 파싱 (xlsx → JSON)
2. 각 행 검증:
   - 필수 컬럼 검증
   - 이메일 형식 검증
   - 이메일 중복 검증 (DB 기존 + 파일 내 중복)
   - 부서명 매칭 (departments.name)
   - 권한 코드 검증 (ENUM)
   - 직종 코드 검증 (ENUM)
   - 현재 사용자의 권한 부여 가능 범위 검증
3. 결과 분류: 정상 행 / 오류 행
4. 응답: { valid: [...], errors: [...] }
```

### 8.2 [정상 행만 등록]

```
1. 통과 행만 users 일괄 INSERT (트랜잭션):
   - auth_source='SSO' (내부 직원 가정)
     -- 실질적으로 source_type='MANUAL' AND auth_source='SSO' 운영
   - source_type='MANUAL'
   - external_id NULL
   - is_active=true
   - 부서명 → department_id 매핑
   - 권한 코드 → permission_role 매핑
2. 등록된 사용자들에게 새올 포틀릿 알림 (가능 시 — 행정포털 매핑 안 된 사용자는 알림 발송 불가)
3. 운영자(SHM+ 본인)에게 새올 포틀릿 알림 (요약: "구성원 N건 일괄 등록 완료")
4. audit_logs INSERT (action='USER_BULK_CREATE', metadata={count, file_name})
5. STEP 4 결과 표시
```

### 8.3 오류 로그 다운로드

```
오류 내역 엑셀 생성 (xlsx):
  - 컬럼: 행 N / 컬럼 / 오류 메시지 / 원본 데이터
  - 파일명: employees_errors_{YYYYMMDD_HHMMSS}.xlsx
  - 다운로드 헤더 설정
```

## 9. 검증 규칙 (입력 자체)

| 항목 | 검증 |
|------|------|
| 파일 형식 | .xlsx만 허용 |
| 파일 크기 | 최대 10MB (1만 행 정도) |
| 데이터 행 수 | 1건 이상 (헤더 제외) |
| 권한 부여 가능 범위 | 현재 사용자 권한에 따라 (CEO/GM은 전체, SHM은 SM/SHM/WKR까지) |

## 10. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 파일 형식 오류 | "엑셀(.xlsx) 파일만 업로드 가능합니다" | 인라인 안내 |
| 파일 크기 초과 | "파일 크기는 10MB 이하여야 합니다" | 인라인 안내 |
| 파일 파싱 오류 | "파일을 읽을 수 없습니다. 양식을 다시 확인해주세요" | 인라인 안내 |
| 모든 행 오류 (통과 0건) | "등록 가능한 행이 없습니다. 오류를 수정 후 재업로드해주세요" | [정상 행만 등록] 비활성 |
| 트랜잭션 실패 | "일괄 등록 중 오류가 발생했습니다. 다시 시도해주세요" | 토스트 + STEP 2로 복귀 |
