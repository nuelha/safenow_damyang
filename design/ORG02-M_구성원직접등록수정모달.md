## 2. 진입 경로

- ORG02-L `[+ 직접 등록]` (신규 모드)
- ORG02-D `[수정]` (수정 모드 — 사용자 정보 prefill)

> 행정포털 미연동 인원(예: 임시 계약직, 외부 위임 담당자 등) 직접 등록 시 사용.
> 행정포털 SSO 동기화로 자동 생성된 사용자(`source_type='PORTAL_SYNCED'`)는 본 모달에서 일부 필드 readonly.

## 4. 입력 항목

| 필드 | 타입 | 필수 | 검증 | 비고 |
|------|------|------|------|------|
| 이름 | TEXT | ✅ | 1~100자 | SSO 사용자 readonly (행정포털 동기화) |
| 이메일 | EMAIL | ✅ | 형식 + UNIQUE | SSO 사용자도 SHM+ 보완 가능 |
| 연락처 | TEL | ❌ | | |
| 부서 | SELECT (departments) | ✅ | 트리 selector | SSO 사용자 readonly |
| 직급 | TEXT | ✅ | | SSO 사용자 readonly |
| 직종 | SELECT (worker_type) | ❌ | office/field/supervisor/safety_manager/etc | 모든 사용자 SHM+ 변경 가능 (담양 자체 관리) |
| 권한 코드 | SELECT (permission_role) | ✅ | CEO/GM/SM/SHM/WKR | CEO/GM만 변경 가능 |
| 비고 | TEXTAREA | ❌ | 최대 500자 | |

> 외부 사용자(`auth_source='LOCAL'`, SUB/CON)는 본 모달 대상 아님. AUTH03-M에서 별도 처리.

## 5. 권한별 입력 제어

### 5.1 신규 모드 (직접 등록)

| 필드 | 가능 여부 |
|------|---------|
| auth_source | 자동 (SSO 가능 사용자는 행정포털 동기화로 들어와야 함. 본 모달은 source_type='MANUAL' 직접 등록 케이스) |
| permission_role | SHM은 SM/SHM/WKR까지만 부여 가능 / CEO/GM은 모든 권한 가능 |

### 5.2 수정 모드

| 사용자 출처 | 수정 가능 필드 |
|------------|--------------|
| `PORTAL_SYNCED` (SSO 동기화) | 직종(`worker_type`), 이메일·연락처 보완, 권한 (CEO/GM만) |
| `MANUAL` (직접 등록) | 이름·이메일·연락처·부서·직급·직종·권한·비고 (권한은 CEO/GM만) |
| `LOCAL_INVITED` (외부 사용자) | 본 모달 대상 아님 — AUTH06-S에서 처리 |

## 7. 저장 동작

### 7.1 신규 등록

```
1. 필수 필드 검증
2. 이메일 UNIQUE 검증 (시스템 내)
3. 권한 부여 권한 검증 (CEO/GM만 GM 이상 권한 부여 가능)
4. users INSERT:
   - auth_source='SSO' (행정포털 미연동이어도 내부 직원은 SSO 가정)
     -- 실질적으로는 source_type='MANUAL' AND auth_source='SSO' 케이스로 운영
   - source_type='MANUAL'
   - external_id NULL (행정포털 연동 안 됨)
   - department_id, name, email, phone, position, permission_role, worker_type
   - is_active=true
5. audit_logs INSERT (action='USER_DIRECT_CREATE')
6. 본인에게 새올 포틀릿 알림 ("담양 안전관리시스템에 등록되었습니다") — 단, 행정포털 매핑 안 된 상태이므로 알림 발송 불가일 수도 있음 (협의 필요)
7. 모달 닫기 + ORG02-L 목록 새로고침
```

### 7.2 수정

```
1. 변경 필드 검증
2. SSO 사용자의 SSO-동기화 필드 변경 시도 차단
3. 권한 변경 시:
   - CEO/GM 권한 검증
   - 본인 권한 변경 차단 ("본인 권한은 변경할 수 없습니다")
4. users UPDATE
5. 권한 변경 있으면 audit_logs INSERT (action='PERMISSION_CHANGE',
   before_data, after_data)
6. 본인에게 새올 포틀릿 알림 (권한 변경 시)
7. 모달 닫기 + ORG02-D 또는 ORG02-L 새로고침
```

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| 이메일 | 형식 + 시스템 내 UNIQUE (다른 사용자와 중복 불가) |
| 부서 | 권한 범위 내만 선택 가능 |
| 권한 코드 | CEO/GM만 변경 가능. 본인 권한 변경 차단 |
| SHM의 권한 부여 범위 | SM/SHM/WKR까지만. CEO/GM 부여 불가 (CEO/GM이 직접 처리) |
| SSO 사용자 부서 변경 시도 | 차단 ("SSO 사용자는 행정포털에서 부서 변경") |

## 9. 오류 처리

| 케이스 | 메시지 | 처리 |

| 이메일 중복 | "이미 등록된 이메일입니다" | 인라인 에러 |

| 저장 실패 (트랜잭션) | "저장 중 오류가 발생했습니다" | 토스트 + 모달 유지 |
