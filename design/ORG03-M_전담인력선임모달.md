## 4. 입력 항목

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 대상 부서 | READONLY | — | 진입 시 prefill |
| 대상 역할 | READONLY | — | 진입 시 prefill (SAFETY_MGR / HEALTH_MGR / HEALTH_DOCTOR) |
| 선임 대상자 | USER_SEARCH | ✅ | 구성원 검색 모달 (부서 필터 + 자격증 표시) |
| 선임일 | DATE | ✅ | 기본값: 오늘 |
| 선임 근거 첨부 | FILE | ❌ | 선임공문 PDF/이미지 → `appointment_doc_url` |
| 비고 | TEXTAREA | ❌ | 최대 500자 |

## 5. 자격증 보유 검증 (선임 대상자 선택 시)

```
선임 대상자 선택 시 즉시 자격증 검증:
  user_qualifications 조회 → 역할별 필수 자격증 매칭
    - SAFETY_MGR: 산업안전기사·산업안전산업기사·전문대학 이상 산업안전공학과 졸업·실무 경력 등
    - HEALTH_MGR: 산업위생기사·간호사·실무 경력 등
    - HEALTH_DOCTOR: 의사 면허

  결과 인라인 배지 표시:
    ✅ 자격증 보유 (자격증명 표시)
    ⚠ 자격증 미보유 (강제 선임 시 경고 모달 노출)
```

> 자격증 마스터 (역할별 필수 자격증) 초기값은 발주처 협의 후 확정.

## 7. 저장 동작

### 7.1 자격 보유 케이스

```
1. dedicated_personnel UPSERT (
     department_id, role_type, user_id, appointed_at, appointment_doc_url)
2. 본인에게 새올 포틀릿 알림 (category='assignment',
   "{부서명} {역할명}로 선임되었습니다")
3. audit_logs INSERT
4. ORG03-V 갱신 → 모달 닫기
```

### 7.2 자격 미보유 케이스 (강제 선임)

```
경고 모달 표시:
   "{사용자명}은(는) 자격증을 보유하지 않았습니다.
    강제 선임하시겠습니까?
    (강제 선임 이력이 기록됩니다)
    
    [취소]  [강제 선임]"

[강제 선임] 클릭 시:
  1. dedicated_personnel UPSERT
  2. warnings INSERT (warning_type='unqualified_appointment',
     user_id, role_type, metadata={appointer, date})
  3. audit_logs INSERT
  4. 본인에게 새올 포틀릿 알림
  5. 운영자(CEO/GM)에게 새올 포틀릿 알림 (category='system', 강제 선임 이력)
  6. ORG03-V 갱신 → 모달 닫기
```

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| 선임 대상자 | 비활성 사용자 차단 |
| 동일 사용자 중복 선임 | 이미 다른 부서에 선임된 경우 확인 모달 ("이미 다른 부서에 선임된 사용자입니다. 진행하시겠습니까?") |
| 선임일 | 미래 날짜 허용 (예정 선임) |
| 권한 범위 | 본 부서 또는 권한 범위 내 사용자만 선임 가능 |

## 9. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 자격 미보유 + 강제 선임 시 | 경고 모달 (§7.2) | [취소] / [강제 선임] |
| 동일 사용자 중복 선임 시도 | "이미 다른 부서에 선임된 사용자입니다" | 확인 모달 |
| 비활성 사용자 선임 시도 | "비활성 사용자는 선임할 수 없습니다" | 차단 |
| 저장 실패 | "선임 처리 중 오류가 발생했습니다" | 토스트 + 모달 유지 |
