## 4. 표시 항목

### 4.1 연동 설정 카드

| 항목 | 데이터 소스 |
|------|------------|
| 연동 상태 | 마지막 sync_logs.result 기반 ([연결됨]/[연결 안 됨]) |
| 마지막 성공 시각 | `MAX(sync_logs.finished_at WHERE result='SUCCESS' AND sync_type='PORTAL_ORG')` |
| 동기화 주기 | 환경 설정 (기본 매일 06:00) |
| 동기화 범위 | 환경 설정 |
| API 엔드포인트 | 환경 설정 (`{portal}/api/v1/...`) |

### 4.2 동기화 이력 목록

| 컬럼 | 데이터 소스 |
|------|------------|
| 일시 | `sync_logs.started_at ~ finished_at` |
| 결과 | `sync_logs.result` 배지 (SUCCESS/FAILED/IN_PROGRESS) |
| 부서 | `dept_added + ~dept_updated` |
| 사용자 | `user_added + ~user_updated` |
| 부서장 | `~head_changed` (변경 수) |
| 로그 | `[보기]` 버튼 → sync_logs 상세 모달 |

### 4.3 부서장 변경 트리거 결과

`sync_logs.log_detail` JSON에 부서장 변경 시 자동 매핑 트리거 결과 포함:

```json
{
  "head_changes": [
    {
      "department_id": "uuid",
      "department_name": "자치행정과",
      "old_head_user_id": "uuid-A",
      "new_head_user_id": "uuid-B",
      "affected_targets_count": 3,
      "auto_responsible_updated": 3,
      "manual_responsible_kept": 0
    }
  ]
}
```

## 6. 자동 처리 로직

### 6.1 수동 동기화 흐름

```
[전체 동기화] 클릭 → 확인 모달:
  "행정포털 조직도를 즉시 동기화하시겠습니까?
   예상 소요 시간: 약 N분"

확인 시:
  1. sync_logs INSERT (result='IN_PROGRESS', sync_type='PORTAL_ORG')
  2. IF-002 행정포털 조직도 API 호출 (changed_since 또는 전체)
  3. departments UPSERT
  4. users UPSERT
  5. 부서장 변경 감지 시 trg_departments_head_change 트리거 발동:
     - target_assignees (role='RESPONSIBLE', source='AUTO') 자동 갱신
     - 새올 포틀릿 알림 (대상자: 신규 RESPONSIBLE)
  6. sync_logs UPDATE (result='SUCCESS' or 'FAILED', 카운트·log_detail)
  7. 결과 모달 표시 → 진행률 100%
  8. ORG01-V 트리 새로고침
```

### 6.2 정기 자동 동기화

```
스케줄러: 매일 06:00 (환경 설정 변경 가능)
   → §6.1과 동일한 동기화 흐름
   → 동기화 실패 시 운영자(CEO/GM/SHM)에게 새올 포틀릿 알림 (category='system')
```

### 6.3 [수정] (연동 설정)

```
수정 모달:
  - 동기화 주기: 매일 N:NN
  - API 엔드포인트: URL
  - 인증 토큰: (마스킹 표시, 변경 가능)

저장 시:
  환경 설정 UPDATE
  변경 이력 audit_logs INSERT
```

### 6.4 [테스트 호출]

```
1. IF-002 API 단순 호출 (예: GET /api/v1/departments?limit=1)
2. 응답 결과 표시:
   - HTTP 상태 코드
   - 응답 시간 (ms)
   - 응답 본문 일부 (미리보기)
3. 실패 시 오류 메시지 표시 (IF-002 §2.6 협의 사항 참조)
```

## 7. 검증·오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|

| API 응답 없음 | "행정포털 연결 실패: {오류 메시지}" | 토스트 + sync_logs FAILED |
