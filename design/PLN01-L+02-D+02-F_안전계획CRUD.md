| 컬럼 | 데이터 소스 |

### B.5 CMP 자동 연계

```
plan_items.cmp_schedule_id가 NULL이 아니면:
  CMP(이행관리 — Phase 3)의 compliance_schedules와 연동
  → plan_items.status='DONE' 시 compliance_executions 자동 INSERT
  → 또는 역방향: compliance_executions 완료 시 plan_items.status='DONE' 자동
```

### B.6 결재 흐름 (온나라 안내 팝업)

RSK02-D §7.1 동일 패턴:
- `[결재요청]` → 안내 팝업 → `approval_status='REQUESTED'` + 새올 알림
- `[결재 결과 업데이트]` → 결재 결과 수동 입력 → `approval_status='APPROVED'/REJECTED'`

### B.7 종료 처리

```
[완료 처리] 활성 조건:
  - plan_items 모두 `status IN ('DONE', 'SKIPPED')`
  - 최소 1건 이상 'DONE'

전환 시:
  safety_plans.status='COMPLETED'
  audit_logs INSERT
  새올 알림 (담당자·부서장)
```

## C. PLN02-F 안전계획 등록·수정

### C.1 화면 정보

| 항목 | 내용 |
|------|------|
| 화면 ID | PLN02-F |
| URL | `/safety-plans/new` (등록) / `/safety-plans/:id/edit` (수정) |
| 화면 유형 | F (Form, 2-Step) |
| 접근 권한 | SHM+ |

### C.2 단계 흐름

```
STEP 1. 기본정보
        - 관리대상 / 연도 / 계획명 / 담당자
STEP 2. 계획 항목 입력
        - 템플릿 적용 또는 직접 입력
        - 카테고리별 항목 등록 (교육/점검/평가/예산/기타)
```

### C.3 STEP 1 입력 항목

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 관리대상 | SELECT (targets) | ✅ | 권한 범위 내 |
| 연도 | NUMBER | ✅ | 기본: 현재 연도. UNIQUE (target+year) |
| 계획명 | TEXT | ✅ | 기본 제안: "{연도} {관리대상명} 안전관리계획" |
| 담당자 | USER_SEARCH | ✅ | 기본: 현재 사용자 |
| 비고 | TEXTAREA | ❌ | |

### C.4 STEP 2 — 계획 항목 입력

```
[템플릿 적용]: PLN04-S에서 정의한 표준 템플릿 일괄 추가
[+ 항목 추가]: 카테고리·제목·마감일·담당자 입력 모달
```

각 항목 인라인 편집 (제목·카테고리·마감월·담당자·상태).

### C.5 저장 동작

```
신규:
  1. safety_plans INSERT (status='DRAFT')
  2. plan_items INSERT (배열)
  3. audit_logs INSERT
  4. PLN02-D 이동

수정:
  1. safety_plans UPDATE
  2. plan_items 차분 처리 (추가·수정·삭제)
  3. PLN02-D 이동
```

### C.6 자동 처리

- CMP 모듈(Phase 3) 연동 시 `plan_items.cmp_schedule_id` 자동 매핑 (정기 점검·평가 등)
- 계획 수립 완료(STEP 2 통과) 시 `status='ESTABLISHED'` 자동 전환

### C.7 검증

| 항목 | 검증 |
|------|------|
| target_id + year | UNIQUE (한 관리대상에 한 연도 1건만) |
| 항목 수 | 최소 1건 |
| 항목 마감일 | 연도 범위 내 |
