### B.4 결재 흐름 (온나라 안내 팝업)

RSK02-D §7.1 동일 패턴:
- `[결재요청]` → 안내 팝업 → `approval_status='REQUESTED'` + 새올 알림 (대상자: 군수/CEO)
- `[결재 결과 업데이트]` → 결과 입력 모달 → `APPROVED` 시 `is_current=true` 자동 전환 옵션 (CEO 선택)

## C. POL02-F 방침 등록·수정

### C.1 화면 정보

| 항목 | 내용 |
|------|------|
| 화면 ID | POL02-F |
| URL | `/policies/new` (등록) / `/policies/:id/edit` (수정) |
| 화면 유형 | F (Form, 풀페이지) |
| 접근 권한 | CEO / GM / SHM |

### C.2 입력 항목

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 버전 | TEXT | ✅ | UNIQUE. v{n.m} 형식 권장 |
| 제목 | TEXT | ✅ | |
| 시행일 | DATE | ✅ | 미래 일자 가능 |
| 방침 본문 | HTML 에디터 | ✅ | XSS 검증 (sanitize) |
| 근거 법령 매칭 | 다중 입력 | ❌ | 법령 마스터에서 선택 + 매칭 조항 입력 |
| KPI 정의 | 다중 입력 | ❌ | 항목·목표·산출공식 (policy_kpis INSERT) |
| 첨부 | FILE (multi) | ❌ | 초안·의견수렴 결과 등 |
| 작성자 | 자동 | — | 현재 사용자 |

### C.3 KPI 입력 모달

```
KPI 추가 모달:
  - KPI명 (자주 쓰는 항목 자동완성)
  - 목표값
  - 단위
  - 산출 공식 (텍스트 — 시스템 자동 측정 어려운 KPI도 정성 입력 가능)
  - 정렬 순서
```

### C.4 근거 법령 매칭 입력

```
법령 마스터 (CMP 모듈과 공유 - Phase 3 또는 별도 마스터):
  - "중대재해처벌법 시행령 제4조"
  - "산업안전보건법 제5조"
  - ...

본 화면에서:
  법령 SELECT (자동완성) + 방침 매칭 조항 입력 (예: "§1, §2")
  복수 추가 가능
```

### C.5 저장 동작

```
신규:
  1. policies INSERT (is_current=false, approval_status='NOT_REQUESTED')
  2. policy_kpis 일괄 INSERT
  3. audit_logs INSERT
  4. POL02-D 이동

수정:
  1. policies UPDATE (이미 결재 완료된 방침은 수정 제한 — 새 버전 권장)
  2. policy_kpis 차분 처리
  3. POL02-D 이동
```

### C.6 검증

| 항목 | 검증 |
|------|------|
| 버전 | UNIQUE. v{숫자.숫자} 형식 권장 |
| 본문 | HTML XSS sanitize |
| 결재 완료 후 수정 | 차단 ("결재 완료된 방침은 신규 버전으로 등록하세요") |

```

```
