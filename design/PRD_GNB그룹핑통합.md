> 선행 PRD: PRD-DAMYANG-001 (조직·인력 모듈 통합)

## 0. 작업 개요

### 0.1 목적

현재 IA v1.3 기준 GNB 18개로 분산되어 있는 메뉴 구조를 **업무 성격별 8개 GNB 그룹**으로 통합한다. 화면·DB·SFR 구조는 변동 없이 GNB 레벨에서만 그룹핑.

### 0.2 작업 범위

| 분류 | 작업 |
|------|------|
| IA v1.4 작성 | GNB 그룹 8개 + LNB 모듈 구조 명시 |
| RTM v1.3 작성 | GNB 그룹 매핑 추가 (SFR·화면 매핑은 그대로) |
| 화면설계서 갱신 | 각 화면의 "진입 경로" 부분 GNB 명칭 갱신 |
| 모듈 명칭 정리 | "전담조직" → "조직", "안전관리자 인력" → "안전보건 인력" |

### 0.3 변경하지 않는 것

- 화면 수: 90개 유지
- 화면 ID: 변경 없음
- URL 패턴: 변경 없음 (v1.1 LNB 패턴 유지)
- DB 스키마: 변경 없음
- SFR 매핑: 변경 없음
- 결정사항: 변경 없음

### 0.4 선행 PRD와의 관계

`PRD-DAMYANG-001 (조직·인력 모듈 통합)`은 ORG03-V → STF04-V 이동을 다룸. 본 PRD는 **그 결과를 전제**로, GNB 레벨 그룹핑을 진행한다. 두 PRD는 함께 적용되어야 하며, 적용 후 산출물은 IA v1.4 / RTM v1.3으로 통합.

## 1. 변경 사항 — GNB 8개 그룹 구조

### 1.1 GNB 순서 (사용 빈도 기준)

```
홈 (대시보드 — 권한별 자동 라우팅)

GNB 1. 위험성평가          ← 가장 자주 진입 (자동 생성·정기 평가)
GNB 2. 점검·진단           ← 일상 점검 활동
GNB 3. 도급 관리           ← 계약 진행·평가·서약 흐름
GNB 4. 종사자 참여         ← 의견·산안위
GNB 5. 이행·증명           ← 사후 추적·집계·증명
GNB 6. 안전보건 경영       ← 연간 사이클 (빈도 낮음)
GNB 7. 기반 관리           ← 초기 셋팅·가끔 업데이트
GNB 8. 시스템 관리         ← 운영자(CEO/GM/SHM)만
```

### 1.2 GNB별 LNB 구조

#### GNB 1. 위험성평가 (RSK 단독)
```
├─ 현황 (RSK01-V)
├─ 평가 목록 (RSK02-L)
├─ 개선조치 (RSK03-L)
└─ 설정 (RSK04-S, 콘텐츠 마스터)
```
- 단독 GNB 유지 이유: 중대재해처벌법 시행령 제4조 핵심 의무 / 다른 모듈에 자동 트리거 발생 / 사용자 진입 빈도 최고

#### GNB 2. 점검·진단
```
├─ 안전점검 (INS)
│   ├─ 점검 현황 (INS01-V)
│   ├─ 점검 목록 (INS02-L)
│   └─ 설정 (INS03-S)
└─ 업무문서 (DOC)              ← 명칭 "업무문서" 유지
    ├─ 매뉴얼 (DOC01-L)
    └─ 점검표 (DOC02-L)
```

#### GNB 3. 도급 관리 (CON 단독)
```
├─ 도급계약 (CON01-L)
├─ 수급업체 (CON02-L)
├─ 협의체 (CON03-L)
├─ TBM (CON09-L)               ← 도급공사 계약 존재 시만 노출
└─ 설정 (CON10-S, 평가서·서약서 마스터)
```
- 단독 GNB 유지 이유: 외부 사용자(SUB) 협업 환경 / 4-Step wizard·e호조 연동 등 자체 완결적 워크플로우

#### GNB 4. 종사자 참여 (OPN 단독)
```
├─ 현황 (OPN01-V)
├─ 의견 목록 (OPN01-L)
├─ 산업안전보건위원회 (OPN03-L)  ← 별도 메뉴 4단계 (결정사항 #20)
└─ 설정 (OPN04-S)
```

#### GNB 5. 이행·증명
```
├─ 개선조치 (IMP)
│   ├─ 현황 (IMP01-V)
│   ├─ 전체 목록 (IMP01-L)
│   └─ 재발방지대책 (IMP01-L `/incident_prevention`)
├─ 이행 관리 (CMP)
│   ├─ 이행 현황 (CMP01-V)
│   ├─ 이행 목록 (CMP02-L)
│   └─ 법령 마스터 (CMP03-S)
├─ 통계 (STA)                    ← 그대로 유지
│   ├─ 종합 통계 (STA01-V)
│   ├─ 모듈별 통계 (STA02-V)
│   └─ 설정 (STA03-S)
└─ 제증명 (CRT)
    ├─ 발급 이력 (CRT01-V)
    ├─ 증명서 발급 (CRT02-F)
    └─ 설정 (CRT03-S)
```

#### GNB 6. 안전보건 경영
```
├─ 안전보건 방침 (POL)
│   ├─ 방침 관리 (POL01-L)
│   ├─ 목표·KPI (POL03-V)
│   └─ 경영방침 점검 (POL04-V)
├─ 안전보건 계획 (PLN)
│   ├─ 현황 (PLN01-V)
│   ├─ 계획 목록 (PLN01-L)
│   ├─ 캘린더 (PLN03-V)
│   └─ 설정 (PLN04-S)
├─ 안전보건 예산 (BGT)
│   ├─ 예산 현황 (BGT01-V)
│   ├─ 예방 항목 트리 (BGT02-V)
│   └─ 설정 (BGT05-S)
└─ 인력 평가 (EVL)              ← 안전보건 경영에 배치 (결정)
    ├─ 현황 (EVL01-V)
    ├─ 평가 목록 (EVL01-L)
    └─ 설정 (EVL03-S)
```

#### GNB 7. 기반 관리
```
├─ 관리대상 (TGT)
│   └─ 관리대상 현황 (TGT01-V)
│       └─ (화면 내부 서브 탭) 전체/시설/업무/사업 — `?type=`
├─ 조직 (ORG)                   ← 명칭 변경: "전담조직" → "조직"
│   ├─ 조직도 (ORG01-V)
│   └─ 구성원 (ORG02-L)
└─ 안전보건 인력 (STF)          ← 명칭 변경: "안전관리자 인력" → "안전보건 인력"
    ├─ 전체 현황 (STF01-V)
    ├─ 부서별 전담인력 (STF04-V) ← 선행 PRD 결과
    ├─ 공사별 선임 현황 (STF02-V)
    └─ 설정 (STF03-S)
```

#### GNB 8. 시스템 관리 (CEO/GM/SHM 권한)
```
├─ 외부 시스템 연동 (SYS01-S)
├─ 외부 사용자 관리 (AUTH06-S)
├─ 알림 운영 (NTF03-S)
└─ 감사 로그
```

# 도급/종사자/위험성평가는 단독 GNB이므로 명칭만 정리 (기존 그대로일 수 있음)
sed -i 's|GNB > 도급관리|GNB > 도급 관리|g' *.md
sed -i 's|GNB > 의견청취|GNB > 종사자 참여|g' *.md

sed -i 's|시스템 관리 > 외부 시스템 연동|시스템 관리 > 외부 시스템 연동|g' *.md
# 이미 정확하므로 변경 없음

> ```bash
> grep -rn "GNB > 전담조직" *.md
> grep -rn "GNB > 안전관리자 인력" *.md
> # ... 각 패턴별로 영향 파일 사전 확인
> ```

### 6.2 주의 사항

1. **선행 PRD-001 적용 필수**: ORG03 → STF04 이전이 먼저 완료되어야 본 PRD GNB 구조의 STF가 4화면으로 정착함. 두 PRD를 함께 적용하는 게 정합성 측면에서 권장.

2. **sed 적용 시 영향 범위 사전 확인**: 본문에서 "GNB > X" 외 다른 형태로 GNB 언급이 있으면 자동 치환에서 누락. grep으로 패턴 다양성 확인.

3. **변경 이력 history 보존**: 화면설계서 변경 이력 섹션에 과거 "전담조직" 같은 기록이 있으면 그대로 둠. 자동 치환 후 grep으로 검증.

4. **모듈 그룹핑 변경 없음**: 화면 ID, URL, DB 스키마, SFR, 결정사항은 일체 변경하지 않음. 본 PRD는 **순수 IA 레벨 작업**.

5. **GNB 권한**: 권한 매트릭스 §3.2 재구성 시 기존 모듈 권한을 8개 GNB로 합산. CEO/GM/SM/SHM/WKR/SUB/CON 각 권한별로 GNB 단위 접근 결정.
