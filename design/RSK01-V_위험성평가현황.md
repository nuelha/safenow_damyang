## 4. 표시 항목

### 4.1 기준연도 선택기

| 항목 | 내용 |
|------|------|
| 위치 | 탭 바로 아래 단독 영역 |
| 선택지 | 2026년 / 2025년 / 2024년 / 전체 |
| 기본값 | 현재 연도 |
| 판정 기준 | `risk_assessments.start_date` 의 연도 |
| 영향 범위 | 요약 카드 + 목록 + 위험등급별 (개선조치 위젯은 영향 없음 — 별도 마감일 기준) |
| URL 동기화 | `?year=` |

### 4.2 요약 카드 N/n 표기

| 카드 | active (좌측) | done (우측) | 색상 |
|------|--------------|-------------|------|
| 전체 | `cycle_stage IN ('PREPARING','IN_PROGRESS')` | `cycle_stage='DONE'` | gray / green |
| 정기 | 위 + `assessment_type='regular'` | 위 + 정기 | blue / green |
| 수시 | 위 + `assessment_type='special'` | 위 + 수시 | purple / green |
| 완료율 | (단일) `done / total × 100` % | — | green |

> 최초(`initial`)는 전체 카드에만 합산, 별도 카드 없음.

### 4.3 정기 평가 누락 안내 배너

```
검출 로직:
  WHERE NOT EXISTS (active 정기 평가)
    AND (last_done_date + interval) < NOW()
  COUNT 산출 → 1건 이상이면 배너 노출

[확인 →] 클릭: RSK02-L 이동 (?filter=overdue)
```

### 4.4 위험성평가 현황 목록

| 컬럼 | 데이터 소스 | 비고 |
|------|------------|------|
| 평가명 | `risk_assessments.name` | |
| 관리대상 | JOIN `targets.name` + type 배지 | |
| 유형 | `assessment_type` 배지 (정기/최초/수시) | |
| 진행상태 | `cycle_stage` 배지 (사전준비/진행중/완료) | |
| 담당자 | JOIN `users.name` | |
| 기간 | `start_date ~ end_date` | |
| 시행확인 라벨 | `confirmation_label` ("최초"/"수시") | 정기는 미표시 |
| 상세 | → RSK02-D 이동 버튼 | |

### 4.5 개선조치 현황 위젯

| 항목 | 데이터 소스 |
|------|------------|
| 기한초과 | `COUNT(improvements WHERE source_type='risk_assessment' AND due_date < NOW() AND status != 'DONE')` |
| 조치중 | `COUNT WHERE status='IN_PROGRESS'` |
| 예정 | `COUNT WHERE status='PENDING' AND due_date >= NOW()` |

### 4.6 위험등급별 현황

| 등급 | 위험성 점수 (frequency × severity) |
|------|--------------------------------|
| 매우높음 | 16~25 |
| 높음 | 12~15 |
| 보통 | 8~11 |
| 낮음 | 4~7 |
| 매우낮음 | 1~3 |

> 등급 기준은 평가 기법(method)에 따라 다를 수 있음 — 마스터에서 조정 가능.

## 7. 자동 처리 로직

본 화면 자체에서 트리거되는 자동 처리는 없음. 다만 다음 자동 작업의 결과가 본 화면에 반영됨:

- **정기 평가 자동 생성 배치 (00:00 일일)**: DB-001 §9.4 참조
- **차세대 e호조 도급사업 자동 import**: IF-004 §4.6 — 신규 도급공사 → 위험성평가 자동 생성 (`source='EHOJO_AUTO'`)
- **위험요인 불허 → 개선조치 자동 INSERT**: DB-001 §9.3 — 개선조치 현황 위젯에 반영
- **부서장 변경 → RESPONSIBLE 자동 매핑**: DB-001 §9.1 — 평가 담당자 변경 가능

| 필터 조합 결과 없음 | "조건에 맞는 평가가 없습니다" | 빈 목록 + 필터 초기화 |

## 9. 관련 화면 흐름

```
RSK01-V
   ├─ LNB → RSK02-L / RSK04-S (좌측 사이드 메뉴 이동)
   ├─ 개선조치 위젯 [전체보기] → IMP01-L?source=risk_assessment
   ├─ [+ 등록] → RSK02-F
   ├─ 행 [상세] → RSK02-D
   ├─ 누락 배너 [확인] → RSK02-L?filter=overdue
   └─ 개선조치 카드 → IMP01-L?source=risk_assessment
```

## 10. 관련 DB 테이블

- `risk_assessments` (DB-001 §4.1): 평가 목록·요약·통계
- `risk_factors` (DB-001 §4.2): 위험등급별 카운트
- `targets` (DB-001 §2.3): JOIN 관리대상명·type
- `improvements` (DB-001 §7 — Phase 2): 개선조치 현황 위젯

## 11. 관련 인터페이스

- **IF-004 차세대 e호조**: 도급사업 자동 import → 위험성평가 자동 생성 (간접 — RSK04-S에서 동기화 수행)

## 12. 성능 고려

- 요약 카드 N/n 카운트는 캐싱 (5분)
- 위험등급별 카운트는 캐싱 (10분)
- 개선조치 위젯은 실시간 (마감일 기준 D-day 정렬)
