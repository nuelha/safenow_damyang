## 7. 저장 동작

### 7.1 신규 등록

```
1. 필수 필드 검증
2. risk_assessments INSERT
   - cycle_stage='PREPARING'
   - approval_status='NOT_REQUESTED'
   - source='MANUAL'
   - confirmation_label:
     • assessment_type='initial' → "최초"
     • assessment_type='special' → "수시"
     • assessment_type='regular' → NULL
3. participants JSON 저장
4. 담당자가 본인이 아닌 경우: 새올 포틀릿 알림 (대상자: 담당자, category='assignment',
   "{관리대상명} 위험성평가 담당자로 지정되었습니다")
5. audit_logs INSERT
6. RSK02-D 이동 (위험요인 등록 단계 진행)
```

### 7.2 수정

```
1. 검증 → risk_assessments UPDATE
2. 담당자 변경 시 새 담당자에게 알림
3. cycle_stage='DONE' 평가는 수정 불가 (RSK02-D에서 [수정] 버튼이 비활성)
4. RSK02-D 이동
```

## 8. 자동 처리 로직

### 8.1 시행확인 라벨 자동 산출

```
평가유형 RADIO 변경 시 클라이언트에서 자동 산출:
  initial → confirmation_label = "최초"
  special → confirmation_label = "수시"
  regular → confirmation_label = NULL

저장 시 함께 INSERT/UPDATE
```

### 8.2 수시 트리거 필드 동적 노출

```
평가유형 RADIO 변경 감지:
  - special 선택 → 트리거 유형·트리거 설명 필드 노출 + 필수 적용
  - 다른 유형 선택 → 필드 숨김 + 값 초기화
```

### 8.3 관리대상 prefill (TGT02-D에서 진입 시)

```
URL `?target_id=:id` 진입:
  - 관리대상 selector 해당 target으로 prefill + 잠금 옵션 (선택)
  - 평가명 자동 제안 (예: "{관리대상명} 위험성평가 {YEAR}")
```

## 9. 검증 규칙

| 항목 | 검증 |
|------|------|
| 평가명 | 1~200자 |
| 관리대상 | 권한 범위 내만 선택 가능 (자동 필터링) |
| 시행 기간 | 종료일 ≥ 시작일 |
| 트리거 필드 | 평가유형='special' 시 필수 |
| 담당자 | 비활성 사용자 선택 차단 |

## 10. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 필수 필드 누락 | "{필드명}을(를) 입력해주세요" | 인라인 에러 |
| 종료일 < 시작일 | "종료일은 시작일 이후여야 합니다" | 인라인 에러 |
| 권한 범위 외 관리대상 선택 | 자동으로 옵션에서 제외 (필터링) | — |
| 완료된 평가 수정 시도 | "완료된 평가는 수정할 수 없습니다" | 진입 차단 + RSK02-D 이동 |
| 저장 실패 | "저장 중 오류가 발생했습니다" | 토스트 + 폼 유지 |
