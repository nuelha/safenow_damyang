
### 4.2 데이터 소스

`risk_assessment_intervals` 테이블 (target_type, cdpa_category, interval_months) — DB-001 §4.3

### 4.3 [수정] 모달

| 필드 | 타입 | 필수 | 비고 |

| 주기 (개월) | NUMBER | ✅ | 1~36 |

저장 시 `interval_months` UPDATE + audit_logs INSERT. 다음 정기 평가 자동 생성 배치(DB-001 §9.4)에 즉시 반영.

### 5.2 입력 항목 (등록·수정 모달)

| 필드 | 타입 | 필수 | 비고 |

| 표시명 | TEXT | ✅ | 1~50자 |
| 설명 | TEXTAREA | ❌ | |

저장 시 평가 기법 마스터 INSERT/UPDATE. 활성 기법만 RSK02-F 평가 기법 selector에 노출.

## 6. 위험요인 마스터 탭

### 6.1 레이아웃

자주 사용되는 위험요인 사전 등록 (관리대상 type 별, 업종별 분류). 평가 등록 시 빠르게 선택 가능.

```

│ 위험요인 마스터                             │
│ 필터: [type▼][분류▼][검색]                  │
│                       [+ 추가] [일괄 등록]  │
│                                            │
│ 코드   │ 위험요인명          │ type   │분류 │ 관리│
│ ───────┼────────────────────┼────────┼────┼────│
│ RF-001 │ 전기 누전·감전     │ FACILITY│전기│[수정][삭제]│
│ RF-002 │ 추락 위험          │ FACILITY│공통│[수정][삭제]│
│ RF-003 │ 화학물질 누출      │ FACILITY│화학│[수정][삭제]│
│ ...                                        │
└────────────────────────────────────────────┘
```

### 6.2 일괄 등록

ORG02-M2 패턴 차용 (4-Step: 양식 다운로드 → 업로드 → 검증 → 등록). 양식 컬럼: 코드·위험요인명·type·분류·설명.

## 7. 차세대 e호조 연동 설정 탭

### 7.1 레이아웃

```
┌────────────────────────────────────────────────┐
│ 차세대 e호조 연동                                │
│                                                │
│ 연동 상태: [연결됨] 마지막 동기화 2026-05-11 06:00│
│ 동기화 주기: 매일 06:00                          │
│ API 엔드포인트: https://ehojo.gov.kr/api/...    │
│                                  [수정][테스트]  │
│                                                │
│ 동기화 항목                                     │
│ ● 도급사업 목록 (계약명·기간·금액·발주처)       │
│ ● 도급사업별 위험성평가 자동 생성 여부 (☑ ON)   │
│                                                │
│ 동기화 이력 (최근 30건)                          │
│ 일시        │ 결과 │ 신규 │ 변경 │ 자동 생성 평가│
│ 2026-05-11  │ 성공 │ +3   │ ~1   │ +2 (공사 2건)│
│ ...                                            │
│                                                │
│ 수동 동기화                                      │
│ [전체 동기화]                                    │
└────────────────────────────────────────────────┘
```

### 7.2 자동 처리 (IF-004 §4 참조)

```
정기 자동 동기화 (매일 06:00):
  1. IF-004 차세대 e호조 API 호출 (changed_since)
  2. 신규 계약 contracts INSERT (source='EHOJO_AUTO')
  3. 도급공사(CONSTRUCTION) 인 경우 risk_assessments 자동 생성 (source='EHOJO_AUTO')
  4. 알림 발송 (담당자 지정 필요 — 새올 포틀릿)
  5. sync_logs INSERT (sync_type='EHOJO')

수동 [전체 동기화]:
  동일 흐름 즉시 실행
```

### 7.3 [수정] 모달

| 필드 | 타입 | 비고 |
|------|------|------|
| API 엔드포인트 | URL | |
| 동기화 주기 | TIME | HH:MM |
| 인증 토큰 | TEXT (마스킹) | |
| 도급공사 위험성평가 자동 생성 | BOOLEAN | OFF 시 contracts만 INSERT, risk_assessments 생성 안 함 |

## 9. 검증·오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 평가 주기 변경 권한 부족 | "권한이 없습니다" | 차단 |
| 위험요인 마스터 삭제 시도 (사용 중) | "사용 중인 위험요인은 삭제할 수 없습니다. 비활성화 처리하세요" | 인라인 안내 |
| 차세대 e호조 API 응답 없음 | "차세대 e호조 연결 실패" | 토스트 + sync_logs FAILED |
| 동기화 진행 중 재클릭 | "동기화가 이미 진행 중입니다" | 토스트 |

## 10. 관련 화면 흐름

```
RSK04-S
   ├─ 화면 내부 서브 탭 → 평가 주기 / 평가 기법 / 위험요인 / e호조 연동
   ├─ [수정] → 각 설정 수정 모달
   ├─ [+ 추가] / [일괄 등록] → 등록 모달
   ├─ [테스트] → API 테스트 결과 모달
   ├─ [전체 동기화] → 비동기 진행 → 결과 모달
   ├─ 이력 [보기] → sync_logs 상세 모달
   └─ LNB → RSK01-V / RSK02-L (좌측 사이드 메뉴 이동)
```

## 11. 관련 DB 테이블

- `risk_assessment_intervals` (DB-001 §4.3): 정기 평가 주기
- `risk_assessment_methods` (Phase 2 마스터): 평가 기법
- `risk_factor_templates` (Phase 2 마스터): 위험요인 마스터
- `sync_logs` (DB-001 §3.1): 차세대 e호조 동기화 이력
- `contracts` (DB-001 §5.1): 자동 INSERT 대상
- `risk_assessments` (DB-001 §4.1): 도급공사 자동 생성 평가
- `audit_logs` (DB-001 §2.12): 설정 변경 감사

## 12. 관련 인터페이스

- **IF-004 차세대 e호조**: 본 화면의 핵심 연동 대상
- IF-005 새올 포틀릿 (간접): 동기화 실패·자동 생성 평가 알림
