#### C.2.1 통계 기간 마스터

```
표준 기간 옵션 정의:
  - 일간 / 주간 / 월간 / 분기 / 반기 / 연간 / 전기간
  - 회계연도 시작월 설정 (기본 1월)
  - 사용자 정의 기간 (시작/종료일 직접 입력)
```

#### C.2.2 KPI 가중치 설정

```
부서별 종합 점수 산출 시 KPI별 가중치:
  - 의무이행 (가중치 25%)
  - 위험성평가 (20%)
  - 안전점검 (20%)
  - 도급사업 (15%)
  - 사고 (20%)
  - 합계 = 100%

CEO만 편집 가능.
```

#### C.2.3 보고서 템플릿

```
정기 보고서 자동 생성 옵션:
  - 월간 안전관리 보고서 (매월 1일 자동 생성)
  - 분기 종합 보고서 (분기 시작일 자동)
  - 연간 결산 보고서 (다음 연도 1월)
  
생성 시 CEO·GM 새올 알림 + PDF 다운로드 링크
```

## 공통 — 자동 처리

### 정기 보고서 자동 생성 (배치)

```
스케줄: 매월 1일 06:00 (월간), 분기 시작일 (분기), 다음 연도 1월 (연간)

처리:
  1. 전 기간 통계 집계
  2. PDF 보고서 생성 (시각화 + 텍스트 요약)
  3. statistics_reports INSERT
  4. CEO·GM에게 새올 알림 ("정기 보고서 생성 완료")
```

## 공통 — 관련 DB 테이블

> STA 모듈은 자체 테이블보다 다른 모듈 테이블의 집계 view 활용.

```sql
-- 통계 보고서 이력 (선택)
CREATE TABLE statistics_reports (
  id              UUID PRIMARY KEY,
  report_type     ENUM('MONTHLY','QUARTERLY','ANNUAL','CUSTOM') NOT NULL,
  period_start    DATE NOT NULL,
  period_end      DATE NOT NULL,
  generated_by    UUID REFERENCES users(id),
  pdf_url         VARCHAR(500),
  xlsx_url        VARCHAR(500),
  metadata        JSON,                                  -- 집계 결과 JSON 보관 (조회 시 즉시 표시)
  created_at      TIMESTAMP DEFAULT NOW()
);

-- 집계 view 예시 (성능)
CREATE MATERIALIZED VIEW v_compliance_summary AS
SELECT
  YEAR(execution_date) as year,
  MONTH(execution_date) as month,
  c.target_department_id,
  c.category,
  COUNT(*) FILTER (WHERE e.status='COMPLETED') as completed_count,
  COUNT(*) as total_count
FROM compliance_schedules c
LEFT JOIN compliance_executions e ON c.id = e.schedule_id
GROUP BY year, month, c.target_department_id, c.category;

-- 일배치로 REFRESH
```

## 공통 — 관련 인터페이스

## 공통 — 성능 고려

- 큰 집계는 매번 실시간 산출 비용이 큼 → 캐싱 또는 materialized view
- KPI 카드: 30분 캐싱
- 모듈별 통계: 진입 시 + 1시간 캐싱
- 엑셀 다운로드: 비동기 처리 (큰 데이터셋), 완료 시 새올 알림
