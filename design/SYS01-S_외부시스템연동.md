## 4. 표시 항목

### 4.1 연동 상태 배지

| 상태 | 표시 | 색상 | 조건 |
|------|------|------|------|
| `HEALTHY` | ✓ 정상 | 초록 | 마지막 동기화 성공 + 예정 시점 미초과 |
| `DELAYED` | ⚠ 지연 | 주황 | 마지막 동기화 실패 또는 예정 시점 초과 |
| `FAILED` | ❌ 실패 | 빨강 | 연속 3회 이상 실패 |
| `DISABLED` | — 비활성 | 회색 | 정책상 비활성화 |

### 4.2 통합 통계 카드

| 항목 | 데이터 소스 |
|------|------------|
| 시스템별 성공·실패 | `sync_logs` GROUP BY sync_type, result (최근 24시간) |
| 재시도 대기 | `sync_logs WHERE result='RETRY'` 카운트 |
| 마지막 오류 | `sync_logs WHERE result='FAILED' ORDER BY created_at DESC LIMIT 1` |

| [규칙 편집] (매핑) | 매핑 규칙 편집 모달 | CEO |
| [정책 편집] | 동기화·자동 처리 정책 편집 모달 | CEO |

### 5.2 모달 별 액션

| 모달 | 액션 |
|------|------|
| 설정 수정 모달 | API 엔드포인트·인증·주기 등 수정 → audit_logs INSERT |
| 매핑 규칙 편집 | 추가·수정·삭제 + 우선순위 정렬 |
| 동기화 이력 [로그 보기] | sync_logs 상세 모달 (요청 페이로드 + 응답 + 변경 JSON) |
| [실패 큐 보기] | 새올·SMTP 실패 항목 목록 → 개별 재시도 가능 |

## 6. 자동 처리 로직

### 6.1 정기 동기화 스케줄러

```
매일 06:00 (KST):
  1. 행정포털 조직도 → ORG 동기화
  2. FMS → TGT 동기화
  3. 차세대 e호조 → CON·RSK·STF 동기화

각 단계 결과:
  - 성공: sync_logs INSERT (result='SUCCESS', delta 정보 JSON)
  - 실패: sync_logs INSERT (result='FAILED', error_message)
         + 재시도 큐 적재 (백오프 정책)
         + 운영자 알림 (3회 이상 실패 시 긴급)
```

### 6.2 새올 포틀릿·SMTP 비동기 워커

```
스케줄러: 1초마다 큐 폴링
  notifications / email_queue WHERE status IN ('NEW', 'RETRY')
  
처리:
  - 성공: status='SENT'
  - 실패: 백오프 (1분/5분/15분/1시간/6시간) → 5회 초과 시 status='FAILED'
```

### 6.3 운영자 알림 트리거

| 상황 | 알림 대상 | 카테고리 |
|------|---------|---------|
| 동기화 1회 실패 | SHM+ | system |
| 동기화 연속 3회 실패 | CEO/GM/SHM (긴급) | system 긴급 |
| 외부 API 인증 실패 | CEO/GM/SHM (즉시) | system 긴급 |
| 새올 포틀릿 누적 실패 (50건/시간) | SHM+ | system |
| 정기 동기화 예정 시점 초과 (1시간) | SHM+ | system |

### 6.4 모듈 영향 자동 처리

```
e호조 신규 계약 import 시:
  1. contracts INSERT (source='EHOJO_AUTO')
  2. contract_type='CONSTRUCTION' 이면:
     - risk_assessments 자동 INSERT (RSK 모듈)
     - 안전관리자 미달 검증 → contracts.status='BLOCKED' 가능 (STF 모듈)
  3. 담당부서 매칭 → 부서 SHM에게 알림

이 모든 정책은 본 화면 [정책 편집]에서 관리.
```

## 10. 데이터 스코프

- 본 화면은 시스템 전체 운영 정보 — 데이터 스코프 적용 없음
- 모든 외부 시스템 상태·이력 조회 가능 (CEO/GM/SHM)
- 단, 수정·테스트·매핑 규칙 편집은 CEO 또는 CEO/GM 권한 한정

## 11. 관련 DB 테이블

- `sync_logs` (DB-001 §3.1): 통합 동기화 이력
- `fms_sync_logs` (DB-001 §3.2): FMS 동기화 상세 이력
- `notifications`, `email_queue` (DB-001 §2.10·§2.11): 새올·SMTP 통계
- `audit_logs` (DB-001 §2.12): 설정 변경 감사

추가 마스터 테이블 (신규):

```sql
-- 외부 시스템 연동 설정 마스터
CREATE TABLE integration_configs (
  id              UUID PRIMARY KEY,
  system_code     ENUM('SSO','ORG_SYNC','FMS','EHOJO','SAEOL','SMTP') UNIQUE NOT NULL,
  endpoint_url    VARCHAR(500),
  auth_type       VARCHAR(50),
  auth_credentials JSON,                            -- 암호화 저장
  sync_schedule   VARCHAR(50),                       -- cron 표현식 또는 'REALTIME'
  is_active       BOOLEAN DEFAULT true,
  retry_policy    JSON,                              -- 백오프 설정
  updated_by      UUID REFERENCES users(id),
  updated_at      TIMESTAMP DEFAULT NOW()
);

-- 매핑 규칙 마스터 (각 시스템별 매핑)
CREATE TABLE integration_mapping_rules (
  id              UUID PRIMARY KEY,
  system_code     ENUM('SSO','ORG_SYNC','FMS','EHOJO') NOT NULL,
  rule_type       VARCHAR(50) NOT NULL,             -- 'POSITION_TO_ROLE', 'CONTRACT_TYPE', 'FACILITY_CATEGORY' 등
  source_value    VARCHAR(200) NOT NULL,
  target_value    VARCHAR(200) NOT NULL,
  priority        INTEGER DEFAULT 0,
  is_active       BOOLEAN DEFAULT true,
  metadata        JSON,
  created_at      TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_mapping_system_type ON integration_mapping_rules(system_code, rule_type);
```
