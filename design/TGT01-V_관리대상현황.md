## 4. 표시 항목

### 4.1 요약 카드

| 항목 | 데이터 소스 |
|------|------------|
| 시설 N개소 | `COUNT(targets WHERE target_type='FACILITY')` |
| 중대재해법 분류별 카운트 | `COUNT GROUP BY cdpa_category` (FACILITY 한정) |
| 시설물 종별 카운트 | `COUNT GROUP BY facility_grade` (FACILITY 한정) |
| 업무 N건 | `COUNT WHERE target_type='TASK'` |
| 사업 N건 | `COUNT WHERE target_type='BUSINESS'` |

> 데이터 스코프 적용된 결과 (CEO=전체, GM=부서산하, SM/SHM=담당, WKR=소속, SUB=계약, CON=위임).

### 4.2 미할당 안전관리자 안내 배너

```
WHERE target_type='FACILITY' 
  AND NOT EXISTS (target_assignees WHERE role='SAFETY_MGR' AND target_id=:t.id)
COUNT 산출 → N건 이상이면 배너 노출
```

### 4.3 목록 컬럼

| 컬럼 | 데이터 소스 | 비고 |
|------|------------|------|
| type 배지 | `targets.target_type` | 시설/업무/사업 |
| 관리대상명 | `targets.name` | 클릭 시 TGT02-D |
| 부서 | JOIN `departments.name` | |
| 중대재해/종별 | `cdpa_category` / `facility_grade` 배지 | FACILITY만 표시, TASK/BUSINESS는 "-" |
| 담당자 (관리책임자) | `target_assignees WHERE role='RESPONSIBLE'` JOIN `users.name` | AUTO/MANUAL 배지 |
| 상태 | `targets.status` 배지 | ACTIVE/INACTIVE/PENDING |

## 5. 필터 옵션

| 필터 | 옵션 | 비고 |
|------|------|------|
| 부서 | 전체 / departments 목록 | 권한 범위 내만 노출, URL `?department_id=` |
| type | 전체 / 시설 / 업무 / 사업 | 화면 내부 type 서브 탭과 연동 (필터 잠금) |
| 중대재해법 분류 | 전체 / 공중이용시설 / 원료·제조물 / 공중교통수단 | 시설 type 선택 시에만 노출 |
| 종별 | 전체 / 1종 / 2종 / 3종 | 시설 type 선택 시에만 노출 |
| 상태 | 전체 / 활성 / 비활성 / 등록대기 | |
| 검색 | `targets.name` 부분일치 | 디바운스 300ms |

## 7. 데이터 스코프 (권한별)

| 권한 | 보이는 범위 |
|------|----------|
| CEO | 전체 군청 관리대상 |
| GM | 본 부서 + 산하 관리대상 |
| SM/SHM | `target_assignees` 매칭 관리대상 |
| WKR | `target_assignees.role='WORKER'` 매칭 관리대상 |
| SUB | `external_user_access.access_type='CONTRACT'` 연결 관리대상 |
| CON | `external_user_access.access_type='CONSULTING'` 위임 관리대상 |

> 데이터 스코프 미들웨어(IA-001 §6.3)에서 자동 검증.

## 8. 자동 처리 로직

본 화면에서 직접 트리거되는 자동 처리는 없음 (등록·수정은 TGT02-F에서, 자동 매핑은 트리거에서).

## 9. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 데이터 스코프 권한 없음 | "접근 가능한 관리대상이 없습니다" | 빈 화면 + 안내 |
| 필터 조합 결과 없음 | "조건에 맞는 관리대상이 없습니다" | 빈 목록 + 필터 초기화 버튼 |

## 10. 관련 화면 흐름

```
TGT01-V

```
