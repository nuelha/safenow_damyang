### 5.2 위험성평가 탭

SFR-007 RSK 모듈 결과를 위젯 형태로 표시:
- 최근 위험성평가 1건 요약 (평가명·진행상태·기간·담당자)
- 위험요인 수·등급별 분포 카드
- [전체 평가 보기] → RSK02-L `?target_id=` 이동
- [+ 평가 등록] → RSK02-F (해당 target prefill)

### 5.3 점검·이행 탭

SFR-014 CMP + SFR-012 INS 결과를 위젯 형태로 표시:
- 진행 중 이행 항목 목록
- 시기도래 D-day 위젯 (안전NOW 기본 색상)
- [이행관리 전체 보기] → CMP02-L `?target_id=` 이동

### 5.4 개선조치 탭

SFR-003 IMP 결과를 위젯 형태로 표시:
- 진행 중 / 기한초과 / 완료 카운트
- 최근 개선조치 5건 목록
- [개선조치 전체 보기] → IMP01-L `?target_id=` 이동

### 5.5 업무문서 탭

해당 관리대상에 연결된 업무문서 목록 (DOC 모듈에서 `target_id` FK로 연결된 문서):
- 매뉴얼·절차서·점검표·증빙 등
- 카테고리별 필터
- [+ 문서 등록] → DOC02-F (해당 target prefill) — Phase 3

## 7. 자동 처리 로직

### 7.1 부서 이동 시 RESPONSIBLE 자동 재매핑

```
[부서 이동] 클릭 → 부서 선택 모달:
  확인 모달: "기존 관리책임자가 새 부서장으로 자동 변경됩니다. 진행하시겠습니까?"
  
  처리:
  1. targets.department_id UPDATE
  2. 기존 target_assignees(role='RESPONSIBLE', source='AUTO') DELETE
  3. 새 부서의 head_user_id로 target_assignees INSERT (source='AUTO')
  4. MANUAL 담당자(SAFETY_MGR 등)는 유지 — 부서 이동이 직무 변경 의미 아님
  5. 새 RESPONSIBLE에게 새올 포틀릿 알림
  6. 이전 RESPONSIBLE에게 새올 포틀릿 알림 ("부서 이동으로 매핑 해제")
  7. audit_logs INSERT
```

### 7.2 비활성화·활성화 처리

```
[비활성화]:
  확인 모달: "관리대상 '{name}'을 비활성화 처리하시겠습니까?
            진행 중인 위험성평가·이행관리 등은 유지되며 통계에서만 제외됩니다."
  
  처리:
  1. targets.status='INACTIVE'
  2. 진행 중 위험성평가/이행 항목은 유지 (담당자 알림)
  3. 대시보드 통계에서 제외
  
[활성화]: status='ACTIVE' 복원
```

### 7.3 FMS 정기 동기화 영향

```
FMS 동기화로 fms_id 매칭 항목 갱신 시:
  - cdpa_category, facility_grade, address 자동 갱신
  - 변경 사항은 audit_logs 기록
  - 본 화면 진입 시 마지막 동기화 시각 표시
```

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| 비활성화 후 신규 위험성평가 등록 시도 | 차단 ("비활성 관리대상에는 새 평가를 등록할 수 없습니다") |
| 부서 이동 시 권한 검증 | CEO/GM만 가능 |
| AUTO 담당자 해제 시도 | 차단 ("부서장 자동 매핑은 직접 해제할 수 없습니다. 부서장 변경으로만 갱신됨") |
