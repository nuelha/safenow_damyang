## 2. 진입 경로

- TGT01-V `[+ 관리대상 등록]` (신규)
- TGT02-D `[수정]` (수정 모드)
- TGT04-S 등록대기 행 [지정] 모달 후 자동 진입 (PENDING → ACTIVE 전환 시 상세 입력)

## 3. 단계 흐름

```
STEP 1. 기본정보
        ↓ type 선택에 따라 동적 필드
STEP 2. 시설형 추가정보 (type='FACILITY'일 때만)
        - FMS 연동 / 직접 등록 선택
        - FMS 연동 시: FMS ID 입력 → 정보 자동 불러오기
        - 직접 등록 시: cdpa_category, facility_grade, address 직접 입력
STEP 3. 담당자 (선택)
        - RESPONSIBLE은 자동 매핑되므로 생략
        - 안전관리자/보건관리자 등 추가 지정 가능
STEP 4. 검토 및 저장
```

> TASK / BUSINESS 유형은 STEP 2를 건너뜀 (1 → 3 → 4).

## 5. 입력 항목

### STEP 1

| 필드 | 타입 | 필수 | 검증 |
|------|------|------|------|
| 관리대상명 | TEXT | ✅ | 1~200자 |
| 유형(type) | RADIO | ✅ | 시설 / 업무 / 사업 |
| 담당부서 | SELECT (departments) | ✅ | 부서 트리 selector. SSO 동기화된 부서만 |
| 상태 | RADIO | ✅ | 활성 / 비활성 / 등록대기 (기본: 활성) |
| 설명 | TEXTAREA | ❌ | 최대 2000자 |

### STEP 2 (FACILITY만)

| 필드 | 타입 | 필수 | 검증 |
|------|------|------|------|
| FMS 연동 라디오 | RADIO | ✅ | FMS 연동 / 직접 등록 |
| FMS ID | TEXT | FMS 연동 시 ✅ | UNIQUE (다른 관리대상과 중복 불가) |
| 중대재해법 분류 | SELECT / 자동 | ✅ | 3종 ENUM |
| 시설물안전법 종별 | SELECT / 자동 | ✅ | 3종 ENUM |
| 주소 | TEXT / 자동 | ✅ | 우편번호 검색 컴포넌트 |

### STEP 3

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 안전관리자 | USER_SEARCH (multi) | ❌ | 자격증 보유 여부 인라인 표시 |
| 보건관리자 | USER_SEARCH (multi) | ❌ | 자격증 보유 여부 인라인 표시 |
| 관리감독자 | USER_SEARCH (multi) | ❌ | 부서장 자동 매핑되나 추가 가능 |

> 자격 미보유 사용자 선택 시 ⚠ 경고 인라인 표시 (강제 등록 가능 — STF04-V와 동일 정책).

## 7. 저장 동작

### 7.1 신규 등록

```
1. targets INSERT
2. STEP 2 입력값 반영 (FACILITY인 경우)
3. 트리거 자동 실행 (DB-001 §9.2):
   - 해당 부서의 head_user_id 조회
   - target_assignees INSERT (role='RESPONSIBLE', source='AUTO')
   - 새올 포틀릿 알림 (RESPONSIBLE에게)
4. STEP 3에서 입력한 담당자들 target_assignees INSERT (source='MANUAL')
5. 각 담당자에게 새올 포틀릿 알림
6. audit_logs INSERT
7. TGT02-D 이동
```

### 7.2 수정

```
1. targets UPDATE
2. department_id 변경된 경우:
   - 기존 RESPONSIBLE(AUTO) DELETE
   - 새 부서장으로 RESPONSIBLE(AUTO) INSERT
   - MANUAL 담당자는 유지 (이동 후에도 그대로)
   - 새 RESPONSIBLE / 이전 RESPONSIBLE에게 새올 포틀릿 알림
3. STEP 3 담당자 변경 사항 반영 (target_assignees INSERT/DELETE)
4. audit_logs INSERT
5. TGT02-D 이동
```

### 7.3 임시저장

```
status='PENDING' 으로 저장
모든 검증 우회 (부분 입력 허용)
TGT01-V 등록대기 필터에 노출
```

## 8. 자동 처리 로직

### 8.1 FMS 조회 시 자동 채움

```
[FMS 조회] 클릭:
  → IF-003 FMS API 호출 (fms_id 기준)
  → 응답에서 다음 필드 자동 채움 (read-only로 잠금):
     - cdpa_category (한글 → ENUM 매핑, IF-003 §3.4)
     - facility_grade (한글 → ENUM 매핑)
     - address
  → 조회 실패 시 안내 (직접 등록 가능 안내)
```

### 8.2 STEP 2 동적 노출

```
STEP 1 type RADIO 변경 감지:
  - FACILITY 선택 → STEP 2 노출 + 스텝 인디케이터 활성화
  - TASK / BUSINESS 선택 → STEP 2 숨김 + 스텝 인디케이터 비활성
```

### 8.3 부서장 자동 매핑 트리거 연계

```
관리대상 INSERT 직후 트리거(trg_targets_insert, DB-001 §9.2) 실행:
  - 해당 부서의 head_user_id로 RESPONSIBLE(AUTO) 자동 INSERT
  - 알림 자동 적재

본 화면 사용자는 RESPONSIBLE을 명시적으로 입력하지 않음.
```

## 9. 검증 규칙

| STEP | 검증 |
|------|------|
| STEP 1 | 관리대상명·type·담당부서 필수. 부서는 권한 범위 내만 가능 |
| STEP 2 | FACILITY일 때 FMS 연동 / 직접 등록 중 1개 필수. FMS ID UNIQUE 검증. 직접 등록 시 cdpa_category·facility_grade·address 필수 |
| STEP 3 | 자격 미보유 사용자 선택 시 ⚠ 경고 (강제 등록 가능). 권한 범위 외 사용자 선택 차단 |
| 전체 | 모든 STEP 검증 통과 시 [저장] 활성 |

## 10. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 필수 필드 누락 | "{필드명}을(를) 입력해주세요" | 인라인 에러 + STEP 이동 차단 |
| FMS ID UNIQUE 위반 | "이미 등록된 FMS ID입니다. 기존 관리대상을 확인하세요" | 인라인 + 기존 행 링크 |
| FMS 조회 실패 | "FMS에서 정보를 가져올 수 없습니다. 직접 등록으로 진행해주세요" | 토스트 + 직접 등록 모드 자동 전환 |
| 권한 범위 외 부서 선택 | "선택 권한이 없는 부서입니다" | 차단 |
| 저장 실패 (트랜잭션 오류) | "저장 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요" | 토스트 + 폼 유지 |
