## 4. 입력 항목

| 필드 | 타입 | 필수 | 비고 |
|------|------|------|------|
| 관리대상 | READONLY | — | TGT02-D에서 진입 시 prefill |
| 역할 | SELECT | ✅ | SAFETY_MGR / HEALTH_MGR / SUPERVISOR / HEALTH_DOCTOR / WORKER. RESPONSIBLE은 옵션 제외 (AUTO 전용) |
| 대상자 | USER_SEARCH | ✅ | 구성원 검색 모달 (부서 필터 + 자격증 표시) |
| 선임일 | DATE | ✅ | 기본값: 오늘 |
| 비고 | TEXTAREA | ❌ | 최대 500자 |

> 미지정 카드 `[지정]` 진입 시 역할 prefill (변경 가능).

## 5. 자격증 보유 검증

```
대상자 선택 시 즉시 자격증 검증 (ORG03-M §5와 동일 로직):
  SAFETY_MGR / HEALTH_MGR / HEALTH_DOCTOR 역할은 자격증 검증 수행
  SUPERVISOR / WORKER 역할은 자격증 검증 생략

결과 인라인 배지:
  ✅ 자격증 보유 (자격증명)
  ⚠ 자격증 미보유 (강제 지정 시 경고 모달 노출)
```

## 6. 버튼·링크 액션

| 요소 | 위치 | 액션 | 조건·비고 |
|------|------|------|---------|
| `[×]` 닫기 | 모달 우상단 | 모달 닫기 (변경사항 alert) | |
| 역할 selector | 본문 | 역할 변경 시 자격증 요구사항 표시 | RESPONSIBLE 옵션 제외 |
| `[구성원 검색...]` | 대상자 영역 | 구성원 검색 모달 | 권한 범위 내 사용자 |
| `[취소]` | 푸터 좌측 | 모달 닫기 | |
| `[지정]` | 푸터 우측 | 저장 동작 (§7 참조) | 필수 필드 입력 시 활성 |

## 7. 저장 동작

```
1. 필수 필드 검증
2. 동일 (target_id, user_id, role) UNIQUE 검증 → 중복 시 에러
3. 자격 검증:
   - 자격 보유 시: 바로 진행
   - 자격 미보유 (SAFETY_MGR/HEALTH_MGR/HEALTH_DOCTOR):
     경고 모달:
       "{사용자명}은(는) 자격증을 보유하지 않았습니다.
        강제 지정하시겠습니까?"
       [취소] [강제 지정]
4. target_assignees INSERT (
     target_id, user_id, role, source='MANUAL',
     appointed_at, is_primary=false)
5. 자격 미보유 강제 지정 시 warnings INSERT
6. 본인에게 새올 포틀릿 알림 (category='assignment',
   "{관리대상명} {역할명}로 지정되었습니다")
7. audit_logs INSERT
8. TGT02-D 담당자 영역 새로고침 → 모달 닫기
```

## 8. 검증 규칙

| 항목 | 검증 |
|------|------|
| 역할 | RESPONSIBLE 제외 5개 역할 중 1개 |
| 대상자 | 비활성 사용자 차단 |
| 중복 매핑 | (target_id, user_id, role) UNIQUE — 중복 시 에러 ("이미 동일 역할로 지정된 사용자입니다") |
| 자격 미보유 | SAFETY_MGR/HEALTH_MGR/HEALTH_DOCTOR 역할 시 경고 모달 |
| 권한 범위 | 본 관리대상의 부서 또는 권한 범위 내 사용자만 |

## 9. 오류 처리

| 케이스 | 메시지 | 처리 |
|--------|--------|------|
| 중복 매핑 | "이미 동일 역할로 지정된 사용자입니다" | 인라인 에러 |
| 자격 미보유 + 강제 지정 시 | 경고 모달 (§7 3단계) | [취소] / [강제 지정] |
| 비활성 사용자 선택 | "비활성 사용자는 지정할 수 없습니다" | 차단 |
| 저장 실패 | "지정 처리 중 오류가 발생했습니다" | 토스트 + 모달 유지 |
