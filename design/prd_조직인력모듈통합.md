## 0. 작업 개요

### 0.1 목적

담양군 중대재해통합관리시스템 v1.3 기준 GNB 구조에서 두 가지 통합 작업 수행:

1. **조직 모듈 명칭 정리**: GNB "전담조직" → "조직"으로 변경 (LNB 구조 유지)
2. **안전보건 인력 통합**: 분산된 인력 관리 화면(ORG03-V, STF01-V, STF02-V)을 단일 GNB "안전보건 인력" 모듈로 통합

### 0.2 작업 범위

| 분류           | 작업                                                                                          |
| -------------- | --------------------------------------------------------------------------------------------- |
| 산출물 갱신    | IA v1.4, RTM v1.3, 화면설계서 4개                                                             |
| 화면 파일 이동 | ORG03-V → STF04-V, ORG03-M → STF04-M                                                          |
| URL 변경       | `/organization/dedicated` → `/staff/dedicated`                                                |
| 프로토타입     | 기존 안전NOW 프로토타입(`https://nuelha.github.io/safenow_new`)에서 해당 화면 GNB·라우팅 갱신 |

```
[Phase 1] 산출물 스펙 갱신 (MD 파일)
   ↓

```

## 1. 변경 사항 상세

### 1.1 GNB 변경 (Before → After)

**Before (v1.3)**

```
GNB > 전담조직 (ORG)
   ├─ 조직도 (ORG01-V)
   ├─ 구성원 (ORG02-L)
   └─ 전담인력 (ORG03-V)   ← 이동 대상

GNB > 안전관리자 인력 (STF)
   ├─ 인력 현황 (STF01-V)
   ├─ 공사별 선임 (STF02-V)
   └─ 설정 (STF03-S)
```

**After (v1.4)**

```
GNB > 조직 (ORG)                          ← 명칭 변경 (전담조직 → 조직)
   ├─ 조직도 (ORG01-V)
   └─ 구성원 (ORG02-L)

GNB > 안전보건 인력 (STF)                  ← 명칭 변경 + 통합
   ├─ 전체 현황 (STF01-V)                  ← 기본 진입
   ├─ 부서별 전담인력 (STF04-V) ⭐ 신규     ← 구 ORG03-V 이전
   ├─ 공사별 선임 현황 (STF02-V)
   └─ 설정 (STF03-S)
```

## 7. Claude Code 실행 가이드

### 7.1 작업 순서

```bash
# 작업 디렉토리 진입
cd /mnt/user-data/outputs/

# Phase 1 — 영향 파일 식별
grep -rln "ORG03" 화면설계서/
grep -rln "전담조직" .
grep -rln "안전관리자 인력" .

# Phase 2 — STF04-V 신규 파일 생성 (ORG03-V 내용 이전 + 헤더 갱신)
# (ORG03-V_*.md 파일을 view한 뒤 헤더·SFR·URL만 변경하여 STF04-V로 복사)

# Phase 3 — 영향 화면 일괄 갱신 (sed 사용 가능)
sed -i 's|ORG03-V|STF04-V|g' 화면설계서/TGT02-D_*.md
sed -i 's|ORG03-M|STF04-M|g' 화면설계서/CON*.md
sed -i 's|/organization/dedicated|/staff/dedicated|g' 화면설계서/*.md

# Phase 4 — IA·RTM 갱신 (별도 view → str_replace)

# Phase 5 — ORG03 원본 파일 삭제 (이전 확인 후)
rm 화면설계서/ORG03-V_*.md  (있다면)
rm 화면설계서/ORG03-M_*.md  (있다면)

# Phase 6 — 검증
grep -rn "ORG03" 화면설계서/ | grep -v "변경 이력\|이전"
# 결과: 0건이어야 함
```

### 7.2 주의 사항

1. **헤더 자동 치환 금지**: 화면 ID 치환 시 헤더(`# ORG03-V`)는 sed로 자동 치환되지만, **변경 이력 섹션의 "구 ORG03-V" 같은 history 기록은 보존**해야 합니다.

2. **변경 이력 보존**: 각 파일에 변경 이력 표가 있다면 v1.1 항목 추가 (덮어쓰기 금지).

3. **모달 정의 위치**: ORG03-M(전담인력 선임 모달)이 ORG03-V 안에 통합 정의되어 있는지, 별도 파일인지 view로 먼저 확인 후 작업.

4. **TGT02-D, CON 모듈 영향**: 자동 갱신 가능하지만, 본문 설명 텍스트(예: "전담조직 모듈로 이동") 같은 자연어 표현도 점검 필요.

5. **프로토타입 변경은 별도 작업**: 본 PRD의 Phase 3는 별도 코드베이스 작업으로, 산출물 갱신과 분리해서 진행.
