```
┌────────────────────────────────────────────────────────┐
```

#### 관련근거 등록 모달

| 필드               | 타입         | 필수 | 옵션                                          |
| ------------------ | ------------ | ---- | --------------------------------------------- |
| 구분               | SELECT       | ✅   | 법령 / 시행령 / 시행규칙 / 고시 / 지침 / 기타 |
| 법령명             | TEXT         | ✅   |                                               |
| 조항               | TEXT         | ✅   | 예: 제4조, 제4조 제1항                        |

| 관련 점검항목 연결 | MULTI_SELECT | ❌   | 점검표 항목과 매칭                            |

### 1-5. DY-POL-04 경영방침 점검표 (강화)

기존 POL04-S 기반. o/x/텍스트 입력 유형 및 관련근거 직접 등록 기능 추가.

| 유형 코드 | 표시명      | 입력 방식              |
| --------- | ----------- | ---------------------- |
| `yn`      | O/X         | O 또는 X 선택 (라디오) |

| `score`   | 점수        | 숫자 입력              |

#### 점검표 항목 구성

```
경영방침 점검표

연도: [2026 ▼]   점검자: [박안전]   점검일: [2026-04-15]

번호 │ 점검항목            │ 관련근거          │ 결과     │ 비고
─────┼─────────────────────┼───────────────────┼──────────┼────────
  1  │ 안전보건목표 설정    │ 중대재해처벌법     │ [O] [X]  │
     │ 여부               │ 시행령 제4조 [+]   │          │
─────┼─────────────────────┼───────────────────┼──────────┼────────
  2  │ 경영방침 문서화      │ 산안법 제14조 [+]  │ [텍스트] │
     │ 및 공표 여부        │                   │ [________│
─────┼─────────────────────┼───────────────────┼──────────┼────────
  3  │ 목표 달성률 검토     │ -                 │ [점수 __]│
     │                     │                   │ /100     │
```

**[+] 관련근거 직접 등록**

- 항목 옆 [+] 버튼 클릭 → DY-POL-03 관련근거 등록 모달 오픈
- 등록 완료 시 해당 항목에 자동 연결

**점검표 항목 데이터 정의**

| 컬럼                 | 타입                        | 설명            |

| `item_name`          | VARCHAR(200)                | 점검항목명      |

### 1-7. DY-POL-06 온나라 결재 안내 모달 (신규)

**배경**: 행안부 공문(정보공개제도과-1574, 2026.4.16.)에 따라 SSO/문서뷰어 연계 불가.
결재 정보 전달 후 온나라에서 별도 처리하는 방식으로 연계.

#### 모달 구성

```
┌──────────────────────────────────────────────────┐
│                                                  │
│  ✅ 결재 요청이 완료되었습니다.                   │
│                                                  │
│  온나라 업무관리시스템에서                         │
│  결재를 진행해 주세요.                            │
│                                                  │
│  결재가 완료되면 이 화면에서                       │
│  결재 상태가 자동으로 업데이트됩니다.              │
│                                                  │
│  ※ 온나라 시스템에 별도 로그인이 필요합니다.      │
│                                                  │
│          [확인]   [온나라 바로가기 →]             │
│                                                  │
└──────────────────────────────────────────────────┘
```

| 버튼            | 동작                                 |
| --------------- | ------------------------------------ |
| 확인            | 모달 닫기. 결재 상태 → `결재중` 표시 |
| 온나라 바로가기 | 온나라 시스템 URL 새 탭 열기         |

#### 온나라 결재 연계 흐름

```
[담양 중대재해시스템]
결재 요청 버튼 클릭
  ↓
결재 정보 생성 (기안 제목 / 내용 / 결재선)
  ↓
온나라 결재 API 호출 (결재 정보 전달)
  ↓
DY-POL-06 안내 모달 표시
  ↓
approvals 테이블 → status = 'PENDING' (결재중)

[담당 공무원]
온나라에 별도 로그인 → 결재 처리 (승인/반려)

[담양 중대재해시스템]
결재 상태값 수신 (폴링 또는 Webhook)
  → status = 'APPROVED' 또는 'REJECTED'
  → 목록에 결재 상태 자동 업데이트
```

#### 결재 상태값 정의

| 상태 코드  | 표시명   | 배지 색상 |
| ---------- | -------- | --------- |
| `DRAFT`    | 임시저장 | 회색      |
| `PENDING`  | 결재중   | 주황      |
| `APPROVED` | 승인완료 | 초록      |
| `REJECTED` | 반려     | 빨강      |

### 2-6. 담양군 행정포털 조직도 연동

**프로토타입**: 예시 데이터로 구성 (API 연동 없음)
**실제 개발**: 담양군 행정포털 조직도 API 연동

#### 연동 방식 (실제 개발 시)

```
담양군 행정포털 조직도 API 호출
  ↓
부서·담당자 정보 조회
  (기관명, 부서명, 직급, 직위, 성명, 연락처)
  ↓
우리 시스템 조직도에 동기화
  - 연동된 인원: 아이콘 표시 (🔗 연동됨)
  - 직접 등록한 인원: 별도 표시 (✏ 직접등록)
  ↓
통합 조직도에서 두 데이터 함께 관리
```

#### 통합 관리 원칙

| 케이스               | 처리 방법                                      |
| -------------------- | ---------------------------------------------- |
| 행정포털에 정보 있음 | API 연동 자동 로드 + 역할/주요업무만 추가 입력 |
| 행정포털에 정보 없음 | 직접 등록 기능으로 수동 입력                   |
| 두 데이터 공존       | 통합 조직도에서 함께 표시. 출처 구분 표시      |

## 3. DB 설계 참고

### legal_references (관련근거)

```sql
id                UUID PK
site_id           UUID FK
category          ENUM('law','decree','rule','notice','guideline','etc')
law_name          VARCHAR(200)
clause            VARCHAR(100)
summary           VARCHAR(300)
detail            TEXT
created_by        UUID FK
created_at        TIMESTAMP
```

### policy_legal_mappings (방침-관련근거 매핑)

```sql
id                UUID PK
policy_id         UUID FK (safety_policies)
legal_ref_id      UUID FK (legal_references)
```

### checklist_legal_mappings (점검항목-관련근거 매핑)

```sql
id                UUID PK
checklist_item_id UUID FK
legal_ref_id      UUID FK (legal_references)
```

### policy_attachments (방침 첨부파일)

```sql
id                UUID PK
policy_id         UUID FK
file_url          VARCHAR(500)
file_name         VARCHAR(200)
description       VARCHAR(300)   ← 파일 설명 (신규)
created_at        TIMESTAMP
```

### policy_postings (게시현황)

```sql
id                UUID PK
policy_id         UUID FK
posting_location  VARCHAR(200)   ← 게시 장소
posted_date       DATE
memo              TEXT
status            ENUM('active','closed')
photos            JSON           ← 게시 사진 URL 배열
created_by        UUID FK
```

### approvals (결재 — 온나라 연계)

```sql
id                UUID PK
reference_type    VARCHAR(50)    ← 'safety_policy' 등
reference_id      UUID
title             VARCHAR(200)
content           TEXT
approver_info     JSON           ← 결재선 정보
status            ENUM('DRAFT','PENDING','APPROVED','REJECTED')
onnara_request_id VARCHAR(100)   ← 온나라 기안 ID
submitted_at      TIMESTAMP
approved_at       TIMESTAMP NULL
reject_reason     TEXT NULL
```

### dedicated_organizations (전담조직)

```sql
id                UUID PK
site_id           UUID FK
dept_name         VARCHAR(100)   ← 부서명
org_name          VARCHAR(100)   ← 기관명 (담양군청)
parent_id         UUID FK NULL   ← 상위 부서
dept_type         ENUM('dedicated','prevention_target','etc')
headcount         INTEGER
is_synced         BOOLEAN        ← 행정포털 연동 여부
synced_at         TIMESTAMP NULL
```

### dedicated_personnel (전담인력)

```sql
id                UUID PK
org_id            UUID FK (dedicated_organizations)
name              VARCHAR(50)
rank              VARCHAR(50)    ← 직급 (사무관, 주무관 등)
position          VARCHAR(50)    ← 직위 (과장, 팀장 등)
main_duty         VARCHAR(200)   ← 주요업무
contact           VARCHAR(20)
role              ENUM('ceo','safety_chief','safety_manager','supervisor','staff')
is_dept_head      BOOLEAN
is_synced         BOOLEAN
```
